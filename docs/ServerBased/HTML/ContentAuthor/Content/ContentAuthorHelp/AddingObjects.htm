Adding objects to content

Content Author lets you add objects to content frames based on the type of content you are creating. The person who designs the communication sets the content type by using a specific type of content frame in the design. Depending on the content frame that is available in the communication, you can create text or graphic messages, and clauses. If the frame allows you to use clauses, then you can also add sections to organize clauses and other sections.

Graphic messages can contain a mixture of text and graphics, and users can place content anywhere within the content frame. Text messages and clauses can contain the same elements that graphic messages can contain (text, images, and so on); the difference is that all of the elements in a text message or clause are placed inline like text.

Adding objects to a content frame

  1. Click a button on the object palette to add an object to the frame:

    Object Button Things to keep in mind
    A text box
    • For text messages and clauses:Content Author automatically places text boxes at the next available cursor position in the content frame.
    • For graphic messages: After you click the text box button, you can click and drag inside the message canvas to customize the size and placement of the text box.
    A table
    • For text messages and clauses: Content Author automatically places tables at the next available cursor position in the content frame.
    • For graphic messages:Content Author automatically places tables in the center of the graphic message frame.

    Tip: If you're using tables in a communication that will be sent to customers in page-based output (such as PDF), you can adjust table splitting settings within the text message or clause to avoid unexpected page breaks as content flows from one page to the next.
    An image
    • For text messages and clauses: Content Author automatically places images at the next available cursor position in the content frame.
    • For graphic messages:Content Author automatically places images in the center of the graphic message frame.
    A variable

    Note that the cursor must be in an editable text area before the Show variables button becomes active. When you click , the list of variables that are available for the content scrolls into view. You can add variables to your content just the same as you can add text. If you are working with a clause that will be editable in Empower Editor, you will see additional options on the variable card. Select the variable and then click on the variable card to set interactive properties for the variable.

    If your cursor is in an editable text area and the Show variables button is not available, then there are no variables included in the communication. If you have questions about the variables that are available in a communication, you should talk to the person who designed it.

  2. Using the tools and settings that are available for the design object, make any needed updates.

  3. Use the object palette to add other objects as needed.

  4. Click on the right side of the content frame to save the changes and return to either Communications view or to the parent section of the section that you're working with.

As you add objects to your content, remember that the person who designed the communication determines the design elements that are available to you in Content Author (including images, fonts, styles, variables, and rules). If you have questions about content types or design elements, you should talk to the person who designed the communication.

Using table splitting to control content flow across pages

If you're using tables in a communication that will be sent to customers in page-based output (such as PDF), you can configure tables or rows within tables to split so that you avoid awkward breaks as content flows from one page to the next. Keep in mind that you can set tables and rows to split only in text messages and clauses, because content in graphic messages does not flow.

Setting tables to split

You can set any table to split, but it is particularly useful on longer tables. For example, suppose that you're adding content for a letter that will be sent to customers as a PDF. The letter contains a frame at the bottom half of the first page for content, and a full-page frame on the second page for overflow content from the first page. If the content you're adding consists of a short paragraph followed by a long table that is not set to split, then the frame on the first page of the letter will contain nothing but the short paragraph followed by white space, and the entire table will move to the second page. If you set the table to split, however, Content Author automatically splits the table as needed in order to minimize large areas of white space.

To set a table to split:

  1. With a text message or clause open in Authoring view, place the cursor anywhere inside the table.

  2. In the Embedded table properties card, click to open the general properties for the table.

  3. Select the Allow table splitting check box.

  4. Make other changes as needed, then click to save the content.

Setting rows within tables to split

Sometimes, even if you set a table to split, single rows that contain a lot of content can cause page breaks that leave too much white space in a frame. If you have a table with rows that contain a lot of content, you can set those rows to split so that Content Author automatically inserts breaks that minimize white space.

To set a row within a table to split:

  1. With a text message or clause open in Authoring view, place the cursor anywhere inside a row that you want to split.

  2. In the Embedded table properties card, click to open the properties for the row.

  3. Select the Allow row splitting check box.

    Note: Table splitting must be enabled in order for rows to split. If you select Allow row splitting and you later remove splitting for the table, then Content Author automatically clears any row splitting options that you previously set. Additionally, you cannot use the Specify row height setting at the same time as the Allow row splitting setting. If you enable either setting, then Content Author automatically clears the other.

  4. Make other changes as needed, then click to save the content.

Controlling header and footer placement on tables that split

If you need to include header and footer rows in your embedded table, you can control how those rows behave when the table splits onto multiple pages. Specifically, you can use the dynamic table settings to specify where headers and footers appear when an embedded table splits. Dynamic table settings are available only for tables that are set to split. Because content in graphic messages does not split and flow, you cannot use dynamic table settings with graphic message content.

Tip: A table is considered embedded if it has been placed in a text message or a clause.

To configure header and footer placement for an embedded splitting table:

  1. With a text message or clause open in Authoring view, select the row that you want to use as a header or footer. You can set multiple rows to be headers or footers, but you must configure each row separately.

  2. In the Embedded table properties card, click to open the general properties for the table.

  3. To enable header and footer options on the row properties, select the Dynamic table check box.

  4. To open the properties for the row, click .

  5. In the Row type list, select a setting for the row:

    • Standard—A row that is not a header or footer

    • Header (H)—A row that appears at the top of the table, but only on the first page where the table appears

    • Footer (F)—A row that appears at the bottom of the table, but only on the last page where the table appears

    • Repeating header (H+)—A row that appears on the top of a table on each page where the table appears

    • Repeating footer (F+)—A row that appears at the bottom of a table on each page where the table appears

    • Repeating header except first (H-)—A row that appears at the top of a table on each page where the table appears, except for the first page where the table appears

    • Repeating footer except last (F-)—A row that appears at the bottom of a table on each page where the table appears, except for the last page

    The symbol for the row type that you selected appears on the right side of the selected row.

    Note: You can set any row in a table to be a header or footer. No matter which row that you specify as a header or footer, the placement of the row does not change as you edit the table. For example, if you make the second row of a table the header row, that row will remain the second row as you edit the table. In preview and final output, however, the header will appear at the top of the table as expected.

Things to remember when using table splitting

As you work with splitting tables and splitting rows, you might run into a couple of issues where tables don't behave as you would expect:

  • Different font sizes across the cells of a single row: If you have a single line of text in each cell of a row that is set to split, but you use different font sizes within the cells of the row, you might see unexpected results if the row is in a position to split. For example, sometimes some of the smaller text will remain on one page while the larger text will flow to the next page. In that case, you should either use a consistent font size across all cells in a row, or else remove splitting options from the row.

  • Text on the same line as a splitting embedded table: If you place a splitting table inline after text characters (including blank spaces), then the table breaks to a new line in HTML preview and output. The table will remain on the same line as the text characters in PDF preview and page-based output.
