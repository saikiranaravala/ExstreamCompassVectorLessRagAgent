Making content editable in Empower Editor

In some cases, content that you create or edit in Content Author will be made available later for editing in Exstream Empower Editor. Like Content Author, Empower Editor is a browser-based application that lets users edit content that has been designed and packaged in Exstream Design and Production and that will eventually be sent to customers. However, where Content Author deals with smaller, more self-contained pieces of content (like graphic messages or clauses), Empower Editor provides a more word processor-like user interface that lets users edit longer-form documents that tend to incorporate many different pieces of content.

As an example of a typical use case, suppose that you work in the corporate office for an insurance company and that you are putting together the body of a letter that will be sent to customers who are making insurance claims. You need to include text that can be personalized for individual customers, as well as boilerplate legal information about the company that will remain the same for all customers. In that scenario, you might make one clause editable in Empower Editor for the part of the letter that can be personalized for each customer. But, because you do not want the boilerplate legal information changed, you would make the other clause non-editable in Empower Editor. You would use Content Author to set the properties for each clause, and then the insurance agents in field offices who use Empower Editor to update the letter can edit only the clause that you specified as editable.

If you have questions about whether content will eventually be used as part of an Empower document, you should talk to the person who designed the communication.

Making content editable in Empower Editor

  1. Open a clause in Authoring view.

  2. Set interactive properties for a text selection, or for the whole clause:

    • For a text selection: To set interactive properties for text within the clause, select the text that will be editable, and then click on the toolbar. You can set interactive properties for multiple text selections within a clause, but they must be contained within a single paragraph because interactive properties cannot span multiple paragraphs.

    • For the clause as a whole: To set interactive properties for a clause, in the Clause properties pane, click . If you set interactive properties for the clause as a whole, you cannot set different interactive properties for individual text selections within the clause. However, within a clause that is editable in Empower, you can add an embedded text box and then set interactive properties on text selections within the text box.

    Tip: When you make a text selection interactive, the text will appear with an underline bracket. If the text selection is also highlighted, then you will not be able to use the ENTER key to add a line break, even though the cursor appears to be at the end of the line. If the text selection is highlighted and you want to add a line break, you must press the right arrow key until the text selection is no longer highlighted. At that point, you can press the ENTER key to add a line break.

  3. Specify interactive options as needed:

    Decide whether users can make changes to content in Empower Editor
    1. Select an option from the Changes allowed list:

      • No changes—This is the default option. Clauses that use this setting will not be editable in Empower Editor.

      • Change is optional—Use this option if you want users to be able to decide whether to change the content in the clause.

      • Change is required—Use this option if the content in the clause must be changed before it is included in customer output. You might select this option if, for example, your clause contains only placeholder text that should not be sent to customers.

    2. If you want Empower users to be able to change both the content and the text formatting in the clause, select the Allow changes to text formatting check box.

    Important: If you embed objects such as tables or images in a clause, those embedded objects will not be editable in Empower Editor; however, Empower users will be able to delete those items.

    (Inline text only) Add a selection control

    If you want to streamline the editing experience for users in Empower Editor, you can add selection controls to provide more options for entering data. Selection controls—such as drop-down lists, calendars, and free-form entry boxes—let users select from multiple options, instead of simply altering a text string.

    Select an option from the Add selection control list:

    • None—Content Author will not apply a selection control to the selected text.

    • Use drop-down list—Specify multiple text options that users will select from a list. If you select this option, do the following to create a list:

      1. Click Create list.

      2. In the Interactive list dialog box, add list items as needed, and then click Save.

        The list items are associated with the editable area and will appear in the Empower document when you create output. You can edit or delete the list by clicking Edit list on the Apply Empower settings dialog box.

    • Calendar—Users can select a date from a graphical representation of a calendar.

    • Calendar and free-form entry—Users can either select a date from a graphical representation of a calendar, or else they can type the date in a text box.

    Include an optional tooltip

    If you want to provide instructions to users who are editing the content that you create, you can include an informational tooltip that will appear when users point to the content in Empower Editor. The tooltips that you specify for Empower Editor will look much the same as the tooltips in Content Author. To see an example, simply point to a button on the Content Author toolbar.

    Enter the information for your tooltip:

    • Tooltip title—Add a title, which will appear in bold above the body of the tooltip in Empower Editor.

    • Tooltip description—Add the body of the tooltip, which will appear below the title of the tooltip in Empower Editor.

    Add editable variables

    You can also allow end users in Empower Editor to change variable values or make selections based on variables. Several of the options that are available for interactive variables are the same as for other interactive elements (for example, configuring which changes are allowed, setting the editing options, and adding tooltips). One difference, however, is that you can specify a list of options from which end users can select text, number, and currency variables. You can also select the data-entry type for date variables.

    To add variables that end users can edit in Empower Editor:

    1. Double-click the variable in the content to select it, and then click .

    2. To make the variable editable, select an option from the Changes allowed list.

    3. Specify a list of options or a data entry type for the variable:

      Task Action
      Specify a list of options for text, number, or currency variables

      1. Select the Use drop-down list check box, and then click Create list.

      2. In the Interactive list dialog box, add list items as needed, and then click Save.

        The list items are associated with the variable and will appear in the Empower document when you create output. You can edit or delete the list by clicking Edit list on the card for the variable.
      Specify a data-entry type for date variables

      Select an option from the Data-entry type list:

      • Calendar—Users can select a date from a graphical representation of a calendar.

      • Calendar and free-form entry—Users can either select a date from a graphical representation of a calendar, or else they can type the date in a text box.

      • Free-form entry only—Users can type the date in a text box.
    (Clauses only) Set visibility in Empower Editor

    By default, clauses from Content Author are always visible in Empower Editor. However, you can allow end users to control the visibility of clauses in an Empower document, which also determines which clauses are included in customer output. If you change the default Content visibility setting for a clause, then a check box is added next to the clause in Empower that lets end users choose whether to show or hide the content in the Empower document and in customer output. End users can show or hide hidden content in an Empower document by clicking a button on the Menu bar.

    Select a visibility option for the clause:

    • Always show content—This is the default setting. If you use this setting, then the clause will be visible to end users in Empower Editor, and it will appear in customer output.

    • Initially show content—When end users open an Empower document, if the clause qualifies for placement, then Empower Editor will show the clause. End users can select or clear the check box next to the clause to determine whether to include or exclude the clause in customer output.

    • Initially hide content—When end users open an Empower document, if the clause qualifies for placement, then Empower Editor will initially hide the clause. End users can show or hide hidden content in an Empower document by clicking a button on the Menu bar. With the clause showing, end users can then select or clear the check box next to the content to determine whether to include or exclude the clause in customer output.

  4. Add content to the clause as needed, and then click next to the clause to save your changes and return to Communications view.

Creating an Empower selection group from a section

Selection groups let Empower Editor users easily add or remove blocks of content that were created in Content Author. Within Content Author, a section correlates to a selection group in Empower, and a clause correlates to a block of content within the selection group. With a selection group in place, end users click an icon in the Empower document that opens the selection group dialog, where they can simply click to include or exclude clauses in the Empower document.

Selection groups are particularly useful if you have blocks of approved content that are available to be included in an Empower document, but you want to let Empower users decide which blocks to include. For example, suppose that you work in the marketing department for your company, and you're in charge of the language used to describe the products that your company sells. After you complete your work, salespeople in regional offices will use the marketing-approved language to create sales brochures for potential customers. However, the salespeople need to target the brochures for individual customers. In that scenario, you can use selection groups to make the marketing-approved content available to salespeople, who can then use Empower Editor to build sales brochures that include only the product information that each customer needs to see.

Keep in mind that you can't create an Empower selection group if the target section is configured to allow nested sections. However, you can create a selection group from a child section within another section. If you use a section to create an Empower selection group and you later add a nested section within that section, the selection group won't appear in any Empower documents that include the parent section.

To create an Empower selection group from a section:

  1. In Communications view, create a section and then add any clauses that you want to make available in Empower Editor. Alternatively, you can also add clauses to the section after you open it in Authoring view in step 2.

  2. On the card for the frame, double-click the section, or click to open the section in Authoring view.

  3. In the section properties, clear the Allow nested sections check box, and then click to open the selection group properties for the section.

  4. Choose an option to determine how the selection group will behave in Empower Editor:

    • None—This is the default option. If you select this option, the section will not be used as a selection group in Empower.

    • Select one clause (optional)—If you select this option, Empower users can add a single clause from the section to their Empower document, but they are not required to.

    • Select specific number (required)—If you select this option, end users will be required to include the number of clauses that you specify. When you select this option, the box to specify a number becomes available.

    • Select range (required)—If you select this option, end users can add the number of clauses within the range that you specify. When you select this option, the boxes to specify a range become available.

  5. Include an optional tooltip that will appear when end users point to the selection group icon in Empower Editor:

    • Tooltip title—Add a title, which will appear in bold above the body of the tooltip in Empower Editor.

    • Tooltip description—Add the body of the tooltip, which will appear below the title of the tooltip in Empower Editor.

Identifying editable areas in Empower Editor

When Theme view is enabled in Empower Editor, the following colors indicate editable areas so that end users can find them more easily:

Color Type of editable area
  Optional editable areas
  Optional editable areas that have been edited
  Required editable areas
  Required editable areas that have been edited

To toggle theme view in Empower Editor, end users can click on the View Menu bar.
