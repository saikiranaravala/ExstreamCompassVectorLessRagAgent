Adding accessibility features to your content

You can use the accessibility features in Content Author to control and optimize the way that customer output will be read by assistive technology tools such as screen readers and text-to-speech converters. For example, suppose that you are creating correspondence that will be emailed to customers in PDF format. Visually impaired customers will then use screen readers or text-to-speech converters to read the correspondence.

Important: If your communication has not been configured to include accessibility settings, then any accessibility settings that you specify in Content Author will not be honored in customer output. If you have questions about whether accessibility settings will be included in customer output, you should talk to the person who designed the communication.

Content Author provides options that enable you to add compliant accessibility features to individual design objects (such as text boxes, tables, and images), as well as to messages and clauses.

Add accessibility features to individual design objects

  1. In Authoring view, with the object selected in the frame, click on the card for the object.

    Note: With an object selected, you might see accessibility options on multiple object cards. For example, if you click to place your cursor inside of a text box, then accessibility options will be available for both the text paragraph and for the text box itself.

  2. Configure the accessibility options as needed:

    • Read options—These options allow you to specify how screen readers will process text in customer output.

      • Do not read text—If you select this option, then no accessibility options will be included in the output for the object.

      • Read alternate text—This option allows you to add alternate text for the object. Alternate text is content that a screen reader reads in place of the text within an object or to describe a non-textual object, such as an image. If you include alternate text for an object that contains text, such as a text box, then screen readers will read the alternate text instead of the text in the text box. If you select this option, then the Alternate text field becomes available for you to add the alternate text that screen readers will read.

      • Read object text—This is the default option and specifies that screen readers will read the text within the object, such as the text in a text box.

    • Accessibility language—This option specifies the language that screen readers will use for pronunciation rules in customer output.

      Complete list of available accessibility languages

      • Default—This option is the default option and specifies that the default language for the communication will be used as the accessibility language. Keep in mind that there might be cases where you want to use a separate accessibility language for an individual design object, or for an individual text paragraph that is within a design object. For example, suppose that you are producing a customer letter that uses English as the default accessibility language, but you want to include one paragraph in Spanish. In this case, you can set Default as the option for all other content, but specify Spanish as the accessibility language for that particular paragraph so that the screen reader will read the text using the correct Spanish pronunciation.

Add accessibility features to hyperlinks

Specifying accessibility options for hyperlinks ensures that assistive technology tools can identify and process hyperlinks separately from regular text. For example, suppose that you need to add an inline hyperlink that uses regular (non-URL) display text. In that case, you might want to include alternate text on the hyperlink so that sight-impaired customers can easily determine that the display text represents a hyperlink. Without alternate text, screen readers would read the hyperlink as if it were no different from the text that surrounds it.

To add accessibility features to hyperlinks:

  1. In Authoring view, type or select the text that you want to use for a hyperlink, then click .

  2. Configure the accessibility options as needed:

    • Read options—These options allow you to specify how screen readers will read the hyperlink in customer output.

      • Do not read text—If you select this option, then no accessibility options will be included in the output for the hyperlink.

      • Read alternate text—This option allows you to add alternate text for the hyperlink. Alternate text is content that a screen reader reads to describe the hyperlink. If you select this option, then the Alternate text field becomes available for you to add the alternate text that screen readers will read.

      • Read object text—This is the default option and specifies that screen readers s will read the text within the message or clause.

    • Accessibility language—This option specifies the language that screen readers will use for pronunciation rules in customer output.

      Complete list of available accessibility languages

      • Default—This option is the default option and specifies that the default language for the hyperlink will be used as the accessibility language.

Add accessibility features to messages and clauses

You can specify accessibility options for a message or clause as a whole if you want assistive technology tools to call particular attention to a message or clause. For example, suppose that you are working on a customer letter and you create a clause that contains legal information about the company that you work for. You can flag the clause with alternate text that identifies the content of the clause so that sight-impaired customers can decide whether to have their screen reader read all of the legal information as it is processing the letter. Keep in mind that the accessibility features that you include on a message or clause apply only to the message or clause and not to the objects that they contain. So, for example, a text paragraph does not inherit the accessibility text that you set on the message that holds it.

Important: Accessibility tagging at the message or clause level does not apply to HTML customer output.

To add accessibility features to messages and clauses:

  1. In Authoring view, with a clause or message open, click on the property card for the clause or message.

  2. Configure the accessibility options as needed:

    • Read options—These options allow you to specify how screen readers will read text in customer output.

      • Do not read text—If you select this option, then no accessibility options will be included in the output for the message or clause.

      • Read alternate text—This option allows you to add alternate text for the message or clause. Alternate text is content that a screen reader reads to describe the message or clause. If you select this option, then the Alternate text field becomes available for you to add the alternate text that screen readers will read.

      • Read object text—This is the default option and specifies that screen readers will read the text within the message or clause.

    • Accessibility language—This option specifies the language that screen readers will use for pronunciation rules in customer output.

      Complete list of available accessibility languages

      • Default—This option is the default option and specifies that the default language for the communication will be used as the accessibility language.
