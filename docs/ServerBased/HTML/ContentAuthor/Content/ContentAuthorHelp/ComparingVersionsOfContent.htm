Comparing changes between versions

If you're a content reviewer, Content Author provides tools in Authoring view that let you compare text changes between versions of content in clauses and text messages. Specifically, you can compare the latest version that has been submitted for approval against the previously approved version. Keep in mind that version comparison functionality won't be available if a previously approved version doesn't exist, if the latest version has not been submitted for approval, or if you're working with a graphic message.

The comparison tools clearly highlight text changes between versions so that you can inspect each one either within the context of the current content, or side by side with the previously approved version in the Version comparison pane.

To use the version comparison tools:

  1. Open a text message or clause that has been submitted for approval and that has a previously approved version.

  2. In Authoring view, on the toolbar, click . If there are no text changes between versions, you'll be notified and then return to Authoring view.

  3. To select the first or last change in the content, click or , respectively.
    Content Author highlights the change in the color that is associated with the user who made it, and then places its number to the left of the frame.

  4. Click or to go to the next or previous change, respectively.

  5. (Optional) With a change highlighted, click to open the Version comparison pane so that you can compare each change side by side against the previously approved version. You can also view the properties for the submitted version on the cards in the property pane.

    Tip: If you place your cursor in the text on either side of the Version comparison pane, Content Author automatically places a cursor at a corresponding point on the other side of the pane. You can then use the left and right arrow keys on your keyboard to move simultaneously through the content in both versions. Keep in mind, however, that the dual cursors move at the same time only when there is corresponding content on both sides. So, for example, if there are two sentences in the previously approved version, but the latest version contains two new sentences between the original two sentences, then the cursor will remain stationary in the previously approved version as you move through the new sentences in the current version.

  6. When you are finished reviewing the changes, click or to return to the current version in Authoring view.

If you're satisfied with the content as it stands, you can use the workflow tools to approve the content. If, however, you use the version comparison tools and find a change that is either incorrect or has been omitted, you can reject the content and add a note explaining your decision. However, remember that if the content is not in draft state, you cannot add any revisions or annotations to the content itself. If you need to add specific tracked revisions and revision comments, you must first return the content to draft state. After you do that, all of the revision tracking tools in Content Author will once again be available to you.
