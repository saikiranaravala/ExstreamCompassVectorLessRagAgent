Design environment

The design environment provides the user interfaces (or views) that you use to set up the structure of your communication and to create and modify your communication design.

The design environment comprises the following views—Structure view, Design view, and Simulation view.

Structure view

You use Structure view to create communication objects to make up the basic structure of a communication. In Structure view, you use the Structure map to create communication objects, and use the Properties panel to update the properties of the communication and individual communication objects. The structure of your communication is displayed in the Structure map, and can contain the following objects:

  • Documents are containers for all of the designs that are required to create a personalized communication. You use documents to logically group together individual designs in a communication, and to specify the output order for those designs. When the Exstream engine processes a communication, the generated output contains documents in the order that you see in the Structure map. You can click and drag a document to place it in the correct output order, and also set timing rules and document rules for the engine in the document properties. To produce valid output, your communication must contain at least one document that contains at least one design.

  • Designs contain the content that a customer sees. Designs contain both static content as well as the content that can change based on customer data. When the Exstream engine processes a communication, the generated output contains designs in the order that you see in the Structure map. You can click and drag a design to place it in the correct output order, and also add targeting rules to include or exclude the design in the output. To produce valid output, your communication must contain at least one design. You can create designs for page-based and HTML-based output. The type of design that you choose to create depends on your output needs.

  • Components are reusable design objects that are stored in CAS as named objects and can be inserted in multiple designs. In the Structure map, you can see the components that are used in each design. You cannot create new components in Structure view; they must be initially created in a design. You can then use Structure view to update its properties or to open the component directly in Design view and edit it.
Design view

You use Design view to design layouts for customer communications. In Design view, you can add design objects and insert variables to create personalized communications. Design view contains the following user interface elements:

  • The design toolbar lets you edit and configure design objects, apply formatting to text, undo or redo actions, save your design, and so on.

    The toolbar displays different user options depending on the type of design object that you are editing, and depending on the fonts and style sheets that are available in your communication. In addition, you can use the hamburger menu () to access the help, select design units, display version information, or return to the Content Launcher.

  • The object palette is a floating toolbar that lets you select objects to add to a design.

    By default, the object palette appears on the left edge of Design view. To change the position of the object palette on your screen, position your mouse cursor on the top edge of the toolbar to change the cursor into a move handle, and then click and drag the toolbar to the desired location.

  • The properties panel is a dynamic panel that displays different property cards based on the type of object that is selected. You can use the properties panel to view and modify the properties for a selected object, add variables to your design, and even specify some properties for the design itself, such as targeting rules for that design. The properties panel is always located on the right edge of Design view.

  • The design area contains the layout of your page or email design. The size of the design area varies depending on the type and properties of the design that you are creating. You use the design toolbar, object palette, and properties panel to add and format design elements in this area.
Simulation view

You use Simulation view to preview your communications before a production engine run. Simulation view contains the following user elements:

  • The preview toolbar lets you select the type of preview you want to see and includes the option to see the engine messages generated during the preview engine run. The toolbar displays different options depending on the design types in your communication. In addition, you can use the hamburger menu () to access the help, display version information, or return to the Content Launcher.

  • The Simulation list contains a list of the simulations that are available for your communication and lets you switch between different simulations to change the sample data that is used in the communication preview.

  • The Data panel lists the available variables and lets you change the sample variable values as needed for a particular preview.

  • The preview area contains the communication preview. You use the preview toolbar, the Simulation list, and the Data panel to customize the preview options that determine what is displayed in this area.
