Previewing communications

Before you generate production output, you can use Simulation view to run the engine in preview mode to see your communication as it would appear in the final output. Depending on the types of designs that are present in your communication, you can preview page-based output as well as HTML-based output for email and web designs.

Prerequisites for using Simulation view

Make sure that you meet the following prerequisites for previewing a communication:

  • Output queues and design views—The Design Manager application that is associated with your communication must contain appropriate output queues and design views. The preview options that are available to you in Simulation view are determined by the output queues and design view settings. For web and email designs, the design view and the associated preview device in the packaged application must also match the corresponding container design label in the HTML (email) or HTML5 output object. If a design type does not have a preview device associated with it, then the preview option for that design type will be hidden.

  • Correctly configured Exstream engine settings—The Exstream engine plug-in configuration in your Communications Builder project must reference the Exstream package file that was created from the Design Manager application for your communication. The engine plug-in must also be configured to process Communications Designer designs, and reference the CAS ID for the Exstream application object that corresponds to your Communications Designer communication.
  • Running Communications Server application—The export file that is created from the Communications Builder project must be deployed to a running Communications Server application.

  • Sample file with data mappings, and if required, simulations—Make sure that at least one mapped sample file is uploaded to the common asset service (CAS). If you are using a sample file that is mapped to a Design Manager data source, or if you are using multiple sample files, each sample file must also have a corresponding simulation in Workshop that is associated with the Exstream application template for your communication.

If you are not sure that these requirements have been met, you might need to talk to the person who created the design pack or to a system administrator. For information about configuring your environment and enabling previews, see Designing for Exstream Web Applications in the OpenText Exstream Design and Production Online Help.

Understanding how Simulation view works

In Simulation view, the engine runs in preview mode to create output that is displayed in the browser window. The variable values that are displayed in the preview are populated using a sample file that was mapped to the data source for your communication. The sample file that is used depends on the data source that is associated with the communication:

  • If your communication uses a data source that was mapped in Design Manager, then the sample file that is attached to the simulation for your Exstream application template is used to populate variable values.

  • If your communication uses a data source that was mapped in Communications Designer, then the sample file that is associated with the data source is used to populate variable values.

Mapping a sample file to a data source associates individual variables with specific data areas in that file. In Simulation view, the engine uses the data in the sample file to populate variables and customize the communication preview. Keep in mind that if your sample file contains data sections for multiple customers, the engine populates the variables with the data from the first customer section only.

In some cases, such as when you use rules to include or exclude content based on customer data, using different customer data can significantly change the output. You can run the engine with multiple sample files to see how your communication looks with different variable values. This functionality is especially useful if the included documents or designs might change when the sample customer data changes, such as when regulations in a particular state require additional content to be included in a communication. For example, suppose that you have a communication that contains a document that is targeted for customers who live in the state of Florida. If the sample file that you are using has Georgia as the default state value, then the document will not appear in the communication preview in Simulation view.

You can use either a Design Manager data source or a Communications Designer data source to map a sample file. You then upload any sample files that you want to use to CAS. To make the sample files available for previewing your communication, you must create a simulation for each sample file in Workshop, and associate the simulation with the Exstream application template for your communication.

Tip: If you are using only a single sample file that is mapped to a Communications Designer data source, then you do not need to create a simulation for that sample file.

When using multiple sample files to create simulations, keep in mind that the different sample files must have a structure similar to the sample file that was originally mapped to the data source for your communication. Sample files with a mismatched structure might produce unexpected results.

The first time that the engine runs in Simulation view, it uses the default simulation that is associated with the application template. If no simulation is specified as the default, then the engine uses the simulation that was created first.

Note: Simulations are created in Workshop. You might need to talk to a system administrator or another user with access to Workshop to configure simulations. For information about simulations, see OpenText Exstream Workshop User Guide.

Using Simulation view to preview a communication

In Structure view, click Simulation to open Simulation view and display the communication preview for the design view that was set as the default in the Design Manager application.

To preview a communication:

  1. Click the following preview buttons to see your communication in the respective output formats:

    • Pages—Displays a preview of the page-based output for the communication
    • Emails—Displays a preview of the HTML (email) output for the communication
    • Web—Displays a preview of the HTML5 output for the communication

    Note: Design views must be configured correctly in the Design Manager application for your communication. If the Design Manager application does not contain a preview device associated with a particular design type in the design view settings, then the option to preview that design type will not be available in Simulation view.

  2. For email and web designs, you can also choose to preview your communications as they would appear on mobile or desktop devices. To preview the communication for a specific device type, click the following buttons on the toolbar:

    • Desktop view button —Shows the communication as it would appear on desktop devices. This is the default option.
    • Mobile view button —Shows the communication as it would appear on mobile devices
  3. To see the engine messages that were generated during the preview engine run, click the Messages button .

  4. To change the simulation that is used during a preview engine run, click the Simulation list, then select the simulation that you want to use. The engine runs in preview mode and uses the sample data from the selected simulation to populate the variables in the communication preview.

  5. By default, the communication preview uses the variable values from the sample file that is associated with the selected simulation. To use custom variable values in the preview, in the Data panel, change the sample variable values as needed, and then click Run. The variable values are updated in the communication, and the updated variable values are shaded green in the Data panel.

  6. To reset the variables to their original sample values, click Reset. Variable values are also reset to the original values if you select a different simulation.

When you are finished previewing your communication, click Exit simulation, just above the Data panel. Simulation view closes and you are returned to Structure view. You can also use the hamburger menu () to return directly to the Content Launcher.
