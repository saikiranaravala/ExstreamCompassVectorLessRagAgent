Inserting design objects

In Design view, you use the object palette to add different design objects to your design. Design objects are not stored as individual objects in CAS; the objects exist only in the design. However, if you select a design object and add it to the component library, the component is stored in CAS and can be reused in other designs. You can use Structure view to see the components that are associated with each design.

To add an object to your design, click the corresponding button on the object palette.

The following table lists the object palette buttons and the corresponding design objects, including a brief description of placement considerations for each design object:

Button Design object and placement considerations

Insert text box

You can place a text box directly in a design, or you can embed it in another design object, such as a table cell or another text box. To easily identify a text box in your design, you can also specify a name for it on the General card in the object properties panel. Adding text boxes is one of the primary ways to add text to your design.

Placing a text box in a page design

In page designs, when you add a text box, the cursor changes to a drawing cursor, and you can click and drag in the design area to customize the size and placement of the text box on the page. Use the text box properties to make adjustments to the text box size and placement later, if needed.

Placing a text box in an email design or a web design

In HTML-based designs, text boxes are automatically placed in the selected container. You cannot specify the dimensions of a text box that is placed directly in a container. The width of the text box corresponds to the width of the selected container. Because HTML-based designs are responsive by default, the height of the text box depends on the content that is placed within it.

Embedding a text box in another object

When you embed a text box in another object (such as a table cell or another text box), then the text box is automatically placed inline at the cursor position in that object. The default height and width of the embedded text box is 96 pixels. You cannot change the position of the embedded text box from the text box properties; it is always inserted inline at the cursor position. However, you can use the margin settings of the parent object to change the position of the embedded object.

Insert table

You can place a table directly in a design, or you can embed it in another design object, such as a table cell or a text box. To easily identify a table in your design, you can also specify a name for it on the General card in the object properties panel.

Placing a table in a design

In all designs, tables are automatically placed in the center of the page or selected HTML container. You can choose to specify the number of rows and columns before you place the table, or use the table options in the design toolbar to modify the number of rows and columns. Use the cards in the Table properties panel to make changes to the format and placement of the table.

Embedding a table in another object

When you embed a table in another object (such as a table cell or a text box), then the table is automatically inserted inline at the cursor position in that object.

Insert image

You can place an image directly in a design, or you can embed it in another design object, such as a table cell or a text box. To easily identify an image in your design, you can also specify a name for it on the General card in the object properties panel.

Placing an image in a design

When you add an image, it is automatically placed in the center of the page or selected HTML container.

Embedding an image in another object

When you embed an image in another object (such as a table cell or another text box), then the image is automatically inserted inline at the cursor position in that object.

Insert frame

Frames are used to designate content flow areas, or to indicate space that is reserved for use in Content Author. The default graphic message templates for Content Author message frames, and for any text or graphic message types, are defined in the Design Manager application for your communication. For more information about graphic message templates, see OpenText Exstream Design and Production Online Help.

You cannot embed frames in other design objects, or embed other objects in frames; you must place a frame directly in a page or email design. You cannot add frames to web designs. To easily identify a frame in your design, you can also specify a name for it on the General card in the object properties panel.

Placing a frame in a page design

In page designs, when you add a content frame, the cursor changes to a drawing cursor, and you can click and drag to customize the size and placement of the frame on the page. When you add a message frame, the frame is automatically placed in the center of the page, and the dimensions of the frame correspond to the dimensions of the default graphic message template. You can modify frame dimensions on the General card in the properties panel.

Placing a frame in an email design

In email designs, frames are automatically placed in the selected container. The width of a frame always corresponds to the width of the selected container. For content frames, the default height is 96 pixels. For message frames, the default height corresponds to the height of the default graphic message template. If a graphic message template is not specified, then the height for message frames defaults to 96 pixels. You can modify the height of the frame on the General card in the properties panel.

Insert component

Components are design objects that are stored and managed from the central CAS repository, but can be added to designs by all users. Components can be created from text boxes, tables, or images. You cannot embed components within other design objects; you must place a component directly in a design.

When you click the Insert component button, the Component browser opens. To add a component to your design, select the component that you want to add from the component library, and click Insert.

Placing a component in a page design

When you add a component to a page, it is automatically placed where it was originally placed on the page when it was first created.

Placing a component in an email design or a web design

When you add a component to an HTML-based design, it is automatically placed in the center of the selected container.

Insert variable

Variables let you use customer data to add unique text to your design so that you can create a single communication that can be customized for different customers. You can also use variables to display other information such as page count. To enable the option to add variables, you must place your cursor in an editable text area, such as a text box or a table cell. If your cursor is in an editable text area and the option to show variables is not available, then there are no variables available in this communication. To modify the set of available variables, you can update the design pack that is associated with the communication or create new variables in Communications Designer.

When you click the Insert variable button, the Variables property panel scrolls into view and displays a list of available variables. To add a variable to your design, click the variable name in the list.
