Creating accessible page designs

You can use the accessibility features in Communications Designer to control and optimize the way that customer output is read by assistive technology tools such as screen readers and text-to-speech converters. You can add compliant accessibility features to individual design objects (such as text boxes, tables, and images), as well as to inline text such as hyperlinks.

Specifying accessibility settings for page designs

When you specify the accessibility settings for a page design in Structure view, the selected setting applies to how that page is handled by a screen reader. You can also update these accessibility settings for your page designs in Design view.

On the Accessibility tab, update the following settings as needed:

  1. In the Read options list, specify how the this object should be read by screen readers:

    • Read object text—Read the text that appears on the page. This is the default option.
    • Read alternate text—Read alternate text in place of the text on the page. You must also specify the alternate text in the Alternate text box.
    • Do not read text—Skip this page when reading the communication.

  2. In the Accessibility language list, select the language that you want a screen reader to use for pronunciation rules.

Specifying accessibility settings for design objects

Select the design object, and then in the object properties panel, click the Accessibility button.

Note: With an object selected, you might see accessibility options on multiple object cards. For example, if you click to place your cursor inside of a text box, then accessibility options will be available in the properties panel for both the text box and the text paragraph.

On the Accessibility card, configure the following settings as needed:

  1. In the Read options list, specify how the object should be read by screen readers:

    • Do not read text—Skip this object when reading the design.
    • Read alternate text—Read alternate text in place of the object text. You must also specify the alternate text in the Alternate text box.
    • Read object text—Read the text as it appears in this object. This is the default option.

  2. In the Accessibility language list, select the language that you want a screen reader to use for pronunciation rules.

  3. In the Read order box, specify the order in which the selected object will be read when the communication is processed by a screen reader.

When you specify accessibility settings for design objects, keep in mind that some special considerations apply to the following objects in page-based output:

  • Components—The Read alternate text setting on a component in an email design or in a web design is not a supported setting for HTML-based output, and is not honored in the output. If you apply the Read alternate text setting to a component in a page design, the Exstream engine ignores this setting if that component is used in email or web designs.

  • Frames—For frames, you can specify only the read order. For content and message frames that will be used in Content Author, the accessibility options for the content of those frames is specified by the Content Author user. For flow frames, the accessibility options for the content in the frame are determined by the accessibility options that are set on the content that overflows into that frame.

  • Tables—In addition to the basic accessibility settings, you can also specify how an assistive technology tool should process the accessibility tags for the table. For example, you might use a table with invisible borders as a design element to control placement of objects. In that scenario, you can use this setting to specify that screen readers ignore table structure tags (such as header rows) when processing the table in customer output.

    To specify how the table tags are processed, in the Read text as list, select one of the following options:

    • Table tags and content—Screen readers will announce table structure tags (such as header rows) when processing tables in customer output.
    • Content only—Screen readers will ignore table structure tags and will read only the content within each cell when processing tables in customer output.
Specifying accessibility settings for hyperlinks

Specifying accessibility options for hyperlinks ensures that assistive technology tools can identify and process hyperlinks separately from regular text.

For example, suppose that you need to add an inline hyperlink that uses regular (non-URL) display text. In that case, you might want to include alternate text on the hyperlink so that sight-impaired customers can easily determine that the display text represents a hyperlink. Without alternate text, screen readers would read the hyperlink as if it were no different from the text that surrounds it.

Note: When you specify accessibility options for a hyperlink that is set on a design object, the hyperlink accessibility options do not override the accessibility options set for the contents of that design object.

To specify the accessibility options for a hyperlink, select the hyperlink, and then on the design toolbar, click the Hyperlink button .

In the Insert hyperlink dialog box, configure the following settings as needed:

  1. In the Read options list, specify how the a screen reader should read the hyperlink:

    • Read object text—Read the hyperlink text. This is the default option. If the hyperlink is placed on a design object, the screen reader will read the URL.
    • Read alternate text—Read alternate text in place of the hyperlink. You must also specify the alternate text in the Alternate text box.
    • Do not read text—No accessibility options will be included in the output for the hyperlink.

  2. In the Accessibility language list, select the language that you want a screen reader to use for pronunciation rules.
