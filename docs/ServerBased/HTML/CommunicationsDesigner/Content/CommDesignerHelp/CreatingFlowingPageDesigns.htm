Creating flowing page designs

Because Exstream lets you create a single design that is then customized for each customer with that person's data, the way that you design a page reflects this flexibility. For example, if you want to include a table that lists each of the customer's calls during the month, you add one table to your design. Then, when the engine runs and populates the table with the customer's unique data, the table can grow and change to reflect the data. Therefore, when you create a design, it is important to keep in mind that the page will change during engine processing, based on the inclusion or exclusion of design objects on the page and the data that is available for the customer.

You can use flow settings for your designs to accommodate design objects within those designs that are set to flow.

Specifying flow options for page designs

When you create a page design, you can specify whether that page will be used as a flow page, or how any overflow content on that page should behave. In Structure view, you use the Properties panel to update the flow settings for a page design.

  • To specify that a page will be used as a flow page, on the General tab, select the Flow page only check box. Flow pages are included in the Structure map of a communication, but do not appear in the output unless they are used.

  • To specify how the overflow content on a page should behave, on the Flow tab, in the Flow type list, select one of the following options:

    • Ignore overflow—Select this option to specify that overflow content should be ignored. When you produce output, if the engine encounters overflow content, it will issue an error and ignore that content, and continue processing the design. The overflow content will not appear in the output.
    • Copy this page—Select this option to specify that the current page should be duplicated to contain overflow content. You should select this option only if the page will contain a flow frame. This option lets you create a standard flow page that will contain all of the overflow content. For example, if the original content is a transactional table that can span many pages, then this specifies that the page will be duplicated as many times as necessary to display all of the transactional data.

    • Flow to another page—Select this option to specify the flow page that is set up to contain the overflow content. When you select this option, the Flow page list is enabled. This list contains all of the page objects that are available in the current application domain. In the Flow page list, select the destination flow page for overflow content.

Specifying flow options for objects in a page design

In Design view, you can specify flow settings for individual text box, table, and image objects. When you set a text-based object to flow, the overflow content from that object will flow to the flow page that you specified for the design in Structure view. Design objects can also be set to change position according to the flow of objects above them.

To specify flow options:

  1. In the object properties panel, click the Flow properties button to open the Flow card for the selected object.

  2. To set the selected object to split and flow to a different page, select the Allow splitting check box. Selecting this check box also updates the following settings:

    • In the Flow card, the Flow limit box is displayed. This setting specifies how far down the page the selected object can move or grow to accommodate growing content or other growing objects on the page.
    • For text boxes, in the General card, the Autosize height check box is selected, and the text box height will now change dynamically based on the content that it contains. If you clear the Autosize height check box, the Allow splitting check box will also be cleared.

  3. Select the Ignore for relative flow check box if you do not want the position of the object to change, regardless of the growth or position change of other objects in the design.

  4. In the Move relative to above object list, you can choose how the position of the selected object should change when other objects flow:

    • No—The position of the selected object is not affected by the changing position of other objects on the page.
    • Above—The position of the selected object is relative to the position of the object above it. If that object moves or grows, then the position of the selected object also changes.
Setting up frames for content flow

When you create a flow page, you must add content flow frames to that page to receive the overflow content from other pages. If multiple objects in a design are set to flow, the object to which other objects are relative is placed in a valid flow frame first.

For example, if a text box at the top of a page is set to flow, and a table below the text box is also set to flow, then the text box content is placed in the available flow frame first. By default, the order in which you add flow frames to a flow page is the order in which they will receive content. However, because the order in which you design flow frames might not be the order in which you want them to be filled, you might need to change the fill order for frames to suit your design. If you change the order of one frame, the order of the other frames is updated automatically.

To add a content flow frame to a page:

  1. On the object palette, click the Insert frame button.

  2. In the Frame type box, select Content flow area, then click OK and draw the frame in the design area.

  3. On the General card in the frame properties panel, specify the following settings:

    • In the Fill order box, enter the order in which this content frame will be filled with overflow content, compared to all other frames on the page.

    • Optionally, modify the dimensions and position of the frame on the page.
