Configuring data sources

To enable the Exstream engine to connect customer data to your communications, you must map a data source using a sample data file. When you map a data source, you associate a variable with each data area in the sample file, which creates data maps that the engine uses to find customer data in your production data file. The engine then uses that data to populate variables and customize your communications.

Communications Designer uses the data source to identify which sample file to use for a particular data map. The same sample file can be used in multiple data sources, and be mapped differently for each data source with which is it associated. The data source defines several aspects of the file to be mapped. For example:

  • The name of the data source as it should appear in Workshop

  • The sample data file to be used for the data map

  • An associated design pack

  • The type of data file it represents

  • The file format

  • The production data source file

In Communications Designer, you use the Communication asset management page to set up a data source for use in your communications, and associate a sample file and design pack with a data source. You then use Mapping view to create variables, map the data source, and update variable properties. In Mapping view, you can map the data source using variables from the following sources:

  • Communications Designer—You can create new variables directly in Mapping view. These variables are stored in CAS and are available to all Communications Designer data sources. CAS variables that are mapped in a data source are also available to any communication that uses that data source.

  • Design pack—When you create a Design Manager application for use with Communications Designer, you can include variables in the design pack. When you select a design pack to set up a data source in Communications Designer, the variables in that design pack are available for use in that data source. For more information about configuring a design pack, see Designing for Exstream Web Applications in OpenText Exstream Design and Production Online Help.
