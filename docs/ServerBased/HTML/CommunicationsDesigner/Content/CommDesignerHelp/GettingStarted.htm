Getting started

Communications Designer is an intuitive web application with a modern approach to rapid development of personalized customer communications that are fulfilled using the Exstream engine.

Communications Designer includes an asset management environment and a design environment. Authorized users can use the asset management environment to configure design and communication resources—such as data sources, variables, and fonts—and add these resources to the common asset service (CAS). Designers can then use these resources to create design layouts for customer communications in the design environment.

Communications Designer communications can leverage resources that are set up in Design Manager—such as fonts, styles, data files, variables, and output queues—as well as the resources that are directly created in Communications Designer. When you fulfill your communications, the engine orchestration features in Communications Server also let you take advantage of all of the available content integration and delivery capabilities in the Exstream Platform.

You can also use Communications Designer in conjunction with other Exstream web applications to produce customer communications. For example, you might want to use the design environment in Communications Designer to create a design that will be used as a template for customer correspondence in Content Author. On the other hand you can use Communications Designer to design an Empower document, and then include the edited Empower document in another communication to create customer output.

Prerequisites for using Communications Designer

In Communications Designer, you can use resources that are set up in Design Manager. To create and customize new communications, these resources must be set up correctly and accessible to you in Communications Designer.

Note: You might need to talk to an Exstream system administrator or another user to complete the prerequisites for using Communications Designer.

The following table provides an overview of the steps that must be completed to set up Design Manager resources and make them available for use in Communications Designer:

Step 1: Create a Design Manager application for use with Communications Designer.

A Design Manager application is required to create and fulfill a Communications Designer communication. The Design Manager application must contain the following objects:

Design pack

Design packs are Design Manager objects that include resources that are available to customize a communication in the design environment, such as fonts, style sheets, colors, rules, and variables. The Design Manager application must contain at least one design pack.

Design views

Design views determine which design categories are available in Communications Designer. Each design view corresponds to a specific preview device that is configured for use with a particular design type. By default, applications are automatically configured to use the standard design view for page designs. To create email or web designs in Communications Designer, the application must also include additional design views for those design types. If a design view is not specified for a particular design type, then the design category and simulation option for that design type will be hidden in Communications Designer.

Output queues

The application must contain at least one appropriate output queue to produce output from the communication.

Data files

The application must contain at least one customer driver file to produce output from the communication. You can choose to map the sample data for your driver file in Design Manager or in Communications Designer. To use Communications Designer to map a sample file, the customer driver file in your application must use the Communications Designer format.

Step 2: Package and upload the Design Manager application to the common asset service (CAS).

When you upload a package file to CAS, an associated template object is created with the same name as the Design Manager application that you packaged. A template is a representation of the Design Manager application in CAS and is referenced by Communications Designer communications that are based on that application. In addition, you can attach simulations to a template, and then use those simulations to preview the connected communications and themes. A template is visible in Workshop in a specific domain if a running Communications Server application in that domain references the associated package file.

After you upload the package file to CAS, you can use the associated design pack to create a new communication in Communications Designer.

OpenText Exstream Design and Production Online Help

Step 3: Create a Communications Builder project that uses the Exstream engine and references the uploaded package file.

In the Communications Builder project, you must configure the Exstream engine to process designs from Communications Designer. Be sure to select the Communications Designer fulfillment check box and specify the CAS ID for the Exstream application object that corresponds to your Communications Designer communication. If you are creating the project prior to creating a communication, you can use a variable to dynamically obtain the CAS ID from Workshop.

You can choose to add resources such as a sample data file or images to CAS by adding them to the resource set in your project, or you can add those resources to CAS by uploading them directly to Workshop.

Sample data files are required to set up simulations, which allow you to preview communications in Communications Designer.

Step 4: Configure a Communications Server application.

In Control Center, create a Communications Server application and deploy the export file for the Communications Builder project, and then start the application.

A running Communications Server application is required to preview and fulfill Communications Designer communications.
