Adding frames for business content

After you have created your initial design layout, you can consider where you want to insert frames in your design to place additional content that is customized for individual customers—such as legal notices, coupons, news articles, marketing messages, and so on. Business users can then use the authoring features in Content Author to add content to the frames that you inserted in your design.

Selecting the right frame type for your design:

Communications Designer lets you add two types of frames to your designs to indicate the space that is reserved for Content Author users—content frames and message frames. Each frame type can hold text, images, and tables—they just differ in how content is placed and in how it flows. As you set up your design, you need to think about the types of changes that end users will need to make to the content before each communication makes its way into customers' hands. The following list provides a brief overview and use case for each type of frame that you can use:

  • Message frames can support both graphic and text messages:

    • Graphic messages let end users have more control over where objects are placed. Because the size of graphic message frames is static, graphic messages are best suited for graphics-heavy pieces of content that need to fit within a specific area and that do not need to flow. To create a graphic message frame, a graphic message template must have been included in the associated Design Manager application.
    • Text messages can contain the same elements that graphic messages contain (text, images, and so on). However, the difference is that all elements in a text message are placed inline like text. Unlike graphic messages, text message frames can grow to accommodate extra content and can split and flow to the next text message frame. Text messages are best suited for text-heavy content that is limited in scope, like sidebars or callout boxes.
  • Content flow frames let end users add flowing paragraph content (known as "sections" and "clauses" in Content Author) to customer communications. As with text messages, clauses treat non-text content (images and tables) like text and place it inline. Content flow frames are best suited for correspondence content that might need to flow to one or more overflow frames. Content flow frames are the only type of frames that you can use for a Content Author communication that will be edited in Empower Editor.
Adding frames for use in Content Author

To add a frame to your design:

  1. On the object palette, click the Insert frame button.

  2. In the Frame type box, select the type of frame that you want to insert:

    Note: You can add content flow areas to page and email designs only. When you add a content frame to an email design, it is automatically configured to be used in Content Author. You cannot add frames to web designs.

    • Content flow areas—Inserts a content flow frame. By default, for page designs, content flow frames are configured to receive overflow content in page designs. Sections and clauses can be added only to a content flow frame that is marked for use in Content Author. When you add a content flow frame to your design, you must specify if the frame will be used as a flow frame or if it will be used in Content Author.

    • Messages—Inserts a message frame that lets you add text or graphic messages to a Content Author communication. Message frames are used exclusively for indicating space that is reserved for Content Author users. You cannot add any content to a message frame in Communications Designer.
Specifying frame properties

For a frame object to be valid for use in Content Author, you must specify the following settings on the General card in the frame properties panel:

  1. For a message frame, in the Messages allowed list, select the type of message content that can be placed in the frame.

    • If you select Text, then in the Text message type list, then select the message type for text messages that can be placed in that frame, or leave it as Any.
    • If you select Graphic, then in the Graphic message template list, then select the message template that will be used by the graphic messages that are placed in that frame. Graphic messages are not available in HTML-based designs.
    • If you select Text and graphic, then you must select the appropriate options in both the Text message type and the Graphic message list.
  2. For content frames in page designs, select the Use in Content Author check box.
  3. For both content and message frames, in the Frame name box, enter the name that you want to use for the frame in Content Author.
  4. Optionally, you can also specify the dimensions of the frame, and in page designs, the position of the frame on the page.

Using roles to control user access to frames in Content Author

You can use Exstream roles to control user access to frames in Content Author. Because each Exstream role is assigned to a specific OTDS group, this functionality is particularly useful if you're creating a communication that will be edited by users in different OTDS groups, and you want users within each groups to see only the frames that they're allowed to edit in Content Author.

To use roles to control user access to a Content Author frame: 

  1. In Design view, select a frame, then click the User access button in the Frame properties panel. Keep in mind that the User access button is visible only if the selected frame is a Content Author frame.
  2. In the User access card, click the Add role button.
  3. In the Frame restrictions dialog box, select the check box next to each role for which you want to enable frame access, then click Apply.

Important: When you enable access to a frame to specific roles, make sure that you include the roles for content approvers so that they have access to all of the frames that they will need to approve.

To remove access to a frame from a role, select the role in the Restrict editing to roles area, and then click the Remove role button. To remove access from multiple roles at the same time, hold down CTRL and then select the roles that you want to remove.
