Setting up data sources

A data source is what Communications Designer uses to identify which sample file to use for a particular data map. The same sample file can be used in multiple data sources, and be mapped differently for each data source with which is it associated.

Use the Communication asset management page to create and modify the data sources that can be used when fulfilling your communications.

To access the Communication asset management page, in Content Launcher, on the toolbar, click the Communication assets button .

Note: Only Communications Designer user roles that have the proper workflow control permissions allow access to the Communication asset management page. If you have questions about your access privileges, talk to your system administrator.

Creating a new data source
  1. On the Communication asset management page, on the Data sources tab, click the New button to see a list of available design packs.

  2. In the Design pack name column, select a design pack, and click Next.

  3. Specify a name and, optionally, a description for the data source that you want to create.

  4. Click Create.

    The Properties panel for that data source opens to the right of the widget.

  5. In the Properties panel, select the sample data file to be mapped by completing one of the actions described in the following table:

    Action Description

    Use a sample file that is stored in CAS

    1. Click Find sample file.

    2. Select the sample file you want to use and click OK.

    Use a sample file that is stored on your local machine or network.

    1. Click Upload.

    2. Browse to the sample file that you want to use, and click Open.

    3. Optionally, in the Sample file upload dialog, add a description.

    4. Click Add.

      The sample file that you selected is uploaded to CAS and is automatically set as the sample file for your data source.

  6. Optionally, in the File format list, select the format of your sample file from the following options:

    • JSON
    • XML

    The file format is automatically selected based on the information found in the associated sample file, but you can use the File format list to manually select a supported format. For XML files, Communications Designer supports the following features:

    • Elements and values

    • Attributes

    • Namespaces

  7. Click Create.

Opening an existing data source

  1. On the Communication asset management page, click one of the following buttons:

    • Existing—Displays all of the existing data sources

    • Recent—Displays a list of data sources that you have recently opened

    • Favorites—Displays a list of data sources that have been bookmarked

      Tip: If you work frequently with a specific data source, you can bookmark that data source so that it appears in the Favorites list. To bookmark a data source, point to the data source in the Existing or Recent tabs, and click the bookmark icon () in the Favorite column.

  2. Select the data source that you want to open, and click Open.

    The data source opens in Mapping view.

Handling orphaned mappings

Orphaned mappings are data areas in a sample file that previously had variables mapped to them, but that are no longer present in that sample file. Orphaned mappings occur when a sample file that is associated with a data source is changed after it has been previously mapped. When you open a data source that contains orphaned mappings, you receive a message that indicates which previously mapped data elements are missing.

To close the message, select one of the following options:

  • To ignore all of the orphaned mappings and remove them from the data source, click Discard.

  • To retain the variable mappings in the data source, click Keep. You might want to do this, for example, if you are using a production file that has the correct data elements, but the sample file no longer has those elements.

    Note: If you choose to keep orphaned mappings, you will continue to receive the message each time you open the data source.

Updating the data source properties

If you have already created a data source but want to change any of its properties, you can do so by selecting the data source, making the necessary adjustments in the Properties panel, and then clicking Save.

Note: If the data source has been approved, or submitted for approval, the properties cannot be changed.

Viewing the references for a data source

You can use the Used by tab in the Properties panel to see which communications use a selected data source.

To find where a data source is used:

  1. On the Communication asset management page, select the data source.

  2. In the Properties panel, click the Used by tab.

    The list shows all of the communications that use that data source.

  3. Use the indicator in the Status column to determine whether the parent communication is available for editing:

    • A green indicator () means that the parent communication is available for editing.
    • A red indicator () means that the parent communication is locked for editing by another user.
