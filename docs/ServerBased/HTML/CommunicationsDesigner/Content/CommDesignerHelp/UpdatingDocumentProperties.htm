Updating document properties

After you create a document in your communication structure, you can use the Properties panel to view and modify the document properties.

In addition to the properties that are described in this section, you can also use Structure view to view the workflow history for each document.

Updating basic properties

In the Properties panel, you can use the General tab to rename a document, and to add or modify a description for it. Keep in mind that updating the name and description will also update the name and description of the corresponding object in the common asset service (CAS).

Changing the document type

By default, when you add a document to a communication, the document type is set to Design. You can add designs to this type of document to create the communication structure. If you want use a document as a placeholder to insert external PDF content or Empower output in your communication at engine run time, then you must set the document type to Placeholder.

To set the document type to Placeholder, you must have set up the appropriate placeholder variable in Design Manager and included it in the design pack that is associated with your communication. You might need to talk to an Exstream system administrator or another user to set placeholder variables. For information about creating these variables, see Importing External Content in OpenText Exstream Design and Production Online Help.

To change the document type:

  1. In the Properties panel, on the General tab, select one of the following options in the Document type list:

    • Design
    • Placeholder

  2. If you select Placeholder, then in the Placeholder variable list, select the variable that references the file that you want to insert in your communication.

Note: When you change the document type to Placeholder for a document that contains one or more designs, then those designs are removed from that document and are no longer visible in the Structure map. These designs are, however, not deleted in CAS and can be added to a different document.

Controlling the page numbering in a document

By default, when you produce page-based output, the value of the system variables that are used for page numbering increases sequentially until the end of a customer is reached. Because the documents that you create for customers might be complex, you can choose to restart page numbers for each document within a customer, rather than allowing page numbering to continue sequentially through all of the documents.

To restart page numbering for the selected document, in the Properties panel, on the General tab, select the Restart page numbers check box.

Selecting this check box affects the values of the following system variables only:

  • SYS_PagePrintedValue
  • SYS_PagePrint
  • SYS_PageTotalPrinted

Note: These variables must be present in the design pack that is associated with your communication for you to use them in page designs. If the variables are not present, you might need to talk to an Exstream system administrator or another user to add them to the design pack. For information about creating and configuring design packs, see Designing for Exstream Web Applications in OpenText Exstream Design and Production Online Help.

Specifying PDF bookmarks for documents

When you produce PDF output, the default bookmark names for documents are the same as the document name in the Structure map. For example, if a document is named Notice in the Structure map, the corresponding bookmark in PDF output will be Notice.

To create more meaningful bookmark names for the bookmarks and make it easier for customers to navigate the file, you can use a variable to specify a custom bookmark name. You can use anything from a simple static string variable to a more complex formula variable to specify the bookmark name. For example, to make a document easier to understand in the PDF output, you might create a string variable with a more descriptive name, such as Privacy Notice.

To use a variable for the document bookmark name, in the Properties panel, on the General tab, click the ID for bookmark list, then select the variable that you want to use.

The ID for bookmark list contains variables that are present in the design pack that is associated with the communication. If a Communications Designer data source is associated with your communication, then the ID for bookmark list also contains the variables that are mapped in that data source.

Note: By default, PDF output settings in Design Manager are configured so that Exstream automatically produces bookmarks for each document in the output. However, this setting can be disabled on the output object in Design Manager. Talk to an Exstream system administrator or another user if you have questions about the output settings. For information about configuring output settings, see Creating Output in OpenText Exstream Design and Production Online Help.

Viewing advanced details for a document

In Structure view, you can select a document and see additional information about it that is stored in the common asset service (CAS).

To see these advanced details for a selected document, click the Advanced button on the General tab in the Properties panel. The Document details dialog box opens and displays the following information:

  • Name—The name of the document object in CAS
  • Version—The current version of the document. The version number is incremented every time that a new draft is created from an approved version of the document.
  • Description—The description of the document, if available
  • Created—The OTDS user that created the document, and the date and time that it was created
  • Last updated—The OTDS user that last updated the document, and the date and time that it was updated
  • Reserved by—The OTDS user that has reserved the document. This information is available only if the selected document is locked for editing.
Specifying targeting rules

Targeting rules determine whether a document is included in the output when you run the Exstreamengine. By default, rules are executed when the object to which the rule is applied is composed by the engine. However, you can delay rules for the inclusion or exclusion of the document until after all other processing is complete. For example, you might want to determine whether a customer receives a document based on the customer's account totals, which will not be known until the end of processing.

Important: Library rules and variables that are available for creating custom rules depend on the design pack and the data source (if any) that is associated with your communication. If you insert documents into your communication from other communications that use different design packs, targeting rules on the inserted documents might not function correctly.

Adding a new targeting rule

  1. On the Targeting tab, in the Document rule area, click Add.

  2. In the Rule Composer, select the type of rule that you want to apply:

    • Library rule—Use this tab to select a rule from the list of available rules. You can also use the Search box to filter the list of rules.
    • Custom rule—Use this tab to define a custom targeting rule using the variables that are available in that communication.
Updating an existing targeting rule

  1. On the Targeting tab, in the Document rule area, click Edit.

  2. In the Rule Composer, you can choose to modify or remove the existing rule:

    • To modify the rule, make any needed changes, then click Insert.
    • To remove the rule, click Remove.
Specifying the rule timing

In the Rule timing list, select the timing option that you want to use to process rules:

  • Normal—Use this option to process the document targeting rules when the document is composed. This is the default option. If you select this option, all documents are composed at customer run time and removed at the queue level after they are composed.
  • At end—Use this option to delay processing the document targeting rules until after all other customer processing is complete. When you delay the rule run time of the document, the document is composed at customer run time; however, it is not removed from the output queue until after all other processing is complete and after the At end rule is executed.
Viewing references for a document

You can use the Used by tab in the Properties panel to see which communications use a selected document.

To find where a document is used:

  1. In the Structure map, select the document.

  2. In the Properties panel, click the Used by tab.

    The list shows all of the communications that use that document.

  3. Use the indicator in the Status column to determine whether the parent communication is available for editing:

    • A green indicator () means that the parent communication is available for editing.
    • A red indicator () means that the parent communication is locked for editing by another user.
