Adding a document

A valid communication structure contains at least one document. You can choose to create a new document or to insert a reference to an existing document that is stored in the common asset service (CAS). If you have already created a document in your communication structure, you can copy the existing document to create a new document that has the same structure and layout.

Creating a new document

  1. In the Structure map, in the Documents area, click the Add button .

  2. In the Properties panel, on the New tab, specify a name and, optionally, a description for the document.

  3. Click Create.

After you create a document, you can use the General tab in the Properties panel to view and modify the properties that you specified when you created that document. Keep in mind that updating the name and description will also update the name and description of the document object in the common asset service (CAS).

Inserting an existing document

When you insert an existing document in a communication, the newly inserted document is simply a reference to the existing document. When you add a document in this manner, references to any designs in the existing document are also added to the Structure map. If you make changes to any one instance of a referenced document, all other references to that document are also updated. After you add a document to the communication structure, you can define additional document properties such as numbering, bookmark names, and rules.

Note: If you insert documents into your communication from other communications that use different design packs, targeting rules on the inserted documents might not function correctly.

To insert an existing document:

  1. In the Structure map, in the Documents area, click the Add button .

  2. In the Properties panel, click the Existing tab, and choose one of the following options to display a list of existing documents:

    • Browse—By default, the list displays the first 100 documents that are found in the current application domain. Use the Communications list and the Status list to filter the list of displayed documents. Click Reset if you want to reset the filter criteria.
    • Search—By default, the displayed list is empty. Use the Search box to enter characters from a document name or description. As you type, the Document name column is updated to display the first 100 documents that are found in the current application domain that have names or descriptions containing the entered text. To further filter the list of documents, use the Status list. Click Reset if you want to reset the search criteria.
  3. In the Document name column, select the document that you want to add, and then click Insert.
Copying an existing document

To create a copy of an existing document, in the Structure map, in the Documents area, select the document that you want to reuse, then click Duplicate.

Documents that are created in this manner are not references to existing documents, and exist as separate objects in CAS. However, the duplicate documents have the same name as the original documents. To avoid confusion, OpenText recommends that you select and rename the newly created documents in the Structure map. This will also rename the corresponding object in CAS.

Important: When you copy an existing document, references to any designs in that document are also added to the copied document in the Structure map. To create copies of the designs within a document, you must individually copy each design.

Adding a placeholder document

By default, when you add a document to a communication, the document type is set to Design. You can add designs to this type of document to create the communication structure. If you want to use a document as a placeholder to insert external PDF content or Empower output in your communication at engine run time, then you must set the document type to Placeholder.

Before you can create a placeholder document, you must have set up the appropriate placeholder variable in Design Manager and included it in the design pack that is associated with your communication. Communications Designer supports PDF placeholder variables and Empower placeholder variables. You might need to talk to an Exstream system administrator or another user to set up placeholder variables in the Design Manager application. For information about creating these variables, see Importing External Content in OpenText Exstream Design and Production Online Help.

To add a placeholder document:

  1. In the Structure map, in the Documents area, click the Add button .

  2. In the Properties panel, on the New tab, specify a name and, optionally, a description for the document.

  3. Click Create.

  4. In the Properties panel, on the General tab, in the Document type list, select Placeholder.

  5. In the Placeholder variable list, select the variable that references the file that you want to insert in your communication.

Removing a document

To remove a document from the communication structure, select the document in the Structure map, then click Remove.

Important: Removing a communication object only removes the reference to that object from the Structure map; it does not delete the corresponding object in CAS. To delete an object in CAS, you must select and delete that object in Workshop. For more information about using Workshop, see the OpenText Exstream Workshop User Guide.
