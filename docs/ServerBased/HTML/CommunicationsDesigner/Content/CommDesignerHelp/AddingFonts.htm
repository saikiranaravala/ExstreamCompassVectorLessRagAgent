Adding fonts to CAS

You can use the Design asset management page to add fonts to the common asset service (CAS) for use in a communication. These fonts supplement any fonts that are included in your design pack, and will be available for all Communications Designer users. Keep in mind that if the design pack that you are using includes a style sheet, Communications Designer honors the font information that is included in the style sheet over fonts from CAS.

To access the Design asset management page, in Content Launcher, on the toolbar, click the Design assets button .

Note: Only those Communications Designer user roles that have the proper workflow control permissions allow access to the Design asset management page. If you have questions about your access privileges, talk to your system administrator.

When you add a font file to Communications Designer, you are technically adding a font family, rather than just an individual font. Some font families are made up of two or more subfamilies. This means that if you add a font, and then later add another font file that is part of a subfamily in that font (such as a bold serif version of the font), it will not appear as a separate font in the Design assets list. Instead, it will be added to the properties of the original font family. The distinction between family and subfamily can vary based on the font and how it was created.

If you add a variable font in the form of an OpenType Font Variations (TTF) file, you will see only the default font face in the Design view font list (for example, "Regular"). Any subfamily font faces that are incorporated in the font file—such as a bold or italic version—will not be available for use.

Each user needs to have a local installation of any fonts that are used in the communication that they are editing. If a user's workstation does not have a font that is used in the design, then the default font for the application will be used instead. Because fonts are generally distributed in the same manner as software—that is, they are licensed rather than sold outright—many font licenses place restrictions on how fonts can be used. Before installing and using any fonts in your communication, be sure that you are complying with any font licensing restrictions. For example, while some fonts cannot be embedded as part of a file, others can be embedded but not edited—instead, they are used only for the purposes of printing or previewing a file. Conversely, fonts without licensing restrictions can be edited, printed, and added to end users' systems without limitations.

Note: If you have questions about font installation or licensing restrictions, talk to your system administrator. For information about the licensing restrictions of a particular font, contact the vendor that supplied the font.

Adding a font

  1. On the Design asset management page, click the Add asset button.

  2. Browse to the font you want to add, then click Open.

    Note: Only OTF and TTF formats are supported.

    A Create new font file area opens in the Properties panel.

  3. Optionally, in the Description box, enter a brief description for the font family.

  4. Click Create.

Viewing the references for a font

You can use the Used by tab in the Properties panel to see which designs use a selected font.

To find where a font is used:

  1. On the Design asset management page, select the font.

  2. In the Properties panel, click the Used by tab.

    The list shows all of the designs that use that font.

  3. Use the indicator in the Status column to determine whether the parent design is available for editing:

    • A green indicator () means that the parent design is available for editing.
    • A red indicator () means that the parent design is locked for editing by another user.
Deleting a font

If a font is not needed, and is not used in any communication objects, it can be deleted from Communications Designer. This action will also delete it from the CAS repository.

To delete a font, on the Design asset management page, on the Design assets tab, select the font you want to delete and click Remove.

Tip: If the Remove button is not available, click the Used by tab in the Properties panel to see if the selected font is used in the current version of a communication object.

If no objects are listed in the properties panel, then the font is used in a previous version of a communication object. Review the properties of the font in Workshop to determine where it is used. For more information about using Workshop, see OpenText Exstream Workshop User Guide.
