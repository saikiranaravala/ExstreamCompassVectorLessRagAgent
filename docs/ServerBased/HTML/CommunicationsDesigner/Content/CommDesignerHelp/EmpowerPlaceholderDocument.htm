Using a placeholder variable for Empower documents

After end users update the Empower document that is generated from a communication, you can use placeholder documents to incorporate the edited Empower document in another communication for processing by the Exstream engine.

Prerequisites for using placeholder variables for Empower documents

The communication that you use to fulfill an edited Empower document (the fulfillment communication) will be different from the communication that you used to create the Empower document (the design communication). However, you must use the same design pack for both the design and the fulfillment communication to keep the design resources consistent for both the creation and fulfillment of Empower output. In addition, OpenText recommends that you also configure the Design Manager applications for both of these communications to use the same design views.

Before you can create a placeholder document to fulfill an Empower document, you must also have configured the placeholder variable that references the Empower document and included the variable in the design pack.

Note: You might need to talk to an Exstream system administrator or another user to complete these prerequisites. For more information about configuring Design Manager applications, see OpenText Exstream Design and Production Online Help.

Creating an Empower placeholder document

By default, when you add a document to a communication, the document type is set to Design. To use a document as a placeholder to insert Empower output in your communication at engine run time, you must set the document type to Placeholder.

To change the document type:

  1. In the Properties panel, on the General tab, select Placeholder in the Document list.
  2. In the Placeholder variable list, select the variable that references the file that you want to insert in your communication.
