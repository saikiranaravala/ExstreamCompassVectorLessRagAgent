Adding a design

If your communication contains only non-placeholder documents, you must add at least one design to create a valid communication structure. You can choose to create a new design or to insert a reference to an existing design that is stored in the common asset service (CAS). If you have already created a design in your communication structure, you can copy the existing design to create a new design that has the same structure and layout.

In Communications Designer, you can create the following types of designs:

  • Page—Page designs are fixed-dimension layouts that you use to create communications for all supported print output queues. You can add multiple pages to multiple documents to make up a single print communication.
  • Email—Email designs are relative-dimension layouts that you use to create communications for HTML (email) output. Each email design corresponds to a single email communication.
  • Web—Web designs are relative-dimension layouts that you use to create communications for HTML5 output. Each web design corresponds to a single web-based communication.

If the Design Manager application that is associated with your communication does not contain a Communications Designer design view that corresponds to a particular design type, then the option to add that design type to your communication will not be available. In addition, if your communication contains multiple email or web designs, make sure that you specify targeting rules for your output needs. If you do not specify targeting rules, the engine processes all of the designs in a communication as they appear in the Structure map, and appends the design output sequentially for each customer into a single file. If your output queue contains a queue break, the engine produces a separate email or web design for each customer.

Creating a new design
  1. In the Structure map, in the Documents area, select the document to which you want to add your design.

  2. In the Designs area, click the design type that you want to add, then click the Add button.

  3. In the Properties panel, on the New tab, specify a name and, optionally, a description for the design.

  4. Specify the following additional properties, then click Create:

    Design type Properties
    Page
    • Paper type—In the Paper type list, select the paper type for your design. Paper types define the physical properties of the sheet or sheets of paper that correspond to the page (for example, the paper dimensions and paper weight). This list contains only the paper types that are present in the design pack that is associated with the Communications Designer application. A design pack contains all of the paper types that are present in the design database in which you created that design pack.

    • Flow page—Optionally, if you are creating a page that will be used as a flow page, select the Flow page only check box. Flow pages are included in the Structure map of a communication, but do not appear in the output unless they are used.     Email

    Width (pixels)—In the Width (pixels) box, specify the pixel width of your design. The default width is 600 pixels. The minimum width that you can enter is 150 pixels, and the maximum width is 1200 pixels.
    Web

    Width (pixels)—In the Width (pixels) box, specify the pixel width of your design. The default width is 600 pixels. The minimum width that you can enter is 150 pixels, and the maximum width is 1920 pixels.
Inserting an existing design

When you insert an existing design in a document, the newly inserted design is simply a reference to the existing design. If you make changes to any one instance of a referenced design, the original design and all other references to that design are also updated.

Note: If you insert designs from other communications that use different design packs, fonts and variables that are used in the design might not appear as expected, and targeting rules on the inserted designs might not function correctly.

To insert an existing design:

  1. In the Structure map, in the Documents area, select the document to which you want to add your design.

  2. In the Designs area, click the design type that you want to add, then click the Add button.

    The following image shows the different colors of the Add button for each design type:

    Note: If the design pack that is associated with your communication does not contain a Communications Designer design view that corresponds to a particular design type, then the option to add that design type to your communication will not be available.

  3. In the Properties panel, click the Existing tab, and choose one of the following options to display a list of existing designs:

    • Browse—By default, the list displays the first 100 designs that are found in the current application domain. Use the Communications list, the Documents list, and the Status list to filter the list of displayed designs. Click Reset if you want to reset the filter criteria.
    • Search—By default, the displayed list is empty. Use the Search box to enter a design name. As you type, the <design> name column is updated to display the first 100 designs that are found in the current application domain that have names or descriptions containing the entered text. To further filter the displayed list, use the Status list. Click Reset if you want to reset the search criteria.
  4. In the <design> name column, select the design that you want to add, and then click Insert.
Copying an existing design

To create a copy of an existing design, in the Structure map, in the Designs area, select the design that you want to reuse, then click Duplicate.

Designs that are created in this manner are not references to existing designs, and exist as separate objects in CAS. However, the duplicate designs have the same name as the original designs. To avoid confusion, OpenText recommends that you select and rename the newly created design in the Structure map. This will also rename the corresponding object in CAS.

Removing a design

To remove a design from the communication structure, select the design in the Structure map, then click Remove.

Important: Removing a communication object only removes the reference to that object from the Structure map; it does not delete the corresponding object in CAS. To delete an object in CAS, you must select and delete that object in Workshop. For more information about using Workshop, see the OpenText Exstream Workshop User Guide.
