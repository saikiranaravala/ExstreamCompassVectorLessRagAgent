Searching for documents

To find a particular document in the Empower repository, you can use the Empower Server Console to search for specific text in the fields of the document properties that are stored in Empower. Keep in mind that you cannot search the contents of the documents.

This page includes the following topics:

  • Searching for text in document fields
  • Which document fields does Empower search?
  • Best practices when searching for documents
  • Viewing large sets of results

Searching for text in document fields

  1. In the Empower Server Console, click DOCUMENTS.

  2. In the Search table columns... box, enter the text to use for the search, and click SEARCH.

    The table is updated to include only the documents that have a field that contains your text.

    For more information about the specific fields that Empower searches, see Which document fields does Empower search?

  3. By default, search results are sorted in ascending order by document ID. To sort your search results by a specific field, click the table header of the field that you want to use to sort the list of results.

    The blue column heading indicates the field by which your document list is currently sorted. When you click a blue heading, the list is toggled between ascending and descending order, as indicated by the up or down arrow in the heading.

    When you click a gray column heading, the table is sorted in ascending order by the values in that column.

  4. To further refine your results according to whether they have been added to the trash, select All, Active, or Trash from the drop-down list. The default value for this setting is Active.

    Note: When you select All or Trash from the drop-down list, you can view (but not edit) documents that have been added to the trash. To edit a document, you must first restore it from the trash. For more information about restoring a document that is no longer active, see Restoring documents from the trash .

Which document fields does Empower search?

When you search for documents in Empower using the Empower Server Console, your search results might include documents that do not appear to match your search criteria. This is because some of the fields that Empower searches  might not be displayed in the documents table, but are included in the Document Details page.

Empower searches the following document fields for the specified text:

  • Access group
  • Application ID
  • Application name
  • Document file name
  • Document ID
  • Document version
  • Document owners
  • Editor version
  • Engine version
  • External document ID
  • Package file name
  • Package version
  • Preview package file
  • User-defined tags
  • User ID

For more information about Empower document fields, see the corresponding API parameter descriptions in Get documents. Keep in mind that document field names might differ slightly between the Empower Server Console and the Empower API.

Best practices when searching for documents

When you use the search box or click a table heading to refine your Empower document list, the Empower Server Console retrieves real-time data from the repository, based on your search and sorting criteria. This means that if there are document changes in the repository after your initial search, your search results will also change.

Use the following best practices to ensure that your search results are as up-to-date as possible and that they appear as expected:

  • View search results on a single page—Because Empower retrieves real-time data from the repository each time you move from one page to another in your search results, the results can change when you move between pages. To produce search results that can be viewed on a single page, use the search parameters to narrow your search, or use the Page Size drop-down list to increase the number of results that can be displayed in the table.
  • Do not view search results when the scheduled cleanup process is running—Do not use the Empower Server Console when the scheduled cleanup process is running because the process could move documents from the Active view to the Trash view, or permanently remove documents from Empower. For more information about viewing and configuring the cleanup schedule, see Removing documents and editors.

  • Refresh your search results—It is a good idea to periodically refresh your search results to reflect document changes that have been made by other users (especially in environments where there are many documents changes). To refresh your search results without losing your search and filter selections, click REFRESH.

    Important: Do not use the browser refresh button to refresh your search results. If you use the browser refresh button to refresh the page, your search and filter selections will be lost.

  • Remove unused documents—To help reduce the amount of time that it takes Empower to search for documents, you can delete documents that are no longer used. For information about removing unused documents, see Removing documents and editors.

Viewing large sets of results

If your Empower repository includes a large number of documents, the list of documents might be split across several pages of results. Keep in mind that the Empower Server Console returns only the first 500 documents that match your search criteria. If there are more than 500 documents that match your search, Empower displays the plus symbol next to the total number of items and next to the last page number, to indicate that additional items exist. However, those additional items are not displayed as part of your initial set of search results.
