Testing documents before moving into production

Before you move an Empower document into production, it is important that you test the document to verify that it behaves as expected. This applies to new Empower documents, and to existing production documents that have been updated to use a new version of the editor (and therefore need to be validated before being returned to production). To learn more about updating an existing document to use a new version of the editor, see Changing the editor used by a document.

During the testing phase, you can use the Empower Server Console to make sure that you are able to create an Empower application using the document, and to make sure that you can upload the document, open the document in the editor, and export the document. Verifying each of these operations helps to ensure that in a production environment, integration applications will be able to upload and export the document, and end users will be able to open the document for editing.

Important: For best results, you should avoid testing documents in a production environment. Instead, use one of the following scenarios for testing:

  • Test documents using a test server—Use a test server that has the same hardware, software, and configuration settings as your production server. For help with using a test server, contact your system administrator.

  • Test documents in maintenance mode—To test documents on your production server, first place the application server into maintenance mode. This lets you test documents without affecting existing or newly-added documents. For help with placing your application server into maintenance mode, contact your system administrator.

To test a document before moving it into production, you must complete the following tasks:

  1. Adding an application

  2. Uploading a document

  3. Opening a document

  4. Exporting a document

Adding an application

Note: If you are testing an existing document that is already associated with an Empower application, then you can skip this task.

Empower applications store information about the Exstream application, the package file, the database, and the Exstream engine that were originally used to create the documents that are associated with an Empower application. Therefore, before you can test an Empower document, you must make sure that there is an Empower application to support that document.

To add an Empower application:

  1. In the Empower Server Console, click APPLICATIONS.

  2. On the Applications page, click ADD.

  3. In the ADD APPLICATION dialog box, configure the following properties as needed:

    • Editor Version—(Optional) Select the version of Empower Editor that will open documents that are associated with this application. If you do not specify an editor version, then documents that are associated with the application will use the system default version of Empower Editor.

    • Preview package file—(Optional) If Engine as a Web Service (EWS) is your preview provider, enter the name of the package file that you want to use to create previews in Empower Editor.

    • Access groups—(Optional) Enter the name(s) of an existing access group (or groups) that can edit the Empower documents that are associated with the application. If you do not specify an access group, then all authenticated users are permitted to open and edit the documents that you associate with this application.

      Note: You can apply additional access restrictions to an Empower document by specifying one or more owners for the document. For information about adding document owners, see Restricting document ownership.

  4. Click DOCUMENT, and use the file selection dialog box to choose the Empower document that you want to use to create the application.

    Keep in mind that the document is used only to extract application metadata and is not uploaded to the repository.

  5. Click ADD.

Note: When you add an application, Empower compares certain attributes of the document that you want to upload to the attributes of applications that already exist in the repository. If the attributes of the document are not found in an existing application, then the new application is added to the repository.

Uploading a document

Note: If you are testing an existing document that has been updated to use a different version of the editor and you have already uploaded the document to Empower, then you can skip this task.

  1. In the Empower Server Console, click DOCUMENTS.

  2. On the Documents page, click UPLOAD.

  3. In the DOCUMENT UPLOAD dialog box, you can optionally enter your own business-defined document identifier in the External document ID box.

    You can use this identifier to search for your document on the Documents page, however, keep in mind that it does not replace the unique document ID that Empower generates for every Empower document. The external document ID cannot be changed after the document is uploaded.

  4. If you want to associate the document with a specific Empower application, enter the ID of that application in the Application ID box.

    You are not required to provide an application ID because Empower automatically assigns the document to the Empower application that matches a specific set of document attributes (the Application Object Identifier (OI), the application path, the database ID, and the package creation date).

    Note: Keep in mind that specifying an Empower application ID that does not match the document can produce unexpected results in a production environment, and should be used for testing purposes only.

  5. In the Document owners box, add a unique identifier for each user that you want to add as a document owner (optional).

    The identifier that you enter (such as a user ID or an email address) must match the identifier that your authorization provider passes to Empower for the user.

    If you do not specify a document owner, then all authenticated users are permitted to open and edit the document.

    Note: If access to an Empower document is restricted by access groups (which are inherited from the Empower application used to create the document), then each user that you add as a document owner must be a member of an access group that is permitted to open and edit the document.

  6. In the User-defined tags box, enter one or more tags for your document (optional).

  7. Click DOCUMENT, and use the file selection dialog box to choose the Empower document that you want to use to create the application.

  8. Click UPLOAD.

Opening a document

When you open documents from the Empower Server Console, you automatically enable debugging mode in Empower Editor, which lets you use the developer tools that are included within each supported browser to double-check and validate the logic within the document itself.

To learn more about the debugging functionality that is available in Empower Editor, see "Testing Empower documents before moving into production" in Designing for Empower Editor in the Exstream Design and Production documentation.

To open a document:

  1. In the Empower Server Console, click DOCUMENTS.

  2. On the Documents page, use the sorting, filtering, or search tools to locate the document that you want to open.

    Note: You can open only active documents. If a document has been added to the trash, you must restore the document from the trash before you can open it. For information about restoring a document that is no longer active, see Restoring documents from the trash .

  3. To open the details page for the document that you want to open, click the document ID.

  4. To open the document in Empower Editor, click OPEN.

    Empower Editor launches in the browser window, and the document is displayed in the editor, ready to begin an editing session.

Exporting a document

Note: You can export only active documents. If a document has been added to the trash, you must restore the document from the trash before you can export it. For information about restoring a document that is no longer active, see Restoring documents from the trash .

  1. From the Empower Server Console, click DOCUMENTS.

  2. On the Documents page, locate the document that you want to export.

    For more information about how to search for Empower documents, see Searching for documents.

  3. To open the details page for the document that you want to export, click the document ID.

  4. To export the document, Click EXPORT.

    The document is downloaded based on how your browser is configured to handle file downloads. After the document is downloaded, it is added to the trash.
