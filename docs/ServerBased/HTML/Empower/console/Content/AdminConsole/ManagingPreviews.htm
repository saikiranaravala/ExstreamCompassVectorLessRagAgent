Managing document previews

When you install and set up Empower for the first time, you can choose whether to configure Exstream Engine as a Web Service (EWS) to provide real-time document previews for your Empower Editor end users. Following the initial Empower setup, you can use the Empower Server Console to disable or enable the preview service, to adjust the timeout setting for previews, or to change the package file information that an application uses to generate previews.

To manage the document preview settings, complete the following tasks as needed:

Disable previews

The preview service is enabled by default if you configure a preview provider when you set up Empower. If you do not want Empower Editor end users to be able to generate previews of the documents they are editing, then you can disable the preview service. When the preview service is disabled, end users will not see the preview option.

To disable the preview service:

  1. In the Empower Server Console, click CONFIGURATION.

  2. On the Configuration Settings page, clear the Preview check box.

  3. Click UPDATE.

Enable previews

  1. In the Empower Server Console, click CONFIGURATION.

  2. On the Configuration Settings page, select the Preview check box.

  3. Click UPDATE.

Note: Keep in mind that you can enable previews only if a preview provider was configured during Empower setup.

Adjust the timeout for document previews

You can adjust how long Empower will wait for a document preview to be generated. The timeout interval should allow enough time for a document to be sent to EWS, for output to be produced by the Exstream engine, and for the output to be returned to the browser.

To adjust the timeout for previews:

  1. In the Empower Server Console, click CONFIGURATION.

  2. On the Configuration Settings page, make sure that the Preview check box is selected.

  3. In the Preview timeout box, specify the amount of time (in seconds) to wait for a preview to be returned.

  4. Click UPDATE.

Note: If Empower is running on an IBM WebSphere application server, changing the preview timeout setting in the Empower Server Console will not affect the amount of time that Empower waits for previews to generate, because the timeout setting on the application server overrides the setting that is specified in Empower. To change the timeout setting in WebSphere, consult your WebSphere administrator.

Change the preview package file information

When an end user clicks Preview in Empower Editor, a real-time preview of the active document is generated using the preview package file information stored in the Empower application that is associated with the document. In some cases, you might find it necessary to change the preview package file for an Empower application; for example, if you upgrade Exstream Design and Production to use a later version of the Exstream engine.

To change the preview package file for an existing Empower application:

  1. In the Empower Server Console, click APPLICATIONS.

  2. In the list of applications, click the ID of the application that you want to change.

  3. On the Application Details page, in the Preview package file box, enter the name of the preview package file that you want to use.

    Note: If the preview service works for some documents but not for others, the preview package file for the document might not be located in the directory that EWS expects. For information about the location of the EWS server, contact your system administrator.

  4. Click UPDATE.

    Empower documents associated with this application that are imported following the change will use the new preview package file information to generate document previews.
