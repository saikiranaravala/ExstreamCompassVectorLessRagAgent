Adding an access group

You can use access groups alone or in conjunction with interactive roles that you create in Design Manager to control access to Empower documents. Access groups determine who can open an Empower document, while interactive roles determine who can edit the individual areas of an Empower document. In order to apply the interactive roles that you create in Design Manager to the Empower documents in your repository, you must associate each interactive role with an access group in Empower.

For information about how to set up interactive roles in Design Manager, see "Setting up role-based editing in Empower documents" in Designing for Empower Editor in the Exstream Design and Production documentation.

When you add an access group to Empower, keep in mind the following considerations:

  • Access groups that you create and assign to Empower applications must be mapped to the groups already set up on your LDAP server.
  • When possible, you should add access groups to Empower before you add the Empower applications with which you intend to associate them.
  • If you assign a new access group to an existing application, documents that have already been created with that application do not inherit the new permissions.
  • You can associate an access group with more than one Empower application.

To add an access group to Empower:

  1. In the Empower Server Console, click ACCESS GROUPS.

  2. On the Access Groups page, click ADD.

  3. In ADD ACCESS GROUP dialog box, in the Access group box, enter a name for the access group.

    Access group names are limited to 250 characters, and can contain only letters, numbers, periods (.), hyphens (-), underscores (_), and spaces.

  4. If this access group will be used to support role-based editing of Empower documents, in the Interactive roles box, enter one or more of the interactive roles that you created in Design Manager.

  5. Click ADD.

    The dialog box closes and the new access group is added to the table on the Access Groups page.
