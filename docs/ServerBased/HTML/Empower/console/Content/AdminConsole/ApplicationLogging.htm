Application logging

The Empower application log is written to the empower.log file, and is stored in either the empower.home location, or in a directory named empower that is created relative to where the Empower application is running (if empower.home has not been defined as an environment variable or set as a system property).

By default, the application log file rolls over every 24 hours, or when the size of the file reaches 25 MB, whichever comes first. Files that have been rolled over are placed into an archived sub-directory next to the current log, and are retained for 30 days.

Tip: Some Empower application events are also captured in the application server log. The empower.log file is designed to provide additional context about Empower application events.

Adjusting the application log settings

  • If you want to change the default application log settings after you have installed Empower, you can edit the logback.xml configuration file, and then and redeploy the Empower WAR file. The logback.xml configuration file is located in the WEB-INF\classes directory of the Empower WAR file, which is contained in the Empower_Server ZIP file.

  • If you want to override the default application log settings at run time, enter the following expression at the command prompt to point Java to a custom logback.xml file that you create:

    -Dlogback.configurationFile=/path/to/logback.xml
