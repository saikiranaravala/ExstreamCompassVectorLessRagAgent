Searching for applications

Empower applications store information about the Exstream application, the package file, the database, and the engine that were used to create the documents that are associated with an Empower application. Therefore, before you can test an Empower document, you must add an Empower application to Empower to support that document. During testing, you might need to find a particular application so that you can make changes to the editor used by a document, or you might want to delete an application from Empower after testing.

To find a particular application in the Empower repository, you can use the Empower Server Console to search for specific text in the fields of the application properties that are stored in Empower. Keep in mind that you cannot search the contents of the applications.

This page includes the following topics:

  • Searching for text in application fields
  • Which application fields does Empower search?
  • Best practices when searching for applications
  • Viewing large sets of results

Searching for text in application fields

  1. From the Empower Server Console, click APPLICATIONS.

  2. In the Search table columns... box, enter the text to use for the search, and click SEARCH.

    The table is updated to include only the applications that have a field that contains your text.

    For more information about the specific fields that Empower searches, see Which application fields does Empower search??

    For more information about Empower advanced search techniques, see Using search operators.

  3. By default, search results are sorted in ascending order by application name. To sort your search results by a specific field, click the table header of the field that you want to use to sort the list of results.

    The blue column heading indicates the field by which your application list is currently sorted. When you click a blue heading, the list is toggled between ascending and descending order, as indicated by the up or down arrow in the heading.

    When you click a gray column heading, the table is sorted in ascending order by the values in that column.

  4. To further refine your results so that you see only Empower applications that use a specific version of the editor, select the version of the editor that you want to use to filter the results from the drop-down list. If there is more than one version of the editor installed, the default value for this setting is Show all.

Which application fields does Empower search?

When you search for applications in Empower using the Empower Server Console, your search results might include applications that do not appear to match your search criteria. This is because some of the fields that Empower searches  might not be displayed in the applications table, but are included in the Application Details page.

Empower searches the following application fields for the specified text:

  • Access group
  • Application ID
  • Application name
  • Application OI
  • Application path
  • Database ID
  • Editor version
  • Engine version
  • Package file name
  • Package version
  • Preview package file

For more information about Empower application fields, see the corresponding API parameter descriptions in the Get applications API call. Keep in mind that application field names might differ slightly between the Empower Server Console and the Empower API.

For information about advanced search techniques, see Using search operators.

Best practices when searching for applications

When you use the search box or click a table heading to refine your Empower application list, the Empower Server Console retrieves real-time data from the repository, based on your search and sorting criteria. This means that if there are application changes in the repository after your initial search, your search results will also change.

Use the following best practices to ensure that your search results are as up-to-date as possible and that they appear as expected:

  • View search results on a single page—Because Empower retrieves real-time data from the repository each time you move from one page to another in your search results, the results can change when you move between pages. To produce search results that can be viewed on a single page, use the search parameters to narrow your search, or use the Page Size drop-down list to increase the number of results that can be displayed in the table.

  • Refresh your search results—It is a good idea to periodically refresh your search results to reflect application changes that have been made by other users (especially in environments where there are many applications changes). To refresh your search results without losing your search and filter selections, click REFRESH.

    Important: Do not use the browser refresh button to refresh your search results. If you use the browser refresh button to refresh the page, your search and filter selections will be lost.

  • Remove unneeded applications—To help reduce the amount of time that it takes Empower to search for applications, you can delete applications that are no longer needed. For information about removing unneeded applications, see Deleting applications.

Viewing large sets of results

If your Empower repository includes a large number of applications, the list of applications might be split across several pages of results. Keep in mind that the Empower Server Console returns only the first 500 applications that match your search criteria. If there are more than 500 applications that match your search, Empower displays the plus symbol next to the total number of items and next to the last page number, to indicate that additional items exist. However, those additional items are not displayed as part of your initial set of search results.
