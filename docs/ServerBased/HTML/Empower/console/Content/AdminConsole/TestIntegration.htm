Testing your integration applications

Before moving into production, it is a good idea to test the integration applications that you develop for Empower, to make sure that the workflow and the Empower API calls you have included function as expected.

Note: If you are using a reverse proxy server in front of Empower and you are terminating HTTPS at the reverse proxy server, you might need to pass the X-Forwarded-Host, X-Forwarded-Proto, and X-Forwarded-Port headers with the appropriate values when using the Empower Open API method. For more information about setting X-Fowarded headers, see the documentation that came with your reverse proxy server.

To test an integration application:

  1. Log in to the application server that hosts Empower as a user who has at least the editor and integrator permissions.

  2. Capture your session security token information using the Get security token API call. You will use this token in the header of POST and DELETE API calls.

    Note: Manual security token management is only necessary for making API calls. If you use the Empower Server Console or Empower Editor, Empower manages the security tokens for you.

  3. Create an Empower application.

  4. Import the Empower document that you used to create the application in the previous step.

  5. Open the Empower document in a web browser.

  6. With the document open in Empower Editor, perform the following tasks, in order:

    1. Edit text or formatting in the document.
    2. Upload a resource to the document.
    3. Save the document.
    4. Revert to the previously saved version of the document.
    5. Generate and view a PDF preview of the document (if preview is enabled).
  7. Leave the browser window open.
  8. Export the Empower document to your Exstream document repository.

  9. Refresh the browser window that contains the Empower document.

    When the browser window refreshes, you receive the following message that indicates that the document has been successfully exported and is no longer available:

    The following document could not be found: {document ID}

  10. Use the Exstream production engine to fulfill the Empower document, and then view the output.

    For information about using the Exstream engine to fulfill documents, see "Fulfilling Empower documents" in Designing for Exstream Empower Editor in the Exstream Design and Production documentation.
