Information for system integrators

The following table lists some common tasks that a system integrator might perform while integrating Empower into a document editing and production workflow, and the information that will be needed to perform those tasks:

This task Requires this information

Configuring Empower system settings, managing versions of the editor, or managing documents

The URL of the Empower Server Console

For example:

http://server02.mycompany.com:8080/mpw/console/index.html

Signing in to Empower

A user name and password with administrator, editor, and integrator permissions

Note: If your network uses domain accounts to authenticate users to network resources, you can let users access Empower with network credentials, rather than set up individual accounts on the application server.

Allowing the integration application to access the application server

 A user name and password with integrator permissions

Making API calls from the integration application

The URL (root + context) to include in API calls

For example:

http://server02.mycompany.com:8080/mpw

Enabling the integration application to connect to a preview provider to install and reference the package files that will be used to generate PDF preview documents

The URL of the instance of EWS that Empower Editor will use to support PDF previews. For example:

http://server01.mycompany.com:8091/EngineService

Note: If you did not set up a preview provider when you installed Empower, then you do not need to collect this information.
