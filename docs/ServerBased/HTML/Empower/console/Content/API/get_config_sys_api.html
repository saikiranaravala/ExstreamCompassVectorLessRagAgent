Get a configuration

The GET API call for configurations returns detailed information about a specified system configuration attribute.

Who can access this call

Access to the Empower API is defined in the customSecurityContext.xml file during installation, and is controlled by your authorization provider. By default, Empower supports the administrator, editor, and integrator authorities.

Authenticated users with any of the following authorities can use this API call:

  • administrator
  • editor

Request

Request method Syntax
GET /<context>/resource/config/{configParam}

The <context> variable is configured during Empower installation.

Request header

Header Value Use Description
Accept application/xml Optional Requests that Empower return an XML response instead of the default JSON response

Request parameters

Use one of the following parameters for the {configParam} variable:

Parameter Data type Description
defaultEditor String
 The version of the editor that is assigned to all new Empower applications
defaultLocale String The alternative language that appears in Empower Editor and in the Empower documentation if the end user's web browser is configured to use a language that is not supported by Empower
idleDocCleanup String

Indicates whether the scheduled cleanup process is set to mark documents for deletion when they have not been edited within the retention period that is specified in the idleDocRetention parameter

This parameter can have a value of Enabled or Disabled.
idleDocRetention Integer

The number of days that documents are retained after they are imported or edited

When the scheduled cleanup process runs, if the idleDocCleanup parameter is enabled, documents that have not been edited within the specified retention period are marked for deletion.
preview String

Indicates the state of the document preview service

This parameter can have a value of Enabled or Disabled.

The default value for this parameter is Enabled.
previewTimeout

Integer

 The length of time (in milliseconds) that Empower Editor will wait for a response from the PDF preview
retentionTime

Integer

The length of time (in milliseconds) that objects such as documents or editors are retained after they are marked for deletion

When the retention period expires, eligible objects are deleted during the next scheduled cleanup.
servicesVersion String The version of Empower Services Layer that you are running
supportEmail String The email address to which Empower Editor sends messages
triggerDays String The days of the week that are used to define the triggerExpression parameter
triggerExpression String

The cron expression that defines when the scheduled cleanup process runs

You can build your own cron expression, or use the triggerDays, triggerHours, and triggerMinutes parameters to set the value for the triggerExpression parameter.
triggerExpressionType String

The cron expression that defines whether to run the cleanup process on a daily basis or an hourly basis

This parameter can have a value of Daily or Hourly.

The default value for this parameter is Daily.

You can build your own cron expression, or use the triggerExpressionType parameter with the triggerExpression, triggerDays, triggerHours, and triggerMinutes parameters.
triggerHours String The hour of the day that is used to define the triggerExpression parameter
triggerMinutes String The minute of the hour that is used to define the triggerExpression parameter

Example request

Element Example

Request header

 GET http://sample-server.example.com:8080/mpw/resource/config/supportEmail
Request body This API call does not support a request body.

Response

Empower supports two formats for API response payloads: JSON and XML.

Example JSON response

Media type: application/json

{
    "header": {
        "status": {
            "code": "SERV100I"
        }
    },
    "body": {
        "config": {
            "supportEmail": "support@example.com"
        }
    }
}

Example XML response

Media type: application/xml

        <?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<ResponseContainer>
    <header>
        <status code="SERV100I"/>
    </header>
    <body>
        <config supportEmail="support@example.com"/>
    </body>
</ResponseContainer>
