JavaScript API documentation: The Editor API

The Empower JavaScript API, which currently consists of the Editor API, allows integrators to configure Empower Editor so that it can communicate with other client-side web applications that are hosting Empower Editor sessions.

The Editor API is for automating Empower Editor when it is integrated with other client-side web applications within an iframe. For example, you can use the Editor API so that a parent HTML page can inspect an Empower document that is open in an iframe to determine whether the document needs to be saved before ending the Empower Editor session.

Note: If you plan to display Empower Editor in an iframe, then you might want to consider configuring additional Spring Security options. Depending on which browsers you expect to use with Empower Editor, it might be necessary for you to configure the X-Frame-Options header, the Content-Security-Policy header, or both in the <headers> element in the customSecurityContext.xml file that comes with Empower. For information about how to configure these options, see the security response headers section of the current Spring Security Reference documentation on the Spring Security site.

Who Will Use this API

Empower integrators will use the Editor API when integrating Empower with other client-side web applications.

Included Files

The following files are located in the EditorAPI directory of the Empower_Server ZIP file:

File Description
EditorAPI.js This file contains the code that allows the parent HTML page to communicate with Empower Editor. It uses the EditorAPI JavaScript object to expose the API.
EditorAPI_Sample.html This is a sample HTML file that lets you test the Editor API before putting it into production.

Implementing the Editor API

To implement the Editor API, complete the following tasks as needed:

  1. Testing the Editor API (optional)
  2. Making the Editor API available on your production server
  3. Making an API call

Testing the Editor API (optional)

You can use the EditorAPI_Sample.html file to test the Editor API before putting it into production.

To test the Editor API:

  1. Copy the EditorAPI.js file to your test web server.

  2. Modify the EditorAPI_Sample.html file as follows:

    1. Change the <script>src attribute to point to the EditorAPI.js file on your test server.
    2. If the EditorAPI_Sample.html file is not hosted on the same server that hosts the Empower Editor session, you must update the window.location.origin argument to reflect the location of the Empower server (for example, http://myEmpowerServer:9090).

    3. Change the <iframe>src attribute to point to an Empower document (for example, http://myEmpowerServer:9090/hes/resource/docedit/editor/1.0.6.0/locale/en_US/empower-develop.html?docId=774cf2cd-c71f-48cc-a896-31f704618b31#/document/1/page/1).

Making the Editor API available on your production server

To make the Editor API available in production, simply copy the EditorAPI.js file to your production web server.

Making an API call

  1. Obtain an empowerInstance object:

    var empowerInstance = EditorAPI.getInstance(document.getElementById("myIframeId").contentWindow, document.location.origin);

  2. Call the save method, which expects a callback method to be passed:

    function saveCallback(response) {
      if (response.success === true)
        alert("Success");
      else
        alert("Failed : " + result.message);
    }
    empowerInstance.document.save(saveCallback);

Methods

EditorAPI.getInstance()

Syntax

function getInstance(targetWindow, targetOrigin)

Return value

(empowerInstance)

The getInstance method returns an empowerInstance object, which is required in order to use the API functionality.

Arguments

Argument Description

targetWindow (window)

 The window object obtained from the iframe
targetOrigin (string) The origin of the Empower Editor server

Example

var empowerInstance = EditorAPI.getInstance(document.getElementById("myIframeId").contentWindow, window.location.origin);

empowerInstance.document.hasChanged()

Syntax

function hasChanged(fnCallback,includeUncommitedChangesInActiveVarArea)

Return value

No return value.

Parameters

Parameter Data type Use Description

fnCallback

 String Required

This parameter is a callback function provided by the integrator. It will be called when the API method has finished executing. The argument must accept a single argument of type object. When fnCallback is called by the API, a callbackobject argument will be passed to it. The callbackobject argument has a success member, as well as optional isDirty and message members.

  • isDirty: This member returns a boolean value. If the value is true, the document needs to be saved; a value of false indicates that the document has not been changed since it was last saved.

  • success: This member returns a boolean value that indicates the success or failure of the hasChanged method.

  • message: This member is a string value that returns an optional failure message.
includeUncommitedChangesInActiveVarArea String Optional

By default, Empower Editor does not evaluate a variable edit area for changes until after an end user has navigated away from the area. Therefore, if an Exstream Empower Editor document is open in an iframe, and the end user is still working in a variable edit area when he or she clicks a button in the parent iframe to save the document, any changes to the variable edit area will be lost if the end user does not navigate away from the area and re-save the document before closing it.

This parameter instructs the Empower Editor API to look for changes in the active variable edit area when determining the result of the hasChanged method. The parameter can have a value of none, true, or false. A value of true means that Exstream Empower Editor will take into account any changes to the active variable edit area—as well as any unsaved changes elsewhere in the document—in its results. A value of none or false means that the Empower Editor API will disregard any changes in the active variable edit area and only take into account unsaved changes elsewhere in the document.

Important: Do not use this parameter if you have implemented autosave in your Exstream Empower Editor environment.

Example 

var empowerInstance = EditorAPI.getInstance(document.getElementById("myIframeId").
contentWindow, window.location.origin);
 
	function hasChangedCallback(response) {
		if (response.success === true) {
			var message = "Success : the document has ";
			if (response.isDirty)
				message += "changed.";
			else
				message += "not changed.";
			alert(message);
		}
		else
			alert("Failed : " + result.message);
	}
	empowerInstance.document.hasChanged(hasChangedCallback, true);
}

Syntax

Return value

No return value.

Parameters

Parameter Data type Use Description

fnCallback

 String Required

This parameter is a function provided by the integrator, and it is called when the API method has finished executing. The argument must accept a single argument of type object. When fnCallback is called by the API, a callbackobject argument will be passed to it. The callbackobject argument has a success member, as well as an optional message member.

  • success: This member returns a boolean value that indicates the success or failure of the save method.
  • message: This member is a string value that returns an optional failure message.
includeUncommitedChangesInActiveVarArea String Optional

By default, Empower Editor does not evaluate a variable edit area for changes until after an end user has navigated away from the area. Therefore, if an Exstream Empower Editor document is open in an iframe, and an end user is still working in a variable edit area when he or she clicks a button in the parent iframe to save the document, any changes to the variable edit area will be lost if the end user does not navigate away from the area and re-save the document before closing it.

empowerInstance.getFirstRequiredEditArea()

Syntax

function getFirstRequiredEditArea(fnCallback)

Return value

No return value.

Parameters

Parameter Data type Use Description

fnCallback

 Function Required

This parameter is a callback function provided by the integrator. It will be called when the Empower API method has finished executing. The callback must accept a single argument of type object. When fnCallback is called by the Empower API, a callbackobject argument will be passed to it.

On success, the callbackobject argument has the following form:

{"success":"true","routeIndex":{"doc":2,"page":1},"pageName":"reqNav_miscCases"}

  • success: This member returns a boolean value that indicates the success or failure of the getFirstRequiredEditArea method.
  • routeIndex: This object points to the first page that contains an untouched required edit area
    • doc: The doc member contains the ordinal of the design document of the first page that contains an untouched required edit area
    • page: The page member contains the ordinal of the design document of the first page that contains an untouched required edit area
  • pageName: This member contains the name of the first page that contains an untouched required edit area

On failure, the callbackobject argument has the following form:

{"success":"false","reason": "Reason for failure"}

  • success: This member returns a boolean value that indicates the success or failure of the getFirstRequiredEditArea method.
  • reason: This member contains the reason for the failure of the call to getFirstRequiredEditArea.

Example

function onClick_btnReq() {
	var EmpowerServerOrigin = window.location.origin;
	var idOfFrameToHostEmpower = "myIframeId";
	var childIFrame = window.document.getElementById(idOfFrameToHostEmpower);
	empowerInstance = EditorAPI.getInstance(childIFrame.contentWindow, EmpowerServerOrigin);

	empowerInstance.document.getFirstRequiredEditArea(function(returnValue) {
		window.BNTempPageParams = returnValue;
		logMessage("empowerInstance.document.getFirstRequiredEditArea() = " + JSON.stringify(returnValue) + "\n");
	});
}
empowerInstance.navigateToFirstRequiredEditableArea()

Syntax

function navigateToFirstRequiredEditableArea(fnCallback, routeIndex)

Return value

No return value.

Parameters

Parameter Data type Use Description

fnCallback

Function

Required

This parameter is a callback function provided by the integrator. It will be called when the Empower API method has finished executing. The callback must accept a single argument of type object. When fnCallback is called by the Empower API, a callbackobject argument will be passed to it.

On success, the callbackobject argument has the following form:

{"success":"true"}

  • success: This member returns a boolean value that indicates the success or failure of the getFirstRequiredEditArea method.

On failure, the callbackobject argument has the following form:

{"success":"false","reason": "Reason for failure"}

  • success: This member returns a boolean value that indicates the success or failure of the getFirstRequiredEditArea method.
  • reason: This member contains the reason for the failure of the call to getFirstRequiredEditArea.
routeIndex Object Required

This parameter is an object that contains a doc member and a page member. This parameter defines the location in the Empower application to which the Empower API will navigate before it finds the first editable area on the page.

e.g. {"doc":2,"page":1}

  • doc: The doc member contains the ordinal of the design document
  • page: The page member contains the ordinal of the composed page

Note: One of the possible sources of the doc and page members is from the result of the getFirstRequiredEditArea call.

Example 

function onClick_btnNav() {
	var EmpowerServerOrigin = window.location.origin;
	var idOfFrameToHostEmpower = "myIframeId";
	var childIFrame = window.document.getElementById(idOfFrameToHostEmpower);
	empowerInstance = EditorAPI.getInstance(childIFrame.contentWindow, EmpowerServerOrigin);

	empowerInstance.document.navigateToFirstRequiredEditableArea(function(returnValue) {
		console.log("navigateToFirstRequiredEditableArea() callback received " + JSON.stringify(returnValue) + "\n");
		}, {"doc":2,"page":1});
}
