Get documents

The GET API call for documents returns a list of the documents that match your request parameters.

Who can access this call

Access to the Empower API is defined in the customSecurityContext.xml file during installation, and is controlled by your authorization provider. By default, Empower supports the administrator, editor, and integrator authorities.

Authenticated users with any of the following authorities can use this API call:

  • administrator
  • integrator

Request

Request method Syntax
GET /<context>/resource/documents?{parameters}

The <context> variable is configured during Empower installation.

The {parameters} search variable is optional. If you do not include any parameters, the search returns a list of all documents.

Request header

Header Value Use Description
Accept application/xml Optional Requests that Empower return an XML response instead of the default JSON response

Request parameters

You can use the sorting, filtering, searching, and page size parameters to limit the number of documents that are returned in a response.

Sorting

Parameter Data type

Use

 Description
sortField String Optional

Returns a list of documents that are sorted by the value of the specified document response parameter

The maximum string length is 50 characters.

This parameter supports the ^[A-Za-z]* characters.
sortOrder String Optional

Sorts the list of documents in ascending or descending order

This parameter can have a value of ASC or DESC.

Note: If you use the sortField parameter without the sortOrder parameter, the list of documents is sorted in ascending order.

Filtering

To filter the results when you search Empower documents, specify values for one or more of the parameters in the following table.

You can use these tips to help you refine your search:

  • To exclude a string from a filter request, precede the string with a dash (-). When you precede a string with a dash, Empower excludes documents that match the string that follows the dash.

  • To look for a string that begins with a dash, add the backslash (\) escape character just before the dash in your search string. For example, docTag=\-documentname.

  • To narrow the filtering results, you can string multiple parameters together with an ampersand (&). For example, appID=task&accessgroupName=Name.
Parameter Data type

Use

 Description

accessgroupName

String

Returns a list of the documents that use the specified access group

The maximum string length is 250 characters.

This parameter supports the ^[\\p{L}\\p{M}\\p{N}_\\-\\.\\*\\s"\\\\]*$ characters.

Note: For more information about how to search for documents by access group, see Using search operators.

Returns documents with a specific business-defined document ID

The maximum string length is 250 characters.

Returns a list of the documents that are either marked for deletion or active

This parameter can have a value of true or false.

Returns a single document that matches the unique document ID

The maximum string length is 36 characters.

This parameter supports the \p{XDigit}{8}-\p{XDigit}{4}-\p{XDigit}{4}-\p{XDigit}{4}-\p{XDigit}{12} characters.

Returns a list of the documents that contain the specified metadata

The maximum string length is 250 characters.

Note: For more information about how to search for document tags, see Using search operators.

The maximum string length is 250 characters.

The maximum string length is 250 characters.

userId

String

 Optional

Returns a list of the documents that were imported by a specified user

The maximum string length is 250 characters.

This parameter supports the ^[\p{L}\p{M}\p{N}_\-\.\s]*$ characters.

Searching

You can search across a set of supported string fields.

Note: Empower does not support using the dash (-) operator to exclude a search term on the q parameter.

Parameter Data type

Use

 Description
q String Optional

Returns a list of the documents that match the search value that is specified for any of the following indexed string parameter fields:

  • appId
  • applicationName
  • busDocId
  • docId
  • docTag
  • documentVersion
  • editorVersion
  • engineVersion
  • fileName
  • ownerId
  • packageFileName
  • packageVersion
  • previewPubFile
  • accessgroupName
  • userId

The maximum string length is 500 characters.

This parameter supports the ^[\\p{L}\\p{M}\\p{N}_\\-\\.\\*\\s"\\\\@]*$ characters.

Note: Empower does not support searching across document date parameter fields.

Using search and filter operators

You can narrow your search and filtering results by structuring your request and using operators. Keep in mind that if you specify more than one term, Empower combines the terms and returns documents that match all of the terms.

This search Returns

Single word search

q=search

Documents that contain the specified word in any supported field

Searches are not case-sensitive.

For example, q=search matches documents with the following fields:

  • docTag=search
  • previewPubFile=Search

Exact word search

docTag=search

Documents that contain the exact word in any supported field on the docTag, ownerId, and accessgroupName parameters

Searches are not case-sensitive.

For example, docTag=" typical" -" typical_*" matches documents with the following field:

  • docTag=typical

docTag=" typical" -" typical_*" does not match documents with the following fields:

  • docTag=atypical
  • docTag=typically

 

Wildcard search

q=sea*

Documents that contain the specified partial word with or without additional text in the wildcard (* or _) location

Empower supports the * and _ wildcard characters.

  • * matches zero or more characters
  • _ matches one character

For example, q=sea* matches documents with the following fields:

  • docTag=search
  • userId=Sean
  • previewPubFile=Seat21

For example, q=sea_, matches documents with the following fields:

  • userId=Sean

q=sea_, does not match documents with the following fields:

  • previewPubFile=Seat21
  • docTag=search

Multiple word search

q=search here

Documents that contain a single field that includes all specified words in any order

For example, q=search here matches documents with the following fields:

  • docTag=search here
  • docTag=search one Here

Exact phrase search

q="search here"

Documents that contain a single field that includes the specified phrase

For example, q=search here matches documents with the following fields:

  • docTag=search here
  • busDocId=search here Sean

Keep in mind that placing quotation marks around a single word will not return an exact word match.

Dashes, underscores, and periods

q=search-me

q=search_for

q=search.for

Documents that contain the specified hyphenated word, or words that are joined by an underscore or a period

For example, q=search-me matches documents with the following fields:

  • docTag=search-me now
  • busDocId=Don't Search-me

Empower supports using a dash (-) to exclude words for filtering only.

For example, q=search -me is treated as a multiple word search, and would only return documents with a single field that included both "search" and "-me".

For more information about how to filter your results, see Filtering

Or

q=search or me

Documents that contain one of several words

For example, q=search or me matches documents with the following fields:

  • docTag=search
  • userId=me

 

Controlling how search results are displayed

When your repository contains a large number of documents, it is important to consider how your integration application interface will display the results when users search for documents. This API call supports several parameters that you can use to control how your integration application displays pages of results, and to help reduce the amount of time that it takes to return results.

Tip: For best results, design your integration application to use searching, filtering, and pagination to return a single page of results.

Parameter Data type

Use

 Description
limit Long Optional

Sets a limit on the number of documents that Empower can return in the response (the page size)

Note: You must use the limit and offset parameters together.
offset Long Optional

Tells Empower how many results to skip before returning the first search result

Note: You must use the limit and offset parameters together.
maxTotal Long Optional

Tells Empower to count to the last document that is returned by the search, or to the maxTotal number of documents (whichever is larger)

You can use this parameter to decrease the response time for this API call, by limiting the number of documents that are counted during the repository search.

Note: If you use the maxTotal parameter without specifying the limit and offset parameters, Empower sets the default pagination values to limit=maxTotal&offset=0.

Example requests

The following examples show how to create various types of requests using some of the parameters that are supported by this API call. Keep in mind that this API call does not support using a request body.

Example 1:

Request a list of all documents with no pagination.

Element Example

Request header

 GET http://sample-server.example.com:8080/mpw/resource/documents

Example 2:

Request a single document with a specified document ID (if the document exists).

Element Example

Request header

 GET http://sample-server.example.com:8080/mpw/resource/documents?docId=4bbc6032-0826-4d53-a408-2582610c9192

Example 3:

Request a list of all documents that have been uploaded by a specified user, sorted in ascending order by the date that the documents were imported, and displaying only the first two documents.

Element Example

Request header

GET http://sample-server.example.com:8080/mpw/resource/documents?userId=admin&sortField=importDate&limit=2&offset=0

Example 4:

Request a list of all documents, with pagination that returns documents 51-100 and that counts all of the documents in the repository.

Element Example

Request header

GET http://sample-server.example.com:8080/mpw/resource/documents?limit=50&offset=50

Notice that this example uses only the limit and offset parameters. Depending on the size of the document repository, not including the maxTotal parameter with the limit and offset parameters can result in long response times, because Empower counts the total number of documents in the repository. However, using only the limit and offset parameter lets you provide your users with the flexibility to go to any page in the results, up to and including the last page.

When you use this request example, the integration application interface displays the results on page 2, and lets users move forwards or backwards in the search results, or go to a specific page in the results:

Example 5:

Request a list of all documents, with pagination that returns documents 101-150 and that counts to the last document that is returned.

Element Example

Request header

GET http://sample-server.example.com:8080/mpw/resource/documents?limit=50&offset=100&maxTotal=0

In this example, specifying a maxTotal value of 0 lets you minimize the response time for large document repositories, because Empower counts only to the last document that is returned. When you view the response for this request, you can look at the exceedsTotal response parameter to determine whether additional matching documents exist.

When you use this request example, the integration application interface displays the results on page 3, and lets users move forward in the results one page at a time, or select any of the previous pages in the results:

Example 6:

Request a list of all documents, with pagination that returns documents 101-150, and that counts only 450 documents.

Element Example

Request header

GET http://sample-server.example.com:8080/mpw/resource/documents?limit=50&offset=100&maxTotal=450

In this example, specifying a maxTotal value of 450 lets you decrease the search response time in large document repositories, because Empower counts only the first 450 documents that match the search parameters. When you view the response for this request, you can look at the exceedsTotal response parameter to determine whether additional matching documents exist.

When you use this request example, the integration application interface displays the results on page 3, and lets users move forwards or backwards in the search results, or go to a specific page in the results up to page 10:

Response parameters

If a parameter object does not contain a value, a JSON response returns a null value for the object, but an XML response does not include the object.

Parameter Data type Description
accessgroupNames String

The access group (or groups) that control access to the document

Access groups are assigned by the Empower application when the document is imported into Empower.

Note: The accessgroupNames parameter can be used by itself, or in conjunction with document owners. To see whether access to the document is restricted to certain document owners, look at the response that was returned with the ownerIds parameter.

The date that the document was created

deleted Boolean The flag that indicates a document has been exported or deleted, and is marked for permanent removal by the scheduled cleanup process

docTags String The metadata that was added to the document

editorVersion String

The version of Empower Editor that can be used to open the document
