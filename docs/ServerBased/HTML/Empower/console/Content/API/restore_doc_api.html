Restore a document

The restore POST API call for documents restores a specified Empower document that has been marked for deletion by setting the value of the deleted parameter to false, which removes the flag that marks the document for removal by the scheduled cleanup process.

Who can access this call

Access to the Empower API is defined in the customSecurityContext.xml file during installation, and is controlled by your authorization provider. By default, Empower supports the administrator, editor, and integrator authorities.

Authenticated users with any of the following authorities can use this API call:

  • administrator
  • integrator

Request

Request method Content type Syntax
POST application/x-www-form-urlencoded /<context>/resource/documents/{docId}/restore

The <context> variable is configured during Empower installation.

The {docId} is the ID of the existing document.

The maximum string length of the {docId} parameter is 36 characters.

The {docId} parameter supports the \p{XDigit}{8}-\p{XDigit}{4}-\p{XDigit}{4}-\p{XDigit}{4}-\p{XDigit}{12} characters.

Request header

Header Value Use Description
X-CSRF-TOKEN {csrfToken} Required

Provides a session token that authorizes you to access the DELETE and POST API calls

To view the token for your current session, do one of the following:

  • To capture the csrfToken response parameter, see Get security token.
  • Retrieve the XSRF-TOKEN cookie after making an API call from a session that is already in progress.

Keep in mind that if a session expires or changes, you must acquire a new security token.
Accept application/xml Optional Requests that Empower return an XML response instead of the default JSON response

Request parameters

This call does not require parameters.

Example request

Element Example

Request header

 POST http://sample-server.example.com:8080/mpw/resource/documents/beeddca5-ee6b-4475-a9db-c04a1df97742/restore
X-CSRF-TOKEN: 95cf86a3-079f-4860-aa62-dd64f8a8c321
Request body This API call does not support a request body.

Response parameters

If a parameter object does not contain a value, a JSON response returns a null value for the object, but an XML response does not include the object.

Parameter Data type Description
accessgroupNames String

The access group (or groups) that control access to the document

Access groups are assigned by the Empower application when the document is imported into Empower.

Note: The accessgroupNames parameter can be used by itself, or in conjunction with document owners. To see whether access to the document is restricted to certain document owners, look at the response that was returned with the ownerIds parameter.
appId String The ID of the Empower application that is associated with the document in Empower
applicationName String The name of the Exstream application that was used to design the document
busDocId String The business-defined document ID that was assigned to the document
creationDate Date

The date that the document was created

The date format is YYYY-MM-ddTHH:mm:ss+Timezone.
deleted Boolean The flag that indicates a document has been exported or deleted, and is marked for permanent removal by the scheduled cleanup process
docId UUID The document ID that was generated by Empower when the document was uploaded
docTags String The metadata that was added to the document
documentVersion String The version number of the document
editorVersion String

The version of Empower Editor that can be used to open the document

The editor version is assigned by the Empower application when the document is imported into Empower.
engineVersion String The version of the engine that was used to generate the document
exceedsTotal Boolean The response parameter that indicates whether the number of documents that match your request exceeds the value that is returned for the total response parameter

exportDate

Date

The most recent date and time that the document was marked for deletion (deleted=true)

The date format is YYYY-MM-ddTHH:mm:ss+Timezone.
fileName String The name of the document file that was imported
importDate Date

The date and time that the document was uploaded to the server

The date format is YYYY-MM-ddTHH:mm:ss+Timezone.

lastEditDate

Date

The most recent date and time that the document was edited

The date format is YYYY-MM-ddTHH:mm:ss+Timezone.
lastSaveDate Date

The most recent date and time that a document revision was saved

The date format is YYYY-MM-ddTHH:mm:ss+Timezone.
limit Long

The maximum number of documents that the request could have returned (the page size)

If the value of the limit response parameter is 0, then no limit was used, and the response included all of the documents that matched your request parameters.
offset Long

The number of results that were skipped before the first result was returned
ownerIds String

The user ID(s) of the document owner(s) who are permitted to open and edit the document

Note: The ownerID parameter can be used by itself, or in conjunction with access groups. To see whether the document is restricted by access grups, look at the response that was returned with the accessgroupNames parameter.
packageFileName String The name of the package file that was used to generate the document

packageVersion

String

 The version of Exstream that was used to package the document
previewPubFile String

The name of the Exstream package file that is used to generate PDF previews of the document (when Engine as a Web Service is configured as the preview provider)
total Long

The total number of documents that match your request parameters

If your request includes the maxTotal parameter, Empower counts to the last application that is returned by the search, or to the maxTotal number of applications (whichever is larger).

userId

String

 The ID of the user that imported the document
