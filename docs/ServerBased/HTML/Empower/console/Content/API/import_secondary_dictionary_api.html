Import a secondary dictionary

The POST API call for Empower Editor dictionaries imports a Hunspell dictionary file to the Empower Services Layer and maps it to an existing primary dictionary as an additional resource. For each language dictionary in Empower Editor, you can specify a secondary dictionary. Note that Empower supports only a single primary dictionary and a single secondary dictionary for each language.

Who can access this call

Access to the Empower API is defined in the customSecurityContext.xml file during installation, and is controlled by your authorization provider. By default, Empower supports the administrator, editor, and integrator authorities.

Authenticated users with the following authority can use this API call:

  • administrator

Request

Request method Content type Syntax
POST multipart/form-data /<context>/resource/dictionaries

The <context> variable is configured during Empower installation.

Request header

Header Value Use Description
X-CSRF-TOKEN {csrfToken} Required

Provides a session token that authorizes you to access the DELETE and POST API calls

To view the token for your current session, do one of the following:

  • To capture the csrfToken response parameter, see Get security token.
  • Retrieve the XSRF-TOKEN cookie after making an API call from a session that is already in progress.

Keep in mind that if a session expires or changes, you must acquire a new security token.
Accept application/xml Optional Requests that Empower return an XML response instead of the default JSON response

Request parameters

Parameter Data type Use Description
dictionaryName String Required Specifies the name of the existing Empower Editor primary dictionary to which the new secondary dictionary file is mapped

file

File

Optional

Imports the dictionary file and maps it to the specified primary dictionary

Note that if there is already a secondary dictionary, the dictionary file that you import replaces the existing secondary dictionary.

The maximum file size is 200 MB.

Empower supports Hunspell dictionary files with a .dic extension.

To remove an existing secondary dictionary, run the API call without this parameter.

Example request

Element Example

Request header

 POST http://sample-server.example.com:8080/mpw/resource/dictionaries
X-CSRF-TOKEN: 95cf86a3-079f-4860-aa62-dd64f8a8c321

Request body

Content-Type: multipart/form-data;

Content-Disposition: form-data; name="file"; filename="en_GB_medical.dic"
<actual file content, not shown here>

Response

Empower supports two formats for API response payloads: JSON and XML.

Example JSON response

Media type: application/json

{
    "header": {
        "status": {
            "code": "SERV100I"
        }
    },
    "body": {
        "dictionaryMapping": {
            "primaryDictionaryLanguage: "English (British)",
            "primaryDictionaryName": "en_GB",
            "secondaryDictionaryName": "en_GB_medical"
        }
    }
}

Example XML response

Media type: application/xml

        <?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<ResponseContainer>
   <header>
      <status code="SERV100I"/>
   </header>
   <body>
        <dictionaryMapping primaryDictionaryLanguage="English (British)" primaryDictionaryName="en_GB" secondaryDictionaryName="en_GB_medical"/>
   </body>
</ResponseContainer>

Response parameters

If a parameter object does not contain a value, a JSON response returns a null value for the object, but an XML response does not include the object.

Parameter Data type Description
primaryDictionaryLanguage String

The language of the primary dictionary
primaryDictionaryName String

The name of the primary dictionary
secondaryDictionaryName String The name of the secondary dictionary file (if one exists)
