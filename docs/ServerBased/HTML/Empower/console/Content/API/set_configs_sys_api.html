Set configurations

The POST API call for configurations edits the attributes of the Empower system configuration.

Who can access this call

Access to the Empower API is defined in the customSecurityContext.xml file during installation, and is controlled by your authorization provider. By default, Empower supports the administrator, editor, and integrator authorities.

Authenticated users with the following authority can use this API call:

  • administrator

Request

Request method Content type Syntax
POST application/x-www-form-urlencoded /<context>/resource/config

The <context> variable is configured during Empower installation.

Request header

Header Value Use Description
X-CSRF-TOKEN {csrfToken} Required

Provides a session token that authorizes you to access the DELETE and POST API calls

To view the token for your current session, do one of the following:

  • To capture the csrfToken response parameter, see Get security token.
  • Retrieve the XSRF-TOKEN cookie after making an API call from a session that is already in progress.

Keep in mind that if a session expires or changes, you must acquire a new security token.
Accept application/xml Optional Requests that Empower return an XML response instead of the default JSON response

Request parameters

You must use at least one of the following parameters in your request:

Parameter Data type Use Description
defaultEditor String
 Optional

Sets the version of the editor that is assigned to all new Empower applications

The maximum string length is 250 characters.

This parameter supports the ^[a-zA-Z0-9\.]*$ characters.

Note: Only an active editor can be set as the system default editor.
defaultLocale String

Optional

Sets the alternative language that appears in Empower Editor and in the Empower documentation if the end user's web browser is configured to use a language that is not supported by Empower

The default value for this parameter is en_US.
idleDocCleanup String Optional

Sets whether the scheduled cleanup process marks documents for deletion when they have not been edited within the retention period that is specified in the idleDocRetention parameter

This parameter can have a value of Enabled or Disabled.

The default value for this parameter is Disabled.
idleDocRetention Integer Optional

Sets the number of days that documents are retained after they are imported or edited

When the scheduled cleanup process runs, if the idleDocCleanup parameter is enabled, documents that have not been edited within the specified retention period are marked for deletion.

This parameter must have an integer value from 1–30.

The default value for this parameter is 15 days.
preview String Optional

Turns the PDF preview service on or off

This parameter can have a value of Enabled or Disabled.

The default value for this parameter is Enabled.
previewTimeout

Integer

 Optional

Sets the length of time (in milliseconds) that Empower Editor will wait for a response from the PDF preview service

The default value for this parameter is 30000, or 30 seconds.
retentionTime

Integer

 Optional

Sets length of time (in milliseconds) that documents are retained after they are marked for deletion

The scheduled cleanup process will uninstall a deactivated editor immediately after the last document that uses the version of the editor is deleted.

When the retention period expires, eligible objects are deleted during the next scheduled cleanup.

The default value for this parameter is 604800000, or one week.
supportEmail String Optional

Sets the email address to which Empower Editor sends messages

The maximum string length is 50 characters.

This parameter supports the [\p{L}\p{M}\p{N}!#$%&'*+\/=?^_`{|}~-]+(?:\.[\p{L}\p{M}\p{N}!#$%&'*+\/=?^_`{|}~-]+)*@(?:[\p{L}\p{M}\p{N}](?:[\p{L}\p{M}\p{N}-]*[\p{L}\p{M}\p{N}])?\.)+[\p{L}\p{M}\p{N}](?:[\p{L}\p{M}\p{N}-]*[\p{L}\p{M}\p{N}])? characters.
triggerExpressionType String Optional

Specifies the schedule on which the cleanup process runs. Based on the selection, the cleanup process runs on a daily or hourly basis.

This parameter can have a value of Daily or Hourly.

The default value for this parameter is Daily.

If you specify Daily, you must set the triggerDays and triggerHours parameters. The triggerMinutes parameter is optional. However, if you do not set the triggerMinutes parameter, the default is the last value it was set to. If the triggerMinutes parameter has never been set, the default value is 0.

If you specify Hourly, you must set the triggerHours parameter. The cleanup process will run at midnight and then at the interval you specify on the Hourly parameter.
triggerDays String Optional

Specifies the days of the week on which the scheduled cleanup process runs (used in the triggerExpression parameter)

The maximum string length is 50 characters.

This parameter must match the ^(((MON|TUE|WED|THU|FRI|SAT|SUN)(,(MON|TUE|WED|THU|FRI|SAT|SUN))*)$ regular expression.

Example:

triggerDays="MON,TUE,WED"

You can specify multiple values for this parameter, separated by commas.
triggerExpression String Optional

Defines the cron expression that sets the cleanup schedule

You can build your own cron expression or use the triggerDays, triggerHours and triggerMinutes parameters to set the value for this parameter.

The maximum string length is 250 characters.

This parameter supports the ^[a-zA-Z0-9\.]*$ characters.

Example:

triggerExpression="0 15 20 ? * MON,TUE,WED *"

The default value for this parameter is 0 0 12 ? * SAT *, or every Saturday at 12:00:00 PM.

Note: If you use an API call to set the cleanup schedule, the cleanup schedule settings that are displayed in the Empower Server Console might not reflect the values that are set by the API call.
triggerHours String Optional

Specifies the hour of the day in which the scheduled cleanup process runs (used in the triggerExpression parameter)

The maximum string length is two characters.

This parameter supports the ^(0?[0-9]|1[0-9]|2[0-3])$ characters.

Example:

triggerHours="20"

triggerMinutes String Optional

Specifies the minute of the hour in which the scheduled cleanup process runs (used in the triggerExpression parameter)

The maximum string length is two characters.

This parameter supports the ^(0?[0-9]|[0-5][0-9])$ characters.

Example:

triggerMinutes="15"

Example requests

The following examples show how to create a dailly and an hourly request using the parameters that are supported by this API call.

Example 1:

Request to schedule cleanup every week on Saturday at 8:15 PM.

Element Example

Request header

 POST http://sample-server.example.com:8080/mpw/resource/config
X-CSRF-TOKEN: 95cf86a3-079f-4860-aa62-dd64f8a8c321
Request body Content-Type: application/x-www-form-urlencoded;

Content-Disposition: form-data;
triggerExpressionType=Daily&triggerDays=SAT&triggerHours=20&triggerMinutes=15

Example 2:

Request to schedule cleanup every 4 hours.

Element Example
Request header POST http://sample-server.example.com:8080/mpw/resource/config
X-CSRF-TOKEN: 95cf86a3-079f-4860-aa62-dd64f8a8c321
Request body Content-Type: application/x-www-form-urlencoded;

Content-Disposition: form-data;
triggerExpressionType=Hourly&triggerHours=4

Empower supports two formats for API response payloads: JSON and XML.

Media type: application/json

{
    "header": {
        "status": {
            "code": "SERV100I"
        }
    },
    "body": {
        "config": {
            "analyticsNotice": "Enabled",
	     "defaultEditor": "2.0.1.0",
            "defaultLocale": "en_US",
            "idleDocCleanup": "Disabled",
            "idleDocRetention": 15,
            "preview": "Enabled",
            "previewTimeout": 30000,
            "retentionTime": 604800000,
            "servicesVersion": "2.0.1.0",
            "supportEmail": "support@example.com",
            "triggerDays": "?",
            "triggerExpression": "0 0 0/4 1/1 * ? *",
            "triggerExpressionType": "Hourly",
	     "triggerHours": "0,4,8,12,16,20",
            "triggerMinutes": "0"
        }
    }
}
