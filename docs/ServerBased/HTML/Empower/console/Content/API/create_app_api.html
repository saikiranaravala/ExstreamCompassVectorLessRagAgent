Create an application

The POST API call for applications creates a new Empower application.

Empower applications store information about the Exstream application, the package file, the database, and the Exstream engine that were originally used to create the documents that are associated with an Empower application. Therefore, before you can import an Empower document, you must make sure that there is an Empower application to support that document.

Who can access this call

Access to the Empower API is defined in the customSecurityContext.xml file during installation, and is controlled by your authorization provider. By default, Empower supports the administrator, editor, and integrator authorities.

Authenticated users with the following authority can use this API call:

  • administrator

Request

Request method Content type Syntax
POST multipart/form-data /<context>/resource/applications

The <context> variable is configured during Empower installation.

Request header

Header Value Use Description
X-CSRF-TOKEN {csrfToken} Required

Provides a session token that authorizes you to access the DELETE and POST API calls

To view the token for your current session, do one of the following:

  • To capture the csrfToken response parameter, see Get security token.
  • Retrieve the XSRF-TOKEN cookie after making an API call from a session that is already in progress.

Keep in mind that if a session expires or changes, you must acquire a new security token.
Accept application/xml Optional Requests that Empower return an XML response instead of the default JSON response

Request parameters

Parameter Data type Use Description

accessgroupName

String

 Optional

Specifies the name of the access group that will be used to restrict access to all new documents that are created using this application

If you do not specify an access group name, then all authenticated users are permitted to open and edit the documents that are associated with this application.

The maximum string length is 250 characters.

This parameter supports the ^[\\p{L}\\p{M}\\p{N}_\\-\\.\\*\\s"\\\\]*$ characters.

To add multiple access groups to an application, include multiple accessgroupName parameters in the request.

Example:

accessgroupName=group1&accessgroupName=group2&accessgroupName=group33

Note: You can apply additional access restrictions to an Empower document by specifying one or more owners for the document. For information about using the ownerId parameter, see Update a document.

editorVersion

String

Optional

Sets the version of the editor that is stored in an Empower application and that is assigned to all new documents

The maximum string length is 250 characters.

This parameter supports the ^[a-zA-Z0-9\.]*$ characters.

If you do not include the editorVersion parameter, Empower assigns the system default editor to the new application.

Note: You must specify an active version of the editor.

file

File

Required

Uploads the specified document, extracts information about the Exstream application from the document, and then compares this information to existing Empower applications to determine whether an identical Empower application already exists in the repository

The maximum file size is 200 MB.

Note: Empower uses this parameter to extract information about the application, but does not import the document. To import the document, see Import a document.
previewPubFile String Optional

Specifies the name of the Exstream package file that all new documents will use to generate PDF previews (when Engine as a Web Service is configured as the PDF preview provider)

The maximum string length is 250 characters.

This parameter supports the ^[\p{L}\p{M}\p{N}_\-\.\s]*$ characters.

Response

Empower supports two formats for API response payloads: JSON and XML.

Example JSON response

Media type: application/json

The following example shows the type of response that is returned when an application is created:

{
    "header": {
        "status": {
            "code": "SERV100I"
        }
    },
    "body": {
        "app": {
            "appId": "51-1115057666-1531728594-0",
            "applicationName": "EditAreaTextParagraph",
            "applicationOI": "3",
            "applicationPath": "",
            "databaseId": "c042a721-d414-44eb-8d78-72dd7babba43",
            "editorVersion": "1.1.0.7948",
            "engineVersion": "900101",
            "packageCreationDate": "2014-06-16T18:12:57",
            "packageFile": "ExstreamPackage.pub",
            "packageVersion": "900122",
            "previewPubFile": "PdfPreview.pub",
            "accessgroupNames": [
                "sampleGroup"
            ]        }
    }
}

Example XML response

Media type: application/xml

The following example shows the type of response that is returned when a matching application already exists:

<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<ResponseContainer>
    <header>
        <status code="BUSI172E">
            <msg>The application could not be created, because it already exists with the following ID: 51-1115057666-1531728594-0.</msg>
        </status>
    </header>
</ResponseContainer>

Note: Keep in mind that if a matching Empower application already exists, it is because another instance of the same document was previously used to create an application. If you then import the document, the attributes of the existing application will be assigned to the document. However, if you update the application with new attributes, all instances of the document that are imported after the change will be assigned the new attributes.

Response parameters

If a parameter object does not contain a value, a JSON response returns a null value for the object, but an XML response does not include the object.

Parameter Data type Description
accessgroupNames String The names of the access groups that control access to documents that are associated with the Empower application
appId String The application ID that was generated by Empower
applicationName String The name of the Exstream application that was used to design the document
applicationOI Number The Object Identifier (OI) of the Exstream application
applicationPath String

The folder path of the Exstream application in your installation of Design Manager

Note: If the application is located in the root folder of your installation of Design Manager, the applicationPath field will be empty.
databaseId UUID The database ID stored in the Empower application
editorVersion String The version of the editor that is assigned to all new documents
engineVersion String The version of the engine that was used to package the Exstream application
exceedsTotal Boolean The response parameter that indicates whether the total number of Empower applications that match your request exceeds the value that is returned for the total response parameter
limit Long

The maximum number of Empower applications that the request could have returned (the page size)

If the value of the limit response parameter is 0, then no limit was used, and the response included all of the applications that matched your request parameters.
offset Long The number of results that were skipped before the first result was returned

packageCreationDate

Date

The date that the package file was created

The date format is YYYY-MM-ddTHH:mm:ss.
packageFile String The name of the Exstream application package file, including the folder path

packageVersion

String

 The version of Exstream that was used to package the application
previewPubFile String The name of the Exstream package file that all new documents will use to generate PDF previews (when Engine as a Web Service is configured as the preview provider)
total Long

The total number of Empower applications that match your request parameters

If your request includes the maxTotal parameter, Empower counts to the last Empower application that is returned by the search, or to the maxTotal number of applications (whichever is larger).
