Get access groups

The GET API call for access groups returns a list of the access groups that match your request parameters.

Who can access this call

Access to the Empower API is defined in the customSecurityContext.xml file during installation, and is controlled by your authorization provider. By default, Empower supports the administrator, editor, and integrator authorities.

Authenticated users with the following authority can use this API call:

  • administrator

Request

Request method Syntax
GET /<context>/resource/accessGroups?{parameters}

The <context> variable is configured during Empower installation.

The {parameters} search variable is optional. If you do not include any parameters, the search returns a list of all access groups.

Request header

Header Value Use Description
Accept application/xml Optional Requests that Empower return an XML response instead of the default JSON response

Request parameters

You can use the sorting, filtering, searching, and page size parameters to limit the number of access groups that are returned in a response.

Sorting

Parameter Data type

Use

 Description
sortField String Optional

Returns a list of access groups that is sorted by the value of the specified parameter

The maximum string length is 50 characters.

This parameter supports the ^[A-Za-z]* characters.
sortOrder String Optional

Sorts the list of access groups in ascending or descending order

This parameter can have a value of ASC or DESC.

Note: If you use the sortField parameter without the sortOrder parameter, the list of access groups is sorted in ascending order.

Filtering

To filter the results when you search Empower access groups, specify values for one or more of the parameters in the following table.

You can use these tips to help you refine your search:

  • To exclude a string from a filter request, precede the string with a dash (-). When you precede a string with a dash, Empower excludes access groups that match the string that follows the dash.

  • To look for a string that begins with a dash, add the backslash (\) escape character just before the dash in your search string.

  • To narrow the filtering results, you can string multiple parameters together with an ampersand (&).
Parameter Data type Use Description
accessGroupId String Optional

Returns the access group that matches the unique ID

The maximum string length is 250 characters.

This parameter supports the ^[0-9]*$ characters.
accessGroupName String Optional

Returns a list of the access groups that match the specified access group name

The maximum string length is 250 characters.

This parameter supports the ^[\p{L}\p{M}\p{N}_\-\.\s]*$ characters.

Searching

You can search across a set of supported string fields.

Note: Empower does not support using the dash (-) operator to exclude a search term on the q parameter.

Parameter Data type

Use

 Description
q String Optional

Returns a list of the access groups that match the search value that is specified for any of the following indexed string parameter fields:

  • accessGroupId
  • accessGroupName

The maximum string length is 500 characters.

This parameter supports the ^[\p{L}\p{M}\p{N}\.\-\_\*\s"]*$ characters. 

Using page size

To control the number of results that are returned when you search a large set of access groups, you can use page size settings in your integration applications to limit the number of results that are displayed on each page, and to return a defined subset of your search results.

Parameter Data type

Use

 Description
limit String Optional

Sets the maximum number of matching results that are returned (the page size)

Note: You must use the limit and offset parameters together.
offset String Optional

Sets the number of matching results that are skipped before the first result is displayed

For example, if you specify a value of limit=50&offset=100 for this parameter and your search returns 125 results, then Empower would display results 101-125.

Note: You must use the limit and offset parameters together.

Example requests

The following examples show how to create various types of requests using some of the parameters that are supported by this API call. Keep in mind that this API call does not support using a request body.

Example 1:

Request a list of all access groups with no pagination.

Element Example

Request header

 GET hhttp://sample-server.example.com:8080/mpw/resource/accessGroups

Example 2:

Request a single access group by ID (if the access group exists).

Element Example

Request header

 GET http://sample-server.example.com:8080/mpw/resource/accessGroups?accessGroupId=9a48b678-4d65-41b8-9bde-ca4bb5ab1524

Example 3:

Request a list of all access groups, sorted in ascending order by group name, and displaying only the first five access groups.

Element Example

Request header

GET http://sample-server.example.com:8080/mpw/resource/accessGroups?sortField=accessGroupName&limit=5&offset=0

Response

Empower supports two formats for API response payloads: JSON and XML.

Example JSON response

Media type: application/json

{
    "header": {
        "status": {
            "code": "SERV100I"
        }
    },
    "body": {
        "accessGroup": [
            {
                "accessGroupId": "a89c20ae-f2b8-42ef-b098-e36834470221",
                "accessGroupName": "CustomerService",
                "interactiveRoles": [
                    "CSR1"
                ]
             }
        ],
        "total": 1,
        "offset": 0,
        "limit": 0
    }
}

Example XML response

Media type: application/xml

                <?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<ResponseContainer>
    <header>
        <status code="SERV100I"/>
    </header>
    <body>
        <accessGroup-search-result>
            <limit>0</limit>
            <offset>0</offset>
            <accessGroups>
                <accessGroup accessGroupId="9a48b678-4d65-41b8-9bde-ca4bb5ab1524" accessGroupName="CustomerService"/>
                    <interactiveRoles>CSR1</interactiveRoles>
                </accessGroup>
            </accessGroups>
            <total>1</total>
        </accessGroup-search-result>
    </body>
</ResponseContainer>

Response parameters

If a parameter object does not contain a value, a JSON response returns a null value for the object, but an XML response does not include the object.

Parameter Data type Description
accessGroupId String

The ID of the access group

This value is generated automatically by Empower.
accessGroupName String The name of the access group
interactiveRoles String

The interactive roles that are associated with this access group
