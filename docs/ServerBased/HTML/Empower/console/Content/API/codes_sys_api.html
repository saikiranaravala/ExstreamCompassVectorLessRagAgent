View response codes

The GET API call for response codes returns a file that contains the Empower API response messages and their associated codes.

Who can access this call

Access to the Empower API is defined in the customSecurityContext.xml file during installation, and is controlled by your authorization provider. By default, Empower supports the administrator, editor, and integrator authorities.

Authenticated users with any of the following authorities can use this API call:

  • administrator
  • integrator

Request

Request method Syntax
GET /<context>/resource/config/errorCodes

The <context> variable is configured during Empower installation.

Content type

text/html

Request parameters

This call does not require parameters.

Example request

Element Example

Request header

 GET http://sample-server.example.com:8080/mpw/resource/config/errorCodes
Request body This API call does not support a request body.

Response

If the GET API call for response codes is successful, an HTML file that includes the response launches in the browser window.

Example response

The HTML response file includes the reference variable name, the response code, and the response message.

Example:

SERVICES_DATA_FILE_EMPTY = SERV102E|The uploaded file was empty.
SERVICES_MULTIPLE_DATA_FILE_EMPTY = SERV115E|The file associated with the following variable number was empty: {0}.
SERVICES_MISSING_MANDATORY_PARAM = SERV103E|The following parameter is required: {0}.
SERVICES_FILE_TOO_BIG = SERV104E|The file exceeds the maximum file size of {0} bytes.
SERVICES_RESOURCE_FILE_TOO_BIG = SERV105E|The file with the following variable number exceeds the maximum file size of {0} bytes: {1}.
