Get editors

The GET API call for editors returns a list of the editors that match your request parameters.

Who can access this call

Access to the Empower API is defined in the customSecurityContext.xml file during installation, and is controlled by your authorization provider. By default, Empower supports the administrator, editor, and integrator authorities.

Authenticated users with the following authority can use this API call:

  • administrator

Request

Request method Syntax
GET /<context>/resource/editor?{parameters}

The <context> variable is configured during Empower installation.

The {parameters} search variable is optional. If you do not include any parameters, Empower returns a list of all editors.

Request header

Header Value Use Description
Accept application/xml Optional Requests that Empower return an XML response instead of the default JSON response

Request parameters

You can filter editor results by specifying values for one or more of the following parameters:

Parameter Data type

Use

 Description
active Boolean Optional

Returns a list of the editors that are either active or inactive

This parameter can have a value of true or false.

By default, Empower returns all editors.
editorVersion String Optional

Returns an editor with the specified version

The maximum string length is 250 characters.

This parameters supports the ^[a-zA-Z0-9\.]*$ characters.

Example requests

The following examples show how to create various types of requests using some of the parameters that are supported by this API call. Keep in mind that this API call does not support using a request body.

Example 1:

Request a list of all editors.

Element Example

Request header

GET http://sample-server.example.com:8080/mpw/resource/editor

Example 2:

Request a single editor by version number (if the specified editor exists).

Element Example

Request header

 GET http://sample-server.example.com:8080/mpw/resource/editor?editorVersion=1.0.2.7273

Example 3:

Request a list of all editors that have been deactivated.

Element Example

Request header

GET http://sample-server.example.com:8080/mpw/resource/editor?active=false

Response parameters

If a parameter object does not contain a value, a JSON response returns a null value for the object, but an XML response does not include the object.

Parameter Data type Description
active Boolean

Indicates whether the version of the editor is active

The value of this parameter can be true (the editor is active) or false (the editor is inactive).
defaultEditor Boolean

Indicates whether the version of the editor is the system default editor

The value of this parameter can be true or false.

Note: The system default editor is the version of the editor that is assigned to an Empower application the first time that a document designed with the application is imported to Empower.

For more information about the relationship between documents and applications, see Import a document.
supportedLocales String The list of locale-based languages that the specified version of the editor supports
version String The version number of the instance of Empower Editor
