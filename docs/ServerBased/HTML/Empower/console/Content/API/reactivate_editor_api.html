Reactivate an editor

The reactivate POST API call for editors removes the flag that marks the specified version of the editor for uninstallation by the scheduled cleanup process.

Who can access this call

Access to the Empower API is defined in the customSecurityContext.xml file during installation, and is controlled by your authorization provider. By default, Empower supports the administrator, editor, and integrator authorities.

Authenticated users with the following authority can use this API call:

  • administrator

Request

Request method Content type Syntax
POST application/x-www-form-urlencoded /<context>/resource/editor/{editorVersion}/reactivate

The <context> variable is configured during Empower installation.

The {editorVersion} is the version number of the editor that you want to reactivate.

The maximum string length of the {editorVersion} parameter is 250 characters.

The {editorVersion} parameter supports the ^[a-zA-Z0-9\.]*$ characters.

Request header

Header Value Use Description
X-CSRF-TOKEN {csrfToken} Required

Provides a session token that authorizes you to access the DELETE and POST API calls

To view the token for your current session, do one of the following:

  • To capture the csrfToken response parameter, see Get security token.
  • Retrieve the XSRF-TOKEN cookie after making an API call from a session that is already in progress.

Keep in mind that if a session expires or changes, you must acquire a new security token.
Accept application/xml Optional Requests that Empower return an XML response instead of the default JSON response

Request parameters

This call does not require parameters.

Example request

Element Example

Request header

 POST http://sample-server.example.com:8080/mpw/resource/editor/1.0.2.7170/reactivate
X-CSRF-TOKEN: 95cf86a3-079f-4860-aa62-dd64f8a8c321
Request body This API call does not support a request body.

Response

Empower supports two formats for API response payloads: JSON and XML.

Example JSON response

Media type: application/json

{
    "header": {
        "status": {
            "code": "SERV100I"
        }
    },
    "body": {
        "editors": [
            {
                "active": true,
                "defaultEditor": false,
                "supportedLocales": [
                    "de_DE",
                    "en_US",
                    "es_MX",
                    "fr_FR",
                    "pt_BR"
                ],
                "version": "1.0.2.7170"
            }
        ]
    }
}

Example XML response

Media type: application/xml

        <?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<ResponseContainer>
    <header>
        <status code="SERV100I"/>
    </header>
    <body>
        <editor active="true" defaultEditor="false" supportedLocales="de_DE en_US es_MX fr_FR pt_BR" editorVersion="1.0.2.7170"/>
    </body>
</ResponseContainer>

Response parameters

If a parameter object does not contain a value, a JSON response returns a null value for the object, but an XML response does not include the object.

Parameter Data type Description
active Boolean

Indicates whether the version of the editor is active

The value of this parameter can be true (the editor is active) or false (the editor is inactive).
defaultEditor Boolean

Indicates whether the version of the editor is the system default editor

The value of this parameter can be true or false.

Note: The system default editor is the version of the editor that is assigned to an Empower application the first time that a document designed with the application is imported to Empower.

For more information about the relationship between documents and applications, see Import a document.
supportedLocales String The list of locale-based languages that the specified version of the editor supports
version String The version number of the instance of Empower Editor
