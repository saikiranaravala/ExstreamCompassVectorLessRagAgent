Get security token

The GET API call for tokens returns information about the security token that is associated with your current Empower session. This token must be included in the header of Empower DELETE and POST API calls. Keep in mind that if a session expires or changes, you must acquire a new token.

Who can access this call

Access to the Empower API is defined in the customSecurityContext.xml file during installation, and is controlled by your authorization provider. By default, Empower supports the administrator, editor, and integrator authorities.

Authenticated users with any of the following authorities can use this API call:

  • administrator
  • editor
  • integrator

Request

Request method Syntax
GET /<context>/resource/GetToken

The <context> variable is configured during Empower installation.

Request header

Header Value Use Description
Accept application/xml Optional Requests that Empower returns an XML response instead of the default JSON response

Request parameters

This call does not require parameters.

Example request

Element Example

Request header

GET http://sample-server.example.com:8080/mpw/resource/GetToken

Request body This API call does not support a request body.

Response

Empower supports two formats for API response payloads: JSON and XML.

Example JSON response

Media type: application/json

{
    "header": {
        "status": {
            "code": "SERV100I"
        }
    },
    "body": {
        "csrfHeader": "X-CSRF-TOKEN",
        "csrfParameter": "_csrf",
        "csrfToken": "3115153d-bc8f-437d-ba6b-27e87cba97a3"
    }
}

Example XML response

Media type: application/xml

        <?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<ResponseContainer>
    <header>
        <status code="SERV100I"/>
    </header>
    <body>
        <csrf csrfHeader="X-CSRF-TOKEN" csrfParameter="_csrf" csrfToken="c4fd6bde-58c9-422d-b370-abfff6c9269d"/>
    </body>
</ResponseContainer>

Response parameters

If a parameter object does not contain a value, a JSON response returns a null value for the object, but an XML response does not include the object.

Parameter Data type Description
csrfHeader String The header name of the security token
csrfParameter String

The parameter name of the security token

Note: This response parameter is included for informational purposes only. You must use the HTML request header for an API call to supply the session token information to Empower.

For information about structuring Empower API request headers, see Request HTTP Header.
csrfToken String The value of the security token
