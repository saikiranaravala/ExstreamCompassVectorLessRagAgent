Export a document

The POST API call for documents returns the specified Empower document in the body of the API response, and then marks the document for permanent removal by the scheduled cleanup process. You can also include a parameter that prevents the document from being marked for deletion following export.

Who can access this call

Access to the Empower API is defined in the customSecurityContext.xml file during installation, and is controlled by your authorization provider. By default, Empower supports the administrator, editor, and integrator authorities.

Authenticated users with any of the following authorities can use this API call:

  • administrator
  • integrator

Request

Request method Syntax
POST /<context>/resource/documents/{docId}/export

The <context> variable is configured during Empower Services Layer setup.

The {docId} is the ID of the existing document.

The maximum string length of the {docId} parameter is 36 characters.

The {docId} parameter supports the \p{XDigit}{8}-\p{XDigit}{4}-\p{XDigit}{4}-\p{XDigit}{4}-\p{XDigit}{12} characters.

Request header

Header Value Use Description
X-CSRF-TOKEN {csrfToken} Required

Provides a session token that authorizes you to access the DELETE and POST API calls

To view the token for your current session, do one of the following:

  • To capture the csrfToken response parameter, see Get security token.
  • Retrieve the XSRF-TOKEN cookie after making an API call from a session that is already in progress.

Keep in mind that if a session expires or changes, you must acquire a new security token.

Request parameters

Parameter Data type Use Description
preserveDoc Boolean
 Optional

The preserveDoc parameter prevents a document from being marked for deletion in the Empower Services Layer.

This parameter can have a value of true or false and must be sent in the body of the request.

If you omit the parameter or set its value to false, the document is marked for deletion.

Example requests

The following examples show how to create various types of requests using some of the parameters that are supported by this API call.

Example 1:

Export the document and mark it for deletion.

Element Example

Request header

 POST http://sample-server.example.com:8080/mpw/resource/documents/d7cc8545-9e0a-4949-b593-1667d9741326/export
X-CSRF-TOKEN: 95cf86a3-079f-4860-aa62-dd64f8a8c321

Example 2:

Export the document without marking it for deletion.

Element Example

Request header

 POST http://sample-server.example.com:8080/mpw/resource/documents/d7cc8545-9e0a-4949-b593-1667d9741326/export
X-CSRF-TOKEN: 95cf86a3-079f-4860-aa62-dd64f8a8c321
Request body

preserveDoc=true

Response

If the POST API call for exporting a document is successful, the Empower document that you specified is returned in the body of the response, and the document is either kept or marked for deletion in the Empower Services Layer.
