Open a document

The GET API call for documents opens a specified Empower document in the version of Empower Editor that is assigned to the document.

Who can access this call

Access to the Empower API is defined in the customSecurityContext.xml file during installation, and is controlled by your authorization provider. By default, Empower supports the administrator, editor, and integrator authorities.

Authenticated users with any of the following authorities can use this API call:

  • administrator
  • editor

Request

Request method Syntax
GET /<context>/resource/docedit/{docId}/open

The <context> variable is configured during Empower installation.

The {docId} is the ID of the existing Empower document.

The maximum string length of the {docId} parameter is 36 characters.

The {docId} parameter supports the \p{XDigit}{8}-\p{XDigit}{4}-\p{XDigit}{4}-\p{XDigit}{4}-\p{XDigit}{12} characters.

Request header

This call does not require headers.

Note: If your network uses a reverse proxy server to terminate HTTPS traffic and your Empower server does not use HTTPS, then you must add the X-Forwarded-Proto, X-Forwarded-HOST, and X-Forwarded-Port headers to this request in order to receive a response from the Empower server.

For information about what values to use with the X-Forwarded headers, consult the administrator of your reverse proxy server.

Request parameters

This call does not require parameters. To specify the locale version of the editor that the end user will see, use the Accept-Language header in your request. If you specify a locale that Empower does not support, the system default locale will be used. To learn how to set the system default locale, see Set configurations.

Example request

Element Example

Request header

 GET http://sample-server.example.com:8080/mpw/resource/docedit/4bbc6032-0826-4d53-a408-2582610c9192/open
Request body This API call does not support a request body.

Response

If the GET API call for documents is successful, Empower Editor launches in the browser window, and the specified Empower document is displayed in the editor, ready for the end user to begin an editing session.

If access to the document is restricted by an access group or document owner (using the accessgroupName parameter or the ownerId parameter), and a user who is not a document owner or a member of a restricting access group attempts to open the document, then the user will receive the following error:

{"header":{"status":{"code":"BUSI175E ","msg":" Document <docId> could not be accessed, because the following user does not have the necessary permissions: <current userId>."}}}
