Adding variables to content

Depending on how your communication was designed, you might be able to add variables to text boxes or tables, or directly in text messages or clauses. You would do this if you are working on a communication that will be sent to multiple customers. In that case, you can add variables to content so that it is personalized for each customer who receives the communication.

Simply put, variables are placeholders that are populated with data (such as a customer name or account number) before the communication is sent to a customer. The person who designs your communication determines not only which variables are available for you to use in Content Author, but also whether variables are available at all. Additionally, the designer of the communication has the option to provide a user-friendly name for each variable. So if you are confused about the name of a variable, you should talk to the person who designed the communication.

If you are working with a clause that will be editable in Empower Editor, you will see additional options on the variable card. Select the variable and then click on the variable card to set interactive properties for the variable.

Tip: You will see sample data when you add variables to your content. Variables are populated with actual customer data only after you have finalized your changes to each customer communication, and the communication has been approved for production. You can switch between viewing sample data and variable names by clicking on the toolbar in Authoring view.

One potentially confusing aspect of adding variables to your content is choosing how the data that populates them will be formatted before a communication is sent to customers. For example, you might want to specify that date variables appear in a specific format, such as mm/dd/yyyy.

To specify an output format for a variable:

  • In Authoring view, select the variable in the text message or clause, and then select an output format from the Output format list on the card for the variable.

    Note: For multi-value variables, you can also specify which values to include in the output, and how the list of values appears in the output.

Here is a brief description of the output format options that are available for each variable type:

True/false variables

  • Use variable output format—Uses the output format specified by the person who designed the communication.

  • T/F—Returns T when the condition is true or F when the condition is false.

  • Y/N—Returns Y when the condition is true or N when the condition is false.

  • 1/0—Returns 1 when the condition is true or 0 when the condition is false.

  • TRUE/FALSE—Returns True when the condition is true or False when the condition is false.

  • YES/NO—Returns Yes when the condition is true or No when the condition is false.

Text variables

Note that certain text variables do not have output formatting options. If you have questions about the text variables available in your communication, you should talk to the person who designed the communication.

  • Use variable output format—Uses the output format specified by the person who designed the communication.

  • Keep blanks—Preserves spaces as they appear in the value.

  • Trim blanks—Removes all of the spaces found before the first character and after the last character of the text.

  • Lower, keep blanks—Converts the text to all lowercase characters and preserves all of the spaces as they appear in the text.

  • Lower + exceptions, keep blanks—Converts the text to all lowercase characters, except those words found in an exceptions file, and preserves all of the spaces as they appear in the text.

  • Upper, keep blanks—Converts the text to all uppercase characters and preserve all of the spaces as they appear in the text.

  • Upper + exceptions, keep blanks—Converts the text to all uppercase characters, except those words found in an exceptions file, and preserves all of the spaces as they appear in the text.

  • InitCap, keep blanks—Converts the text so that each word begins with an uppercase character and preserves all of the spaces as they appear in the text.

  • InitCap, trim blanks—Converts the text so that each word begins with an uppercase character and removes all of the spaces found before the first character and after the last character of the text.

  • InitCap + exceptions, keep blanks—Converts the text so that each word begins with an uppercase character, except those words found in an exceptions file, and preserves all of the spaces as they appear in the text.

  • Sentence, keep blanks—Converts the text so that only the first word of a sentence begins with an uppercase character and preserves all of the spaces as they appear in the text.

  • Sentence + exceptions, keep blanks—Converts the text so that only the first word of a sentence begins with an uppercase character, except those words found in an exceptions file, and preserves all of the spaces as they appear in the text.

  • Exceptions only, keep blanks—Converts the words found in the exception file to match their format in the file and preserves all of the spaces as they appear in the text.

  • Phone number (999) 999-9999—Converts text values to a US-style telephone number with the area code in parentheses and a hyphen between the first three and last four digits of the number.

  • Phone number (999) 999 9999—Converts text values to a US-style telephone number with the area code in parentheses and a space between the first three and last four digits of the number.

  • Phone number 999 999-9999—Converts text values to a US-style telephone number with the area code without parentheses and a hyphen between the first three and last four digits of the number.

  • Phone number 999 999 9999—Converts text values to a US-style telephone number with the area code without parentheses and a space between the first three and last four digits of the number.

Date variables

  • Use variable output format—Uses the output format specified by the person who designed the communication.

  • Multiple date formats—Content Author provides a long list of options for formatting date values (examples: mm/dd/yy, mm-dd-yyyy, mm/yyyy).

Currency variables

  • Use Exstream settings—Uses the output format specified by the person who designed the communication.

  • General number—Uses simple numbers without any special formatting (such as currency symbols).

  • Fixed decimal—Adds zeros to pad the number of digits after the decimal point.

  • Fixed decimal with currency—Inserts a fixed decimal point and the proper currency symbol for the locale.

  • Fixed or integer with currency—Inserts a whole number if the number does not have a fraction and inserts the proper currency symbol for the locale. If the number has a fraction, the number is formatted as a floating number with the proper currency symbol for the locale in the final customer communication.

  • Significant decimal—Inserts the least number of decimal values possible after the whole number.

  • Fixed or integer—Inserts a whole number if the number does not have a fraction. If the number has a fraction, the number appears as a floating number in the final customer communication.

Number variables (with decimals)

These options apply to variable values that allow decimals. If you have questions about the number variables that are available, you should talk to the person who designed the communication.

  • Use variable output format—Uses the output format specified by the person who designed the communication.

  • General number—Uses simple numbers without any special formatting (such as commas).

  • Percentage—Adds a percent (%) symbol to the end of the number.

  • Percentage x 100—Adds a percent (%) symbol at the end of the number and multiplies the value by 100. This option lets you format fractions as percents. For example, .75 appears as 75%

  • Fixed decimal—Adds zeros to pad the number of digits after the decimal point.

  • Fixed decimal with currency—Inserts a fixed decimal point and the proper currency symbol for the locale.

  • Fixed or integer with currency—Inserts a whole number if the number does not have a fraction and inserts the proper currency symbol for the locale. If the number has a fraction, the number is formatted as a floating number with the proper currency symbol for the locale in the final customer communication.

  • Significant decimal—Inserts the least number of decimal values possible after the whole number.

  • Fixed or integer—Inserts a whole number if the number does not have a fraction. If the number has a fraction, the number appears as a floating number in the final customer communication.

  • Absolute value—Converts the values to the absolute value. This option lets you produce a negative number without a negative sign. For example, both 1 and -1 appear as 1.

Number variables (whole numbers)

These options apply to variable values that require whole numbers. If you have questions about the number variables that are available, you should talk to the person who designed the communication.

  • Use variable output format—Uses the output format specified by the person who designed the communication.

  • General number—Uses simple numbers without any special formatting (such as commas).

  • Percentage—Adds a percent (%) symbol to the end of the number.

  • Text upper (ONE, TWO)—Converts integer values to English uppercase words. For example, 1 appears as ONE.

  • Text mixed (One, Two)—Converts integer values to English mixed-case words. For example, 1 appears as One.

  • Text lower (one, two)—Converts integer values to English lowercase words. For example, 1 appears as one.

  • Alpha upper (A, B, C)—Converts integer values to uppercase alpha characters. Each integer is associated with a letter of the alphabet, all uppercase. If you use a number greater than 26, the alphabet starts over. For example, 1 appears as A and 27 appears as AA.

  • Alpha lower (a, b, c)—Converts integer values to lowercase alpha characters. Each integer is associated with a letter of the alphabet, all lowercase. If you use a number greater than 26, the alphabet starts over. For example, 1 appears as a and 27 appears as aa.

  • Roman upper (I, II, III)—Converts integer values to uppercase Roman numerals.

  • Roman lower (i, ii, iii)—Converts integer values to lowercase Roman numerals.

  • Fixed decimal with currency—Inserts a fixed decimal point and the proper currency symbol for the locale.

  • Fixed or integer with currency—Inserts a whole number if the number does not have a fraction and inserts the proper currency symbol for the locale. If the number has a fraction, the engine formats it as a floating number and inserts the proper currency symbol for the locale.

  • Absolute value—Converts the values to the absolute value. This option lets you produce a negative number without a negative sign. For example, both 1 and -1 appear as 1.

Multi-value variables

A multi-value variable contains a list of values that are all the same type, for example, a list of text strings or a list of dates. The properties on the card for a multi-value variable, such as Variable type and Output format, apply to all values in the variable.

Tip: See the preceding sections in this topic for output format options by variable type.

The card for a multi-value variable also includes an Array element to use list. You can select one of the following options on this list to specify how the values in a multi-value variable appear in customer output.

  • Automatic—Automatically imports the first value in the list of values.

  • Specified—Imports the specified value based on the number that you enter in the box below this option. For example, enter 2 to specify the second value in the list.

  • All, blanks—Imports all values, separated by blank spaces.

  • All, line breaks—Imports all values, separated by line breaks.

  • All, paragraphs—Imports all values, separated by blank lines.

  • All, print as list—Imports all values as a delimiter-separated list. The person who designs your communication determines which delimiter this setting uses.
