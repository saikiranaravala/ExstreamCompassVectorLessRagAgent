Viewing and managing versions of content

The Versions view lets you see and manage all versions of a message or clause. For example, you can use the Versions view to see differences between multiple versions and remove obsolete versions. You can also use the Versions view to see the properties or workflow history for any version of a message or clause.

In addition, if all versions of a message or clause are approved, the Versions view lets you promote any older version to the latest draft version.

Note: You can only edit the latest draft version of a message or clause.

To open the Versions view for a message or clause:

  1. In Communications view, in the PDF or HTML preview, click in the content for a message or clause.

  2. On the Applied Content card, in the content list for the selected frame, select the Change workflow state button next to the selected message or clause.

  3. In the menu that opens below the button, select See all versions. The Versions view for the selected message or clause opens.

To view and select any version of a message or clause:

  1. Open the Versions view for the message or clause.

  2. View and select a version of the message or clause using any of these methods:

    • Scroll down to the version if necessary and click inside the content canvas or footer bar for that version.

    • On the toolbar, use the navigation buttons, Latest , Next newer , Next older , and Oldest , to move to and select the version.

    • On your keyboard, press the arrow keys, HOME, UP ARROW, DOWN ARROW, and END, to move to and select the version.

  3. Verify that the footer bar for the selected version is dark blue. If you do not see the footer bar, scroll down. If the footer bar is gray, try selecting the version again.

    If the footer bar has a red line at the top, you can view that version but you cannot change its content or properties. If you are allowed to change the selected version, the red line disappears after you change the workflow state for that version to draft. To change an older version, promote it to the latest draft version first.

To remove a version of a message or clause:

Important: You can remove a version of a message or clause only if it is unlocked and if it is not used in a different communication, such as an older version of an approved communication.

  1. In the Versions view for the message or clause, select the version that you want to remove.

  2. Select the Remove version button to the right of the canvas for the selected version.

  3. In the confirmation dialog box, select Discard to permanently remove the selected version, or click Cancel.

    Caution: You cannot undo the Discard action or restore the removed version later.

  4. If an error message tells you that you cannot remove the selected version, ask the person who designed the communication to remove the version for you.

To view the properties or workflow history for any version of a clause or message:

In the Versions view for a message or clause, select the version whose properties or workflow history you want to see.

  • To see properties for the selected version, click inside canvas for that version. The properties for the version appear in the Properties panel on the right.

  • To see the workflow history for the selected version, in the footer bar for that version, select the Workflow history button . In the Advanced details for <content_name> dialog box, on the Workflow tab, the list of workflow events for each version of the message or clause appears, listed by version number. Click Done to close the dialog box and return to the Versions view.

Promoting an older version of a message or clause to the latest draft version

Promoting an older version creates a new latest version at the top of the Versions view. The new latest version is the same as the older promoted version, except it has the latest version number.

If an older version has tracked revisions that are not accepted, after you promote the older version, the revisions are still highlighted in the latest version.

After you promote an older version, you can compare the new latest version to the latest approved version using Content Author's version comparison tools.

To promote an older version of a message or clause to the latest draft version:

  1. Open the Versions view for the message or clause and verify that the Change workflow state button to the right of each version is in the approved state .

    Important: You can only promote an older version to the latest draft version if all versions of the message or clause are in the approved state.

  2. Select the older version that you want to promote.

  3. To the right of the content canvas for the older version, select the Change workflow state button for the older version, and then select the Draft button .

    The older version becomes the latest version at the top of the Versions view. The new latest version is in the draft workflow state and has the next version number.
