TREAT_PREFIXES_AND_WHOLEWORDS_THE_SAME

In case conversion exception files, if you include ordinals or other strings that begin with numeric characters (for example, 1st, 2nd, 3rd, or 5678XY), you can use the TREAT_PREFIXES_AND_WHOLEWORDS_THE_SAME engine switch to specify how the engine reads and applies exceptions to prefixes and whole words that begin with numerals. By default, the TREAT_PREFIXES_AND_WHOLEWORDS_THE_SAME switch is set to YES, and the engine ignores numbers when applying both whole words and prefixes from the case conversion exception file. When you set the TREAT_PREFIXES_AND_WHOLEWORDS_THE_SAME switch to NO, the engine still ignores numbers when applying whole-word exceptions, but numbers are included when applying prefix exceptions.

This switch is affected by the CASE_EXCEPTION_SKIP_NUMBERS engine switch, which by default is also set to YES. When the CASE_EXCEPTION_SKIP_NUMBERS switch is set to YES or is not specified, then words are processed for case exceptions between white space, parentheses, or numbers; numbers are not included in the case exception processing. When the CASE_EXCEPTION_SKIP_NUMBERS switch is set to NO, then words are processed for case exceptions only between white space and parentheses, and numbers are processed as part of the words for case exceptions. The following table shows how the two switches determine case conversion behavior.

Value of CASE_EXCEPTION_SKIP_NUMBERS Value of TREAT_PREFIXES_AND_WHOLEWORDS_THE_SAME Behavior
YES (default) YES (default) Numbers are ignored when applying both whole-word and prefix exceptions.
YES (default) NO

Version 8.0.320 and earlier:

Numbers are ignored when applying prefix exceptions, but are considered part of the word when applying whole-word exceptions.

Version 8.0.321 and later:

Numbers are ignored when applying both whole-word and prefix exceptions. (Setting the TREAT_PREFIXES_AND_WHOLEWORDS_THE_SAME switch to NO might have an effect on certain words that are otherwise processed incorrectly because of their sequence in the string.)

NO YES (default) Numbers are considered part of the word when applying both whole-word and prefix exceptions.
NO NO

Versions 8.0.334 and earlier:

Numbers are considered part of the word when applying both whole-word and prefix exceptions. (The TREAT_PREFIXES_AND_WHOLEWORDS_THE_SAME switch has no effect.)

Versions 8.0.335 and later maintenance releases, 8.6.109 and later maintenance releases, and 9.0.102 and later:

Numbers are ignored when applying both whole-word and prefix exceptions. (The CASE_EXCEPTION_SKIP_NUMBERS switch has no effect. Setting the TREAT_PREFIXES_AND_WHOLEWORDS_THE_SAME switch to NO might have an effect on certain words that are otherwise processed incorrectly because of their sequence in the string.)

Syntax 

-TREAT_PREFIXES_AND_WHOLEWORDS_THE_SAME=<value>

The only argument for this switch is YES or NO. Refer to the table in the switch description for the behavior that occurs for each setting.

Supported in

  • 7.0.615 and later maintenance releases
  • 8.0.203
  • 8.0.301 and later versions
