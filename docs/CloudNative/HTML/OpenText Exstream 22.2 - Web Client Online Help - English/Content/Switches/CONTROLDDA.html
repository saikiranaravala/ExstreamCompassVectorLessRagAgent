CONTROLDDA

Note: This switch is not supported when using the Exstream engine in a containerized Exstream deployment.

Use the CONTROLDDA switch to initiate a dedicated command-only channel so on-demand commands can be passed directly to the engine for processing instead of using a driver file connector.

Syntax

-CONTROLDDA=<language>,<module>,<function>,[<buffer>],[<parameters>]
Arguments
Arguments Required Supported values

language

Yes

The language that is used to create the module:

  • ASM

  • COBOL

  • DLL (for C and C++)

  • PL1

module

Yes

The fully qualified name of the file that contains the module

function

Yes

The function name in the DLL or in the routine

buffer

No

The size of the buffer (in bytes), which must accommodate the size of the output

parameters

No

Additional parameters specific to the module

Example

-CONTROLDDA=DLL,C:\path\to\MyDLL.dll,MyFunction,5000,OpenParameter
