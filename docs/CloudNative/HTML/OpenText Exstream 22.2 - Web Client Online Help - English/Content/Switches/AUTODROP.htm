AUTODROP

The AUTODROP switch automatically drops all tables and associated objects for the specified role (design or tracking) with no interaction, but does not drop the database instance.

Requirements

If you use the AUTODROP switch, you must also use the following switches:

  • QUERYPATH
  • One of the following: ACCESSDB or DSN

Syntax

-AUTODROP=<databaseType>,[<databaseNameDB2>],[<tablespaceOption>],[<bufferpoolOption>]
Argument Required Supported values

databaseType

Yes

  • DESIGN

  • TRACKING

databaseNameDB2 No

File name of the DB2 database.

The structure is dropped from the default database unless you use this argument to specify another database name. This argument is required only for DB2 databases on z/OS platforms.
tablespaceOption No

DROPTABLESPACES

For DB2 databases on z/OS platforms, use this argument to drop the default Exstream tablespaces. The structure is dropped from the default database unless you use the databaseNameDB2 argument to specify another database name.

bufferpoolOption No

DROPBUFFERPOOL

For DB2 databases on non-z/OS platforms, use this argument to drop the default Exstream buffer pool.

Example

-AUTODROP=TRACKING
-QUERYPATH=C:\Temp
-DSN=MyDSN,,MyUserName,MyPassword
