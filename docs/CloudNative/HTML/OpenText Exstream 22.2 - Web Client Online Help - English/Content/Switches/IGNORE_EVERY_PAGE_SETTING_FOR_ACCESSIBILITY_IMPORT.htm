IGNORE_EVERY_PAGE_SETTING_FOR_ACCESSIBILITY_IMPORT

In earlier versions of Exstream, the Used resources at page level option in the Resource inclusion list in the PDF output object properties was not applied correctly at engine run time. For applications with only one output queue, no resources at the page level were included, which resulted in duplicate header ID values in the PDF output. For applications with multiple output queues, page-level resources were always included, regardless of which option was selected in the Resource inclusion list.

In later versions, the default behavior has been changed. When you select the Used resources at page level option on the PDF output object, page-level resources are correctly applied. This results in unique table header ID values in the output, regardless of the number of output queues associated with the application.

For backward compatibility after you upgrade, use the IGNORE_EVERY_PAGE_SETTING_FOR_ACCESSIBILITY_IMPORT switch if your design meets the following conditions:

  • The application has only one associated output queue.
  • The design imports one or more PDF documents that contain accessibility information, including table header IDs, at run time.
  • You do not want unique table header IDs applied to tables in the PDF output.

Important: If you want to make sure that unique table header IDs are applied to tables in the PDF output after upgrading, repackage the application and run the engine. This switch should be used only if you want to revert to the previous default behavior.

Syntax

-IGNORE_EVERY_PAGE_SETTING_FOR_ACCESSIBILITY_IMPORT

This switch does not have any arguments.

Supported in

  • 9.5.307 and later maintenance releases
  • 16.2.2 and later maintenance releases
  • 16.3.1 and later maintenance releases
  • 16.4.0 and later versions
