ENABLE_DOCX_METADATA_TYPES

Use the ENABLE_DOCX_METADATA_TYPES engine switch to control headers, deleteIDs, and margins in DOCX output more closely, and to provide a better editing experience in the final DOCX document. When you use the ENABLE_DOCX_METADATA_TYPES engine switch in the command line or control file during production, you can use metadata to explicitly define custom headers, custom deleteIDs, and uniform margins in DOCX output. Optionally, you can also define different headers and deleteIDs for first, odd, and even pages in the DOCX document.

If you want more control of headers, deleteIDs, and margins in DOCX output, and you want to provide a better editing experience in the final DOCX document, you can use the ENABLE_DOCX_METADATA_TYPES engine switch. When you use this engine switch in the command line or control file during production, you can use metadata to explicitly define custom headers, custom deleteIDs, and uniform margins in DOCX output. Optionally, you can also define different headers and deleteIDs for first, odd, and even pages in the DOCX document.

If you want the engine to produce DOCX text boxes instead of DOCX frames in the output, you can use the FORCE_NO_DOCX_TEXT_FRAMES engine switch in the command line or control file during production.

For more information about DOCX output and designing applications in Design and Production, see OpenText Exstream Design and Production Online Help.

When you produce DOCX output without using the ENABLE_DOCX_METADATA_TYPES engine switch, the Exstream engine automatically determines a header, deleteID, and margins. The engine handles headers, deleteIDs, and margins as follows:

  • The engine might create a single header from the first entry in each design page. Repeating headers in the design are not converted to headers in DOCX output, and headers apply to individual pages in the DOCX document.
  • The engine might create a single deleteID from the last entry in each design page. Repeating deleteIDs in the design are not converted to headers in DOCX output, and deleteIDs apply to individual pages in the DOCX document.
  • The engine determines margins using the boundaries of content flow frames. Margins might vary throughout the DOCX document, depending on the design of each page in Exstream.
Using Metadata to Define Custom Headers and Footers

When you use the ENABLE_DOCX_METADATA_TYPES engine switch, you can apply special metadata to design objects to specify that they should be part of a custom header or deleteID in DOCX output. The metadata objects that you use to define custom headers and deleteIDs for DOCX output do not have values.

Keep in mind the following considerations when you use metadata to define custom headers and deleteIDs:

  • You must set the output queue that you are using to produce the output to break before each customer. On the Breaks tab of the output queue properties, in the Customers box, enter 1.

  • If you are also using metadata to define margins, then you should place the objects that you want to appear in the header outside of the specified top margin. Likewise, you should place the objects that you want to appear in the deleteID outside of the specified bottom margin. Otherwise, because design objects are anchored in the DOCX output where they are located on the design page, headers and deleteIDs might overlap other text and objects in the output. Setting a header and deleteID distance in the design does not affect the placement of objects in the header and deleteID in DOCX output.
  • The objects to which you add header or deleteID metadata must be on the page that appears first in the output. If the engine removes the page that contains the header or deleteID objects for a customer due to a rule, the headers or deleteIDs will not appear in the output. If any objects to which you have assigned header or deleteID metadata appear on subsequent pages in the output, the engine omits those objects from the output. If you are producing more than one document for each customer within the same application, these conditions apply across all documents. The objects to which you add header or deleteID metadata must be on the first page output from the first document.
  • When you use the ENABLE_DOCX_METADATA_TYPES engine switch, the engine ignores any design objects that are placed on flow pages (whether or not you have assigned header or deleteID metadata to them), and they will not appear in the output.
  • The engine creates a new section in the DOCX output for each separate design page (whether or not you use the ENABLE_DOCX_METADATA_TYPES engine switch). When you use the engine switch and use metadata to define headers and deleteIDs, the engine applies headers and deleteIDs to all resulting sections.
  • If you delete a section break in the output, headers and deleteIDs might be affected. For example, if you are using a different first-page header, and you delete the first section break in the document, then the first-page header is permanently deleted.
  • You cannot specify custom headers or deleteIDs in page templates. If you assign header or deleteID metadata to objects in a page template, the engine omits those objects from the output.
  • You cannot specify custom headers and deleteIDs if any of the following conditions are true:
    • You are using language layers.
    • You are using the Application Consolidator to combine output from multiple applications.
    • You are using two-step processing.
Defining a Default Header or Footer

You can define a default header or deleteID that appears on each page of the DOCX output. If you define only the default header or deleteID, the header or deleteID that you define is the same for all pages.

Keep in mind that because DOCX specifies a default header and an odd-page header the same way, you use the same metadata object to define a default header or an odd-page header (when defining different headers for odd and even pages). If you do not specify any objects for even-page headers, the odd-page header becomes the default header and applies to all pages.

To define a default header or deleteID:

  1. In Design Manager, create one of the following metadata objects, depending on whether you want to define a header or deleteID:

    Option Description
    Define a default header in DOCX output

    Create a metadata object named DocX-DefaultHeader.
    Define a default deleteID in DOCX output Create a metadata object named DocX-DefaultFooter.
  2. In the Property Panel for the new metadata object, select No value, and then save the metadata object.
  3. In Designer, on the design page that will appear first in the output, add the corresponding metadata to each object that you want to appear in the default header or deleteID.

You can define a different header or deleteID that will appear on the first page of the DOCX output. When you define a first-page header or deleteID, the Different First Page option is selected for the header and deleteID when you open the output in Microsoft Word.

If you want to omit a default or odd-page header or deleteID from the first page, you can create an empty text box with no frame and define it as the first-page header.

The option to define a different first-page header or deleteID applies to both the header and deleteID. That is, if you define either a first-page header or a first-page deleteID, then both the header and deleteID are different for the first page. If you only define one, then the other will be left blank for the first page.

To define a different header or deleteID for the first page:

  1. In Design Manager, create one of the following metadata objects, depending on whether you want to define a header or deleteID:

    Option Description
    Define a first-page header in DOCX output

    Create a metadata object named DocX-FirstHeader.
    Define a first-page deleteID in DOCX output Create a metadata object named DocX-FirstFooter.
  2. In the Property Panel for the new metadata object, select No value, and then save the metadata object.
  3. In Designer, on the design page that will appear first in the output, add the corresponding metadata to each object that you want to appear in the first-page header or deleteID.
Defining Different Headers or Footers for Odd and Even Pages

You can define different headers or deleteIDs that will appear on odd and even pages in DOCX output. Keep in mind that because DOCX specifies a default header and an odd-page header the same way, you use the same metadata object to define a default header or an odd-page header (when defining different headers for odd and even pages). When you specify an object for an even-page header, the Different Odd & Even Pages option is selected for the header when you open the output in Microsoft Word. The specified default header appears on odd pages, and the specified even-page header appears on even pages. Additionally, the default or odd-page header appears on the first page, unless a first-page header is defined.

If you want to define a header or deleteID for only even pages or only odd pages, you can create an empty text box with no frame and define it as the header or deleteID that you want to be blank.

The option to define different odd- and even-page headers or deleteIDs applies to both headers and deleteIDs. That is, whenever you define either an even-page header or an even-page deleteID, both the header and deleteID are different for odd and even pages. If you define only one for even pages, the other will be left blank on even pages.

To define different headers or deleteIDs for odd and even pages:

  1. In Design Manager, create one of the following metadata objects to define an odd-page header, depending on whether you want to define a header or deleteID:

    Option Description
    Define an odd-page header in DOCX output

    Create a metadata object named DocX-DefaultHeader.
    Define an odd-page deleteID in DOCX output Create a metadata object named DocX-DefaultFooter.
  2. In the Property Panel for the new metadata object, select the No value check box, and then save the metadata object.

  3. Create one of the following metadata objects to define an even-page header, depending on whether you want to define a header or deleteID:

    Option Description
    Define an even-page header in DOCX output

    Create a metadata object named DocX-EvenHeader.
    Define an even-page deleteID in DOCX output Create a metadata object named DocX-EvenFooter.
  4. In the Property Panel for the new metadata object, select the No value check box, and then save the metadata object.
  5. In Designer, on the design page that will appear first in the output, complete the following steps to add the appropriate metadata to the desired objects:
    1. Add the corresponding metadata that you created in step 1 to each object that you want to appear in the odd-page header or deleteID.
    2. Add the corresponding metadata that you created in step 3 to each object that you want to appear in the even-page header or deleteID.

When you use the ENABLE_DOCX_METADATA_TYPES engine switch, you can apply special metadata to a page to specify uniform margins for all pages in DOCX output.

Keep in mind the following considerations when you use metadata to specify uniform margins:

  • You must add metadata to the page that appears first in the output. If the engine removes the page to which the metadata is applied for a customer due to a rule, the specified margins will not be set in the output. If you are producing more than one document for each customer within the same application, this condition applies across all documents. You must add the metadata to first-page output from the first document. The engine ignores margin metadata applied to subsequent pages in the output.

  • The specified margins apply to all pages in the output.

  • Any flowing content that extends outside of the specified margins in the design will be fully contained within the margins in the output. Any other objects that extend outside the specified margins will remain at their original locations in the output.
  • The header and deleteID distance settings do not affect custom headers and deleteIDs that are defined for DOCX output. Design objects that you include in a header or deleteID using metadata are anchored in the DOCX output at the location where they are located on the design page.
  • If you specify margins incorrectly (for example, negative values, or overlapping side margins), the engine adjusts the margins to default values and issues an error message.
  • You cannot specify uniform margins if either of the following conditions are true:
    • You are using the Application Consolidator to combine output from multiple applications.
    • You are using two-step processing.

To define a uniform margin setting:

  1. In Design Manager, create one of the following metadata objects, depending on which margin that you want to specify:

    Option Description
    Specify the top margin Create a metadata object named DocX-TopMargin.
    Specify the bottom margin Create a metadata object named DocX-BottomMargin.
    Specify the left margin Create a metadata object named DocX-LeftMargin.
    Specify the right margin Create a metadata object named DocX-RightMargin.
    Specify the header distance setting for the DOCX output Create a metadata object named DocX-HeaderDistance.
    Specify the deleteID distance setting for the DOCX output Create a metadata object named DocX-FooterDistance.
  2. In the Property Panel for the new metadata object, complete the following steps to specify the measurement of the margin:
    1. Clear the No value check box.
    2. In the Default value area, enter a value or select a variable that specifies the measurement of the margin. The value entered or the value of the selected variable specifies the measurement in inches by default, or you can include units to specify inches or centimeters. For example, 1.5 or 1.5in specifies 1.5 inches, and 4cm specifies 4 centimeters.
    3. Save the metadata object.
  3. In Design Manager, add the corresponding metadata to the page object that will appear first in the output.

Syntax

-ENABLE_DOCX_METADATA_TYPES

This switch does not have any arguments.
