PKGOBJECT

Use teh PKGOBJECT switch to package a non-design or environmental object by type and name or object identifier into a subpackage.

When you specify this packaging switch, Packager.exe first attempts to identify the object name. If an object name is not identified, Packager.exe identifies the object identification number (OI).

Syntax

-PKGOBJECT=<objectName>

The argument for this switch is the object type and name. The argument must be specified in one of the following formats:

<objectType>:<objectName>[:<folderName>]
<objectType>:<objectOI>[:<folderName>]

Example

-PKGOBJECT=Barcode:MyBarcode
-PKGOBJECT=Barcode:23:Subfolder1
