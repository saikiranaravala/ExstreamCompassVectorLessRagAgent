DBMAINT_OPTIONS

The DBMAINT_OPTIONS switch lets you specify which options from the Database Maintenance dialog box are selected. This switch is optional. If you do not use the DBMAINT_OPTIONS switch, database maintenance uses the same default behavior as the default behavior when running from the Design Manager interface.

The default behavior for database maintenance is to run with the following options from the Database Maintenance dialog box enabled:

  • Check for unreferenced embedded objects

  • Check serialized LOB integrity

  • Check for unsynchronized variable references

  • Check for unsynchronized rule references

  • Check for orphaned rules > Convert orphaned rules to named rules

  • Check database referential integrity

  • Check for unused duplicate fonts

When you use the DBMAINT_OPTIONS switch, only the options specified in the arguments are enabled during database maintenance. Any option that is not specified in the switch is disabled during database maintenance.

Syntax

-DBMAINT_OPTIONS=<options>
Argument Required Supported values

options

Yes

Use one of more of the following arguments, separated by commas without spaces:

  • CHECK_REF_INTEGRITY—This argument corresponds to the Check database referential integrity check box.
  • CHECK_SERIALIZED_LOB— This argument corresponds to the Check serialized LOB integrity check box.
  • CHECK_UNREF_EMBEDDED_OBJS— This argument corresponds to the Check for unreferenced embedded objects check box.
  • CHECK_UNSYNC_RULE_REFS— This argument corresponds to the Check for unsynchronized rule references check box.
  • CHECK_UNSYNC_VAR_REFS— This argument corresponds to the Check for unsynchronized variable references check box. If you specify the CHECK_UNSYNC_VAR_REFS argument, you must also specify the CHECK_SERIALIZED_LOB argument.
  • CONVERT_ORPHANED_RULES— This argument corresponds to the Check for orphaned rules check box and the Convert orphaned rules to named rules option. If you specify the CONVERT_ORPHANED_RULES argument, you must also specify the CHECK_SERIALIZED_LOB argument. This argument cannot be used with the DELETE_ORPHANED_RULES argument.
  • DELETE_ORPHANED_RULES— This argument corresponds to the Check for orphaned rules check box and the Delete orphaned rules option. If you specify the DELETE_ORPHANED_RULES argument, you must also specify the CHECK_SERIALIZED_LOB argument.This argument cannot be used with the CONVERT_ORPHANED_RULES argument.

Example

-DBMAINT_OPTIONS=CHECK_UNREF_EMBEDDED_OBJS,CHECK_SERIALIZED_LOB,CHECK_UNSYNC_RULE_REFS
