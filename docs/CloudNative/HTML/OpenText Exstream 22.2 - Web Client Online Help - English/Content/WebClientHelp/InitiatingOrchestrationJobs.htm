Initiating orchestration jobs

After you publish your flow model and flow context, you are ready to initiate the production engine run as defined in the published flow resources. Depending on the type of input channel that you are using, and the type of communication being fulfilled, you can initiate the engine run in the following ways:

Using a directory input channel

When you use a directory input channel, you simply place your input file in the specified input directory that is defined in the flow context. This initiates new orchestration jobs with flow instances of any published flow models that use the input channel. Keep in mind, that this triggers orchestration jobs only if the input file name matches the file name pattern that is defined for that directory channel.

For more information about defining directory input channels, see "Directory channel" in OpenText Exstream Communications Orchestrator User Guide.

Using an HTTP input channel

When you use an HTTP input channel, you use an HTTP request to trigger the input service to send data to the HTTP channel.

Any application that posts data to an HTTP channel must be able to access the tenant where this channel is running. An OTDS authentication token (OAuth2) and a cookie for preventing cross-site request forgery attacks (csrfToken) are required to send this request. Both the OAuth2 token and the csrfToken must be included in the requests from the application, and the user in the OAuth2 token must have access to the domain to which they want to post the data.

When a request is made, the data in the request body is sent as input to the HTTP input channel that is specified in the request URI, and any exstream-variable headers that are defined in the request will be available as variables in the flow model.

For more information about defining HTTP input channels, see "HTTP(S) channel" in OpenText Exstream Communications Orchestrator User Guide.
