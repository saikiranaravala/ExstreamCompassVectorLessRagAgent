Setting up output queues

You can use the asset library panel in the Communication asset library view to add output queue objects to the asset library for use with a communication. Output queues function as containers for production settings, including those for output drivers.

You can add multiple output queues to a communication set to produce output with a variety of high-volume settings and output driver settings, including a mix of print and electronic drivers, while running the engine only once in Exstream. When a communication set has multiple output queues, you can designate one to be the primary queue.

Creating an output queue object

  1. In the Delivery panel, click the Add button, then click Output queue in the selection list.

  2. Specify a name, and optionally, a description for the output queue.

  3. In the Output list, select the output driver for this queue from the list of available outputs.

  4. Click Create.

Updating output queue properties

After you create an output queue, you can use the Properties panel to view and modify the properties.

On the General tab, you can update the output queue name, the description, and the output object associated with the output queue.

You can use the options on the Breaks tab to divide output into multiple files (sometimes called document subsets). You can also use breaks for archival purposes. For example, you can create a separate output file for each customer, or you can separate the output into files of 5000 pages each.

Note: Break options are unavailable for output queues that use Messaging as the output type.

The following break options are available:

  • Break options—Indicates where to place a convenience break in the output. Select from the following options:

    • None—No breaks added.

    • Customers—Adds breaks after the specified number of customers.

    • Documents—Adds breaks after the specified number of documents.

    • Designs—Adds breaks after the specified number of designs.

  • Count—Specifies the number of customers, documents, or designs after which to include a break. The default value is 1.

  • Can split customers—Allows documents or designs for a customer to be split into multiple output files. By default, this check box is not selected.

Note: Output queues that use HTML or Empower as the output type automatically have the Break options set to Customers and the Count set to 1. These are fixed settings.
