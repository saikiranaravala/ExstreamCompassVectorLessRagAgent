Enabling Assisted Authoring for a communication

To ensure that the content within communications is as effective as possible, Exstream offers a suite of Assisted Authoring tools that provide text analysis to measure readability, sentiment and tone, and grammar. The Assisted Authoring tools represent an integration with OpenText™ Experience AI Services. The Assisted Authoring tools, which are configured for Content Author or Communications Designer as a whole, are enabled for individual communications in Design asset library view.

Important: Before you can enable Assisted Authoring tools for communications, you must separately deploy Experience AI Services into your Kubernetes cluster, and then configure Experience AI Services for the tenant. For information about deploying and configuring Experience AI Services, see OpenText Experience AI Services Configuration Guide, available on My Support.

To enable Assisted Authoring for a Content Author or Communications Designer communication:

  1. Sign in to the Exstream web client as a tenant administrator user at the following URL:

    https://<host>:<optionalPort>/design?tenant=<tenantName>

  2. In Design asset library view, click the Author or Designer panel, and then select a communication from the list.

  3. In the Properties panel, on the Intelligence tab (which is available only if Experience AI Services has been set up on the Configurations panel for Content Author or Communications Designer in Tenant management view), use the Content language list to set the options available for each supported language. Note that the following languages are available for Assisted Authoring, but not all analysis types are available for each language:

    • Dutch

    • English (American)

    • English (Australian)

    • English (British)

    • French

    • German

    • Portuguese (Brazilian)

    • Portuguese

    • Spanish

    Content Author and Communications Designer use the language that is set for the spelling dictionary to determine whether Assisted Authoring tools are available for users. However, you must configure Assisted Authoring options on the Intelligence tab for each supported language before users will see Assisted Authoring options in Content Author or Communications Designer. For example, if a user has the spelling dictionary set to French, Assisted Authoring tools will not be available in the communication until the Assisted Authoring settings are configured for French on the Intelligence tab.

    If an option is not available for a particular language, then you will not be able to configure that option in the UI. If you have questions about which services are available for each language, speak to your Experience AI Services administrator.

    Important: If you add a German dictionary to your deployment, then you must ensure that the language code for the dictionary uses the de_DE language code used by Assisted Authoring. If you use a different language code, then Assisted Authoring will not work for German content.

  4. Select the following check boxes as needed to enable Assisted Authoring tools for the communication:

    Readability analysis

    The readability analysis tool gives business users statistics based on one of two different reading scales—the Flesch Reading Ease scale, and the Flesch-Kincaid Grade Level index. When you enable readability analysis, you can select a service provider as well as the readability scale that you want to use.

    To configure readability analysis:

    1. After you have selected the Readability analysis check box, in the Service provider list, select a provider:

      • OpenText—Uses OpenText Magellan to provide readability analysis.

      • Google—Uses Google Cloud Platform to provide readability analysis.

    2. In the Readability scale list, select a readability scale:

      • Flesch Reading Ease—The Flesch Reading Ease scale shows statistics based on the average number of words per sentence and the average number of syllables per 100 words.

      • Flesch-Kincaid Grade Level—The Flesch-Kincaid Grade Level readability index is also based on the number of syllables per word and the number of words per sentence. The resulting number indicates the approximate level of education, based on US guidelines, that is required to read and understand the text. This tool is available only when the spelling dictionary in Content Author is set to English (American).

    3. Based on the readability scale that you selected, set parameters in the Target range section and in the Acceptable range section:

      • Flesch Reading Ease scale:

        Value Meaning
        90-100 Very easy
        80-90 Easy
        70-80 Fairly easy
        60-70 Standard
        50-60 Fairly difficult
        30-50 Difficult
        0-30 Very difficult

      • Flesch-Kincaid Grade Level index: Select the approximate level of education, based on US guidelines, that is required to read and understand the text.

    When users in Content Author or Communications Designer open the Assisted Authoring tools, they will receive an overall score that compares their content against the parameters that you set. Additionally, when they point to the numerical value for the overall readability of their content, they will see a tooltip that explains the scores for the scale that is being used.

    Sentiment and tone analysis

    Sentiment and tone analysis uses artificial intelligence provided by either OpenText Magellan or Google, depending on the option you select, to determine the overall attitude of content—positive, negative, or neutral—and is represented by a numerical score that ranges from -100 to 100. Keep in mind that the numerical values vary by provider, and they can be configured within Experience AI Services. For example, here are the recommended ranges for sentiment and tone when using OpenText Magellan as the provider:

    • Positive: A score greater than 33

    • Neutral: A score between -33 and 33

    • Negative: A score less than -33

    For information about configuring settings in Experience AI Services, see OpenText Experience AI Services Configuration Guide.

    Magellan also uses artificial intelligence to determine whether each sentence is fact or opinion. The number shown represents a blended score of all sentences that make up the content. Business users can use the navigation arrows on the toolbar to validate each sentence individually to ensure that sentiment and tone are consistent.

    Grammar analysis

    The grammar tool behaves much like traditional grammar checkers—it identifies issues and then makes individual suggestions to improve the content. In the Grammar analysis section, in the Service provider list, select either LanguageTool or GrammarBot. For information about either of these service providers, consult their online documentation.

    Beside each check box, you'll see an indicator with a descriptive tooltip that shows the level of service that you have licensed for each analysis type. For information about licensing analysis types, see OpenText Experience AI Services Configuration Guide, available on My Support.

  5. Click Save.
