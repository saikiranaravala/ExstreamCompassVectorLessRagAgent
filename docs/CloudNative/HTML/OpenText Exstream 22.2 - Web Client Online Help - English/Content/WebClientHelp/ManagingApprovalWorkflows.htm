Managing approval workflows

The approval workflow is the progression of a resource from initial version to final approval. Using the standard or simple approval workflow in Exstream helps to ensure that new and updated content is properly reviewed for accuracy and conforms to company standards. The standard workflow requires authors to submit their drafts for approval. The simple workflow does not. Rather, the simple workflow skips the "Submitted for approval" workflow state and allows authors to approve their own drafts.

Keep the following in mind when approving resources:

  • All child resources must be approved before a parent resource. For a communication, this includes the communication and all of its associated resources, such as images that are used in designs and the sample files that are referenced by the communication. If you try to approve a resource that references other unapproved resource, you receive a message indicating that you must first approve all of the linked resources before you can approve that resource.

  • You can only approve a new design pack if it contains at least all the attributes and tags in approved variations of the communications that use the design pack. If you try to approve a design pack that is missing attributes or tags, a list of affected variations opens and prompts you to remove the attributes or tags before resubmitting the design pack for approval. For more information about using variations in a communication, see OpenText Exstream Communications Designer Online Help.

    Note: Alternatively, you can use Design Manager to add the missing attributes or tags to the design pack and then re-upload the design pack to DAS. For more information about uploading design packs, see OpenText Exstream Design and Production Online Help.

  • You can change the workflow state of a single resource or multiple resources at the same time. If you select multiple resources, only those whose workflow state matches that of the resource you actively select to move are changed.

    Note: The only exception to changing multiple resources is the Discard draft option; only the draft of the selected resource is discarded.

Workflow state symbols on workflow state buttons

In the design applications and the asset library views, you can select the workflow state button next to a resource to move the resource to its next workflow state. The symbol on the button indicates the current workflow state for the resource.

Moving through the approval process

As you move through the approval process, keep in mind that only resources that are in draft state can be edited. In the standard workflow only, after a resource has been submitted for approval or rejected, you must return that resource to a draft state in order to update it. Similarly, if a resource has already been approved, you must create a new draft of the resource in order to add to or update it.

To move resources through the approval process:

  1. Select one or more resources:

    • If changing the workflow state of a single resource, select the workflow state button for the selected resource.

    • If changing the workflow state of multiple resources at once:

      1. Use the check boxes next to the resource names to select the desired resources.

        Note: You can use the check box in the heading row to select all the resources on that page. You can only select resources on the active page. If you navigate to another page, any previously-selected items are cleared.

      2. Select the workflow state button for one of the resources that you want to advance to the next workflow state.

  2. Depending on the current workflow state of the resource, select one of the following actions in the table below to move the resource through the approval process.
