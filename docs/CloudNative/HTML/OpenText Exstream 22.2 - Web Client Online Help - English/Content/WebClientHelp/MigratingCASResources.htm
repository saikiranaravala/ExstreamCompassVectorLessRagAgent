Migrating CAS resources

To migrate Exstream resources from a tenant in the Exstream 16.x platform to a containerized Exstream 22.2.0 deployment, you must first create CAS export packages using either the migration tools for the Exstream platform or the transport option in Workshop. You can then use the Import option to import resources from that exported package into the destination DAS asset library.

For detailed information about exporting CAS resources, see the following documentation on My Support for your Exstream platform version:

  • OpenText Exstream Migration Tools User Guide
  • OpenText Exstream Workshop User Guide
Supported export package formats
  • You can import resources from the following types of CAS export packages:

    • Release package—Contains all the resources that are linked to a specific commit label of a Communications Builder project release, that is, the Communications Builder project, the project export file, and all of its dependent Workshop resources. Release packages are created in Workshop or using the migration tools.

      OpenText strongly recommends using this type of export package to migrate Content Author or Communications Designer communications.

      Note: Data files that are referenced in Communications Builder projects through Exstream engine switches are not included in the exported release packages. These files must be manually uploaded into the DAS asset library.

    • Resource package—Contains resources that are selected from the default Resources view or Templates view in Workshop, such as themes, templates, and images. Resource packages are created in Workshop. When you select a resource to create a resource package, the selected resource and all of its dependent resources are packaged together. However, a resource package cannot include Communications Builder projects.

      This type of export package is a convenient way to migrate Workshop resources that are related to a Communications Builder project, and were updated after you created a release package to export that project.

    Considerations when migrating CAS resources

    When you migrate CAS resources, keep the following considerations in mind: 

    • Database support—You can export resources from all databases that are supported for the Exstream platform to all databases that are supported in Exstream 22.2.0. For information about supported databases for Exstream, see the OpenText Exstream Release Notes
    • Workflow status of versioned resources—The export package maintains the workflow status of your versioned resources. During import, the workflow status is retained for all resources except for templates and design packs, which are automatically approved. OpenText recommends that you approve all resources before you create a CAS export package.
    • User permissions for locked resources—If an export package contains a resource that is locked, then the import process breaks the lock and imports the resource in an unlocked state so that users in the target tenant can edit it.
    • Unsupported resource types—Resources types that are not supported in DAS are not imported, even if they exist in the export package, and reference links for the unsupported resources are removed during import.
    Resource behavior after migration

    You can migrate only those resource types that are supported in Exstream 22.2.0. The following table lists the CAS resource types and their corresponding DAS resource types after migration:

    Behavior of migrated CAS resources
    CAS resource Corresponding DAS resource Import behavior
    Communications Builder Project - Communications Builder projects are ignored during import. To produce output from an imported communication, you must create new orchestration flows for the Exstream engine.
    Exstream Application Communication (CD) Imported as-is.
    Exstream Application Manifest - Manifests are not available as a separate resource in the DAS asset library. You can view the manifest for a package file from the package file properties.
    Exstream Application Package Package Imported as-is.
    Exstream Component Component Imported as-is.
    Exstream Data Source Data source Imported as-is.
    Exstream Document Document Imported as-is.
    Exstream Email Email Imported as-is.
    Exstream Page Page Imported as-is.
    Exstream Variable Variable Imported as-is.
    Exstream Web Design Web design Imported as-is.
    Clause Clause Imported as-is.
    Design Pack Design pack

    The import process automatically approves design packs that are not already in an approved state.
    Flow model - CAS flow models are ignored during import. However, you can create new flow models in the DAS asset library.
    Graphic Message Graphic message Imported as-is.
    Image Image Imported as-is.
    Orchestration Server Settings - Orchestration server settings are ignored during import. Orchestration settings in Exstream 22.2.0 are defined using flow models and flow contexts.
    Section Section Imported as-is.
    Sample file Sample file Imported as-is.
    Simulation - Simulations are ignored during import. Content Author and Communications Designer directly use sample files for generating previews. The engine run definitions for these previews are generated from the information in the application package and manifest.
    Template Template

    The import process imports and automatically approves only the following template types:

    • Exstream templates (based on Design Manager applications)
    • Communications Designer templates

    StoryTeller templates and templates for the template engine are ignored during import.

    Text Message Text message Imported as-is.
    Theme Communication (CA)

    Only Content Author themes are migrated. StoryBoard themes that exist in the export package are ignored during import.

    Content Author communications that were created in versions prior to 16.4 Update 2 are not supported in Exstream 22.2.0.
