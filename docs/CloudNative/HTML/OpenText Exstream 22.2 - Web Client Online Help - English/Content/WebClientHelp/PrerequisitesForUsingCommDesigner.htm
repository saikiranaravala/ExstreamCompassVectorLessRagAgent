Prerequisites for using Communications Designer

Communications Designer lets designers use resources that are set up either in Design Manager or the DAS asset library to create and customize new communications. To use Communications Designer, these resources must be set up correctly and accessible to Communications Designer users.

Using Communications Designer or Content Author with Design and Production applications relies on the use of design packs created in the application. Alternatively, templates for communications can be based on communication sets created directly in DAS using the Exstream web client. There are advantages to both methods of creating a communication. For instance, a communication set contains only DAS resources, and therefore does not require interaction with Design and Production tools. Those resources can also be used in any communication within the tenant.

On the other hand, Design Manager design packs currently offer more output queues than are available in DAS, and can support more high-volume functionality, sorting and bundling, and complex data structures (for example, reference and initialization files). Additionally, if you have a lot of existing content that you created in Design and Production—such as messages, paragraphs, and sections—you’ll need design packs when you migrate that content into DAS.

Keep in mind, however, that whenever resources in a design pack need to be updated, the corresponding Design Manager application must be repackaged and uploaded to DAS; in contrast, communication sets can be updated directly in DAS.

The following table provides an overview of minimum requirements to enable designers to use Communications Designer:

A Design Manager application for use with Communications Designer or equivalent resources in the DAS asset library

Either a Design Manager application or equivalent DAS library resources are required to create and fulfill a Communications Designer communication.

  • Using a Design Manager application—If you are using a Design Manager application, it must contain the following objects:

    Design pack

    Design packs contain resources for customizing a communication. At a minimum, it must contain a font or a style sheet, but you can also include colors, rules, and variables. The Design Manager application must contain at least one design pack.

    Design views

    Design views determine which design categories are available in Communications Designer. Each design view corresponds to a specific preview device that is configured for use with a particular design type. By default, applications are automatically configured to use the standard design view for page designs. To create email or web designs in Communications Designer, the application must also include design views for those design types. If a design view is not specified for a particular design type, then the design category and simulation option for that design type will be hidden in Communications Designer.

    Output queues

    Output queues determine the output types that can be generated when the communication is processed using the Exstream engine. At least one output queue is required.

    Data files

    At least one customer driver file is required to produce output from the communication. Sample data for the driver file can be mapped in Design Manager or in the Exstream web client. To use Communications Designer to map a sample file, the customer driver file in the Design Manager application must use the Communications Designer format.

  • Using DAS resources—If you are using only DAS resources to create a Communications Designer communication, you must create the following resources:

    Communication resources

    In the Communication asset library view, you set up the DAS output queues, data files, and variables that are to be used with a communication. You must also configure a communication set that contains these resources.

    Output queues determine the output types that can be generated when the communication is processed using the Exstream engine. They also determine which design categories are available in Communications Designer.

    At least one customer driver file is required to produce output from the communication. Sample data for the driver file can be mapped in the Mapping view of the Exstream web client.

    Design resources

    In the Design asset library view, you can set up design resources such as fonts, images, paper types, barcodes, metadata, sample files, and message types that will be used in the communication. All design resources (except barcodes) can be used for applications that were created using a communication set or those created using a design pack. Barcodes can only be used in applications created using a design pack.

    Note: If you use only DAS library resources to create a communication, then you must set the default font for the tenant in the Configurations panel for Communications Designer.

A template or a communication set in the DAS asset library

To create a new Communications Designer communication, users select either a template or a communication set in Communication launcher view. Templates and communication sets must be approved in order to be visible in Communication launcher view.

  • Using a template—Package and upload an appropriate Design Manager application to the DAS asset library and Design Manager to automatically creates a corresponding template and engine run definition for that application. The engine run definition is required to preview communications in Communications Designer.

  • Using a communication set—The communication sets that you create in the Communication asset library view are available to users in Communication launcher view. Creating a communication set also creates a corresponding engine run definition in the DAS asset library.

User permissions to access and work in Communications Designer

Communications Designer users must be members of at least one OTDS group that is assigned a role with at least the Access and Creator web client access permissions enabled for Communications Designer in that domain with the appropriate OTDS group and role pairing. For reviewers, the Approver option must also be enabled.
