Mapping data sources

When mapping data sources, there are several methods that you can use to map a variable to a data area within the sample data file. When a variable has been mapped, it appears as a variable chip in the row for that data area, and is propagated to the equivalent data areas within the data file.

Variable chip mapped to a data area

You can map multiple variables to the same data area, and there is no maximum to the number of variables that can be mapped to a specific data area.

Mapping a variable from the Variables panel

To map a variable from the Variables panel, simply drag and drop the variable that you want to use to the data area to be mapped. A variable chip is added to the data area row.

Note: Variables that are created in Communications Designer appear in italic in the Variables panel. This distinguishes them from Design Manager variables that have been imported from a design pack (which are displayed as non-italic text).

To help you quickly locate a specific variable, you can use the following options in the Variables panel:

  • Search for a variable—If you know part of a variable name, you can begin typing it in the Search box. The variable list is updated to display only variables with names that contain the entered text, whether the characters appear at the beginning, middle, or end of the variable name.

  • Sort by name or type—If you want to sort the variable list by name or by type, click the corresponding column header. The list can be sorted in ascending or descending order.

  • Sort by system or non-system variables—Use the System and Communication buttons to toggle between DAS system variables and those that originate either from a design pack or were created in the asset library.

    Note: Not all system variables are available for mapping a data source.

  • Filter the variable list—If you want to view only specific types of variables, use the filter buttons in the Variables panel. The following table lists the available variable type filters:

    Filter button Variable type

    All

    String

    Integer

    Float

    Date

    Currency

    Boolean

    Placeholder

  • Collapse the search and filter area—If you have a long list of variables and want to view as many as possible at one time, you can collapse the search bar and filter buttons area in the Variables panel by clicking the collapse bar .

Mapping a variable in the Data Source Editor area

  1. Point to the data area that you want to map, then click the Add new mapping button .

  2. In the variable selector, do one of the following:

    • To select an existing variable, select the variable from the suggested list. If the variable that you want to use is not displayed, begin typing any part of the variable name to update the suggested list with variables that contain the entered text, and then select the variable.

    • To create a new variable and map it to the data area, click Add new.

Automapping a data source

Automapping a data source lets you automatically find and map all of the data areas in a data source. You can use the automapping process to map the data source using new variables that are created as part of the automapping process, or using existing variables. The variable that is used to map a data area is determined in the following manner:

  • If a variable with the same name as a data area already exists, and the variable type matches the sample data, the automapping process uses the existing variable.

  • If the name of an existing variable matches the data area name but the type differs, the automapping process creates a new variable and appends the type to the variable name.

  • If no existing variable matches the data area name, the automapping process creates a new variable. The variable type is determined by the sample data.

Keep the following in mind when using the automapping process:

  • Automapping is often faster than manually mapping a sample file, but the automapping process can be less precise. If you see unexpected results, review the automapped data areas and edit the variable properties or data area properties as necessary.

  • You can edit the variable properties of variables that are created during automapping in the same way that you edit manually created variables.

  • Data areas can be both manually mapped as well as automapped. You can automap a data source either before or after manually mapping it. Previously mapped variables are not removed during automapping.

To automap a data source, in the toolbar, click the Automap button .

Changing an existing variable mapping

If you have previously mapped a variable to a data area, but would like to remove that mapping and map that variable to a different data area, simply drag and drop the variable chip to the other data area. Any data area properties that are associated with that variable will revert to their default settings after the variable has been moved.

Searching for a specific data element

If you want to search for a specific data element, type any part of the element name in the Find element box in the Data Source Editor.

As you type, any data elements with names that match the entered text are highlighted. The number of matching results is listed on the right side of the Find element box.

To navigate through the search results, use the up and down arrows, or you can press ENTER to move down and SHIFT+ENTER to move up.

To clear the search results, click X.

Updating data area properties

While the properties of a variable define data during output, the properties of a data area tell the engine how and when to read data.

If a variable is mapped to multiple data areas, the data area properties for that variable can be set individually for each data area.

To update the variable properties for a specific data area:

  1. Select the variable chip that corresponds to the variable and data area that you want to update.

    Example of selected AccountNumber variable

    The Data area properties section on the right of the Mapping view displays the variable settings for that particular data area.

  2. In the Input format list, select an option to specify the format of the data area. The list options are dependent on the variable type that is mapped to the data area:

    String

    • Keep blanks—Preserves spaces as they appear in the value

    • Trim blanks—Removes all of the spaces found before the first character and after the last character of the text

    • Trim leading blanks—Removes all of the spaces found before the first character

    • Trim trailing blanks—Removes all of the spaces found after the last character

    You can also specify the keep blanks, trim blanks, trim leading, and trim trailing options for each of the following additional input formats:

    • Lower—Converts the text to all lowercase characters

    • Upper—Converts the text to all uppercase characters

    • InitCap—Converts the text so that each word begins with an uppercase character

    • Sentence—Converts the text so that only the first word of a sentence begins with an uppercase character

    • Reverse—Reverses the text in the string

    Integer

    General number—Uses simple numbers without any special formatting (such as commas)

    Float

    General number—Uses simple numbers without any special formatting (such as commas)

    Date

    Multiple date formats— Provides several options for formatting date values. For example: mm-dd-yy, dd/mm/yyyy, mm/yyyy

    Currency

    • General number—Uses simple numbers without any special formatting (such as commas).

    • Locale currency—Uses the currency options for each locale as set in the system settings

    Boolean

    • T/F—Returns T when the condition is true or F when the condition is false

    • Y/N—Returns Y when the condition is true or N when the condition is false

    • 1/0—Returns 1 when the condition is true or 0 when the condition is false

    • TRUE/FALSE—Returns True when the condition is true or False when the condition is false

    • YES/NO—Returns Yes when the condition is true or No when the condition is false

    Placeholder

    • Placeholder content—The engine does not alter the data. If you select this option, you must also select the encoding of the content

    • Keep blanks—Preserves spaces as they appear in the value

    • Trim blanks—Removes all of the spaces found before the first character and after the last character of the text

    • Trim leading blanks—Removes all of the spaces found before the first character

    • Trim trailing blanks—Removes all of the spaces found after the last character

    • General number—Uses simple numbers without any special formatting (such as commas)

    There are also keep blanks, trim blanks, trim leading, and trim trailing options for each of the following additional formats:

    • Lower—Converts the text to all lowercase characters

    • Upper—Converts the text to all uppercase characters

    • InitCap—Converts the text so that each word begins with an uppercase character

    • Sentence—Converts the text so that only the first word of a sentence begins with an uppercase character

    • Reverse—Reverses the text in the string

  3. If the variable that is being mapped is a placeholder for a text file, set the Input Format to Placeholder content, and in the Encoding list select the encoding that is applied to the file.

  4. In the Action if data not found list, select one of the following options to specify what the engine does if the data is not found in the area during processing:

    • Ignore—Ignores missing data and continue processing

    • Message—Issues a message and continue processing

    • Message and stop—Issues a message and stop processing

    • Message, skip document—Issues a message and skip the current customer

  5. If the data area contains confidential information, select Design sample in the Make confidential using list.

  6. If the variable that is being mapped is a static array, specify which array element is read by completing one of the actions described in the following table:

    Action Description
    Use a specific array element when an array is defined horizontally on a row or is in a non-sequential order

    1. Select the Specific array element check box.

    2. In the Specific array element box, enter the element number of the array element to be read.
    Use the next available array element Clear the Specific array element check box.
Marking the start of a new customer

For each data source, you must specify which element starts a new customer. You must identify at least one element as a customer in the data; otherwise, the engine will not process your application.

When you identify a data area as the start of a new customer, the Start of customer designation will propagate to the equivalent data areas within the data file.

  1. Select the data area that you want to use as the start of a new customer.

  2. In the toolbar, click the Start customer button .

    Note: Clicking the Start customer button again removes the Start of customer designation.

Marking the start of a new data segment

You can use data segments to group records that are related to one particular type of information, such as all of the debits or all of the deposits for an account.

Since it helps identify groups of similar data, a data segment is especially useful when the order of the data is different for each customer, or when the number of data types varies for each customer. Data segments are also useful in communications that are similar in design but that contain different types of data, and for transactional communications. A data segment lets you design a table that can be used with multiple sets of data in a customer, particularly in transactional communications or in household mailings.

For more information about using data segments to create a table, see OpenText Exstream Communications Designer Online Help.

To mark the start of a new data segment:

  1. Select the data area that you want to use as the start of a new data segment. Typically this would be the starting tag of a group of data.

  2. In the toolbar, click the Start data segment button .

    Note: Clicking the Start data segment button again removes the Start of data segment designation.

  3. In the Data segment dialog box, specify a name and select one of the following occurrence options:

    • Always starts a new data segment—Starts a new data segment at each equivalent data area in the file.

    • First occurrence only—Starts a new data segment only the first time it appears in the customer.

  4. Click Insert.

You can use the Data segment properties panel to make changes as needed.

Selecting a reference key variable

If you are mapping a Reference data source, the Set reference key button becomes available in the toolbar. A reference key is a variable that is used to designate when a reference file is called from the customer driver file. The reference file lets you retrieve a supplemental set of customer data based on the key variable included in the customer driver file. The engine uses the key to locate relevant data in the reference file. There is no limit to how much information you can tie to a key variable.

To set a variable as the reference key:

  1. Select the mapped variable that you want to use as the reference key.

  2. In the toolbar, click the Set reference key button .

    A key icon appears next to the variable in the Data Source Editor area.

    Note: Clicking the Set reference key button again removes the reference key designation.

Once a reference key variable is designated, the variable name will be displayed on the properties panel for the reference file.

Removing a variable mapping

  1. Click the X within the variable chip that you want to remove.

  2. Click the Confirm button .
