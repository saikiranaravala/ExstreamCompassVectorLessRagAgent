Installing product keys for tenants

Exstream deployments can use the following types of product keys:

  • Non-production—A non-production, or development, key is used to design communications and create output for testing. When you use a non-production key to run the Exstream engine, a Demonstration Powered by Exstream watermark is placed at the top of each page in the generated output. The transactions are not counted, because customers have unlimited transactions when they run the Exstream engine using a non-production OTDS product key.

    Contents of a non-production product key

    When an Exstream (Cloud-Native) Non-Production Kit is ordered by OpenText Sales, it delivers a non-production product key which contains the following features:

    • The feature to use the transactional Exstream (Cloud-Native) software and generate watermarked, non-production output

    • One non-production Communications Designer Standard Named User Seat

    • One non-production Designer CEM Floating Design Seat

    The following seats must be purchased separately as non-production seats in order to be available in your non-production key:

    • Content Author Standard Named User Seats

    • Empower Standard Named User Seats

    • Additional Communications Designer Standard Named User Seats

    • Additional Design and Production CEM Floating Design Seats

    • Design PDF CEM Floating Design Seats

    • Design Compare CEM Floating Design Seats

    • PDF Converter Floating CEM Floating Design Seats

  • Production—A production key is required to create production output. Output that is produced using a production key does not contain an Exstream watermark.

    Contents of a production product key

    When an Exstream (Cloud-Native) Production Kit is ordered by OpenText Sales, it delivers a production product key which contains the following features:

    • The feature to use the transactional Exstream (Cloud-Native) software and generate production output

    • One production Communications Designer Standard Named User Seat

    • A quantity of annual transactions for production use

    The following seats must be purchased separately as production seats in order to be available in your production key:

    • Content Author Standard Named User Seats

    • Empower Standard Named User Seats

    • Additional Communications Designer Standard Named User Seats

    Production keys also maintain the transaction count for Exstream engine runs. A transaction is defined as any event in which an input record, or portion of an input record, is processed and measured by the software. This means that each customer record that is processed during an engine run is considered a transaction if final output is generated for that customer. If recipient processing is enabled, then the transaction count is also incremented for each recipient copy that is produced. Keep in mind that the following items are not considered final output:

    • Previews (including Empower previews)

    • Simulations

    • Output generated for Empower output queues or output queues for pre-sort processing (where the pre-sort processing is creating only a sort data file)

    Additional transactions can be ordered and added to your production key, if needed.

    Important: After a production key is used for an Exstream engine run that increments the transaction count, you can no longer remove that key from the OTDS tenant to which it is assigned. Make sure that production keys are assigned to correct tenants before initiating any production engine runs.

    Message transactions

    Exstream message transactions are designed and priced to accommodate the creation and delivery of smaller transactions, which are smaller in byte size than regular transactions. These micro-transactions in Exstream, such as high-volume mobile messages, are referred to as message transactions.

    Message transactions, if purchased, are incremented when the Messaging output driver is used to deliver SMS messages, and no other output driver is included in the production engine run.

    Note: If you are producing message output using the Messaging output driver in Exstream (Cloud-Native), and have either exceeded the transaction count for message transactions or have not purchased message transactions, then the engine will produce the message output and count it as a regular transaction.

  • Temporary—A temporary key is used strictly for evaluation purposes and needs to be installed on a separate tenant from any production and non-production keys. Temporary keys allow only for non-production output. If you’re using a temporary key, you should associate it with a tenant that is not already using a product key—such as the sample tenant that is included, but not automatically activated, in each Exstream deployment.

After you have created a tenant for use in your Exstream deployment, you must install Exstream OTDS product keys in the tenant before you can use it. For specific information about using and configuring OTDS, see OpenText Directory Services Web Client Help.

Step 1: Activate the product key on My Support

To activate a product key, you associate the unique SPS fingerprint for your tenant with the product key on My Support. After that, you'll download the product key so that you can associate the product key with the resource for your tenant. To complete the following steps, you must be able to access My Support on behalf of your organization.

To activate and download a product key:

  1. Locate the SPS fingerprint for your tenant in the OTDS web client:

    1. Sign in to the OTDS web client as a tenant administrator user at the following URL:

      https://<host>:<optionalPort>/otdstenant/<tenant>/otds-admin/

    2. In the INFO menu, click System Status.

    3. In the Version Information section, copy the SPS Fingerprint string.

  2. Sign in to My Support at http://support.opentext.com.

  3. Click the My Accounts list, and then click Activations/Keys.

  4. In the Product Activation area, in the Exstream section, locate the Exstream product key that you want to activate, and then click Produce License Key.

    Note: You can use the same SPS fingerprint for each license type in a tenant.

  5. In the Activations dialog that opens, in the SYSTEM FINGERPRINT field, paste the SPS fingerprint string that you copied earlier.

  6. Click Save.

  7. Click Download File and save the file locally for use when you associate the product key with the resource for your tenant.

Step 2: Activate an OTDS resource for the tenant

Each resource in OTDS represents an OpenText product. A product key that authorizes the use of the Exstream deployment must be associated with a resource. You must have an active resource for your tenant so that OTDS can use it to authenticate users in your Exstream deployment.

Note: If you are using a production product key and a non-production product key on the same tenant, then you must create separate resources for each product key.

To activate a resource for the tenant:

  1. Sign in to the OTDS web client as a tenant administrator user at the following URL:

    https://<host>:<optionalPort>/otdstenant/<tenant>/otds-admin/

  2. In the Setup menu, click Resources and then complete the following steps to create a new OTDS resource:

    Note: If you used a bootstrap configuration to set up your OTDS resource, then you do not need to create a new resource. In the Actions menu for the existing resource, click Activation status, and then go to the next step.

    1. From the button bar, click Add.

    2. On the General tab, in the Resource Name field, specify a name for the resource and other general settings as needed, and then click Next.

      Tip: Use a name that helps you easily identify whether the resource is associated with a production product key or a non-production product key.

    3. On the Synchronization tab, specify synchronization settings as needed, and then click Next.

    4. On the Principal Attribute tab, specify an authentication principal attribute, and then click Next.

    5. From the button bar, click Save.

  3. In the Resource Activation dialog box, copy the resource identifier string.

  4. Go to the Directory Services API for OTDS at the following URL:

    https://<host>:<optionalPort>/otdstenant/<tenant>/otdsws/rest

  5. In the Directory Services API, click Authorize near the top of the screen, and complete the following steps in the Available authorizations dialog:

    1. Specify scopes settings as needed, then click Authorize.

    2. If you are prompted to sign in, use OTDS administrator credentials.

    3. In the Available authorizations dialog, click Close.

  6. In the Directory Services API, complete the following steps to activate the resource: 

    1. In resources, click the /resources/{resource_id}/activate endpoint, then click Try it out.

    2. Paste the resource identifier value that you previously copied into the Resource ID box.

    3. Click Execute and scroll down to verify that you did not receive an error. The returned response should be something in the 200 range.

  7. Return to the OTDS web client, and in the Resource Activation dialog box, click Verify Activation.

  8. Click OK.

Step 3: Associate a product key with the resource in OTDS

With your resource created and active, you must then associate the product key that you downloaded earlier with the resource so that OTDS knows which products to make available to users when they sign in. The type of product key that you have determines product features such as the software components and functionality that is available, the number of users that can access a feature, and how many transactions are allowed.

To associate a product key with a resource:

  1. Sign in to the OTDS web client as a tenant administrator user at the following URL:

    https://<host>:<optionalPort>/otdstenant/<tenant>/otds-admin/

  2. In the Setup menu, click License Keys.

  3. On the button bar, click Add.

  4. On the General tab, in the License Key Name field, specify a name for the product key.

    Tip: Use a name that helps you easily identify whether the product key is a production key or a non-production key. You cannot edit this name after it is created.

  5. In the Resource ID list, select the resource that you created earlier, and then click Next.

  6. On the License Key tab, click Get License File.

  7. Select the Exstream product key that you downloaded earlier and click Open.

    The file information appears in the License Key field.

  8. Click Save.

At this point, your tenant is ready to use in your Exstream deployment. To associate another product key with the tenant, however, you will need to repeat all of the same steps for the second product key.

After you have associated your product key with the tenant, you can allocate users as needed to that product key so that they can use the Design and Production design tools.

Step 4: Allocate users, groups, or partitions to the product key in OTDS

To allow users to access the Design and Production design tools (which include Designer and Design Manager, among others), you’ll need to allocate specific users to the product key. You can also allocate groups or partitions to allocate all of the users in those groups or partitions to the product key. After you allocate users in OTDS, you can then configure Design Manager to use OTDS credentials for user authentication and to allow users to upload resources to DAS.

Depending on which features (design tools) are associated with the product key, the process of allocating users to a product key also grants them permission to utilize one of the Named User Seats (known as reserved seats in OTDS) for the web client design applications (Communications Designer, Content Author, and Empower). However, it does not automatically secure the seat until the user accesses a feature for the first time. Once a user is allocated to a seat, that seat stays reserved to that user unless revoked by an administrator in OTDS.

Note: Deleting a user does not automatically release the seat that is assigned to that user. A seat must be revoked in order for the seat to be available for reallocation.

For specific information about how licensing works with Exstream, see OpenText Exstream Design and Production Online Help. For information about OTDS users, groups, and partitions, or how to allocate or revoke reserved seats, see OpenText Directory Services Installation and Configuration Guide.

Important: You can use only temporary or non-production product keys for Design and Production allocation.

To allocate a user, group, or partition to the product key:

  1. Sign in to the OTDS web client as a tenant administrator user at the following URL:

    https://<host>:<optionalPort>/otdstenant/<tenant>/otds-admin/

  2. In the Setup menu, click Users & Groups or Partitions, depending on the allocation type.

  3. Locate the user, group, or partition that you want to allocate to the product key.

  4. In the Actions menu for each user, group, or partition, click Allocate to License.

  5. In the License list, select the product key.

  6. If you want to change which feature the user, group, or partition has permission to use, select the appropriate feature in the Counter list.

    Counters

    The counters that appear in the Counter list are determined by which features have been licensed.

    As a best practice, allocate counters individually to specific users or user groups.

    Important: EXSTREAM is the primary (root) counter and, if selected, allocates the license key to all of the EXSTREAM counters in the license key for that user, group of users, or all of the users in a partition. OpenText recommends that you control license key access by allocating specific feature counters to groups or individual users instead of to the EXSTREAM root counter.

    For example, if the counter list includes EXSTREAM, EXSTREAM.COMMUNICATIONS_DESIGNER, EXSTREAM.CONTENT_AUTHOR, and EXSTREAM.EMPOWER, selecting EXSTREAM will automatically provide access to Communications Designer, Content Author, and Empower. If you select only EXSTREAM.CONTENT_AUTHOR, then access is granted to a Content Author seat only.

    If needed, the allocation process can be repeated to provide access to more than one counter.

    Important points regarding counters:

    • A Communications Designer seat is required for access to Communications Designer, Communication asset library view, Design asset library view, Orchestration asset library view, and Monitor view.

    • A Content Author seat is required for access to Content Author.

    • It is not required to allocate the EXSTREAM.MESSAGES and EXSTREAM.TRANSACTIONS counters.

  7. Click Allocate to License.

  8. If necessary, repeat step 6 through step 7 to allocate a Named User Seat for any additional features.

  9. Click Close.

With the Design and Production user, group, or partition allocated to the product key, the Integration settings for OTDS in Design Manager can then be configured so that resources can be uploaded to DAS. For information about configuring Integration settings in Design Manager, see OpenText Exstream Design and Production Online Help.
