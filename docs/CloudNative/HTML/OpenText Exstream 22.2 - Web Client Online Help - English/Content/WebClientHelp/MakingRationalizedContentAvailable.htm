Making rationalized content available in communications

After you have rationalized a migration set, you can make the content of that migration set available for use in a Communications Designer communication. If you create a Content Author template from that communication, the content of the migration set will be available for Content Author communications based on that template.

Note: The options in this topic might not be available in your environment. If you have questions about Rationalization, you should talk to your Exstream administrator.

The method for including rationalized content from a migration set in a Communications Designer communication differs depending on whether the communication is based on a communication set or a design pack.

Note: Only migration sets that have been rationalized are available for use.

  • Using rationalized content with a communication set

    To use rationalized content in a communication that is based on a communication set, first add the migration set that includes the rationalized content to a data source, and then add that data source to the communication set. You can use migration sets only with Driver data source types; however, Empower driver files are not supported.

    1. Create a Driver data source and add the migration set that you want to use. Keep in mind that only one migration set can be added per data source.

    2. Create a communication set and add the data source that includes the migration set.

    3. Create a Communications Designer communication using the communication set.

      Any rationalized content that is part of the migration set (including text and image fragments, as well as variables) is then available to be used in the communication.

      If you subsequently create a Content Author template from the communication, then the migration set content will be available for any Content Author communications created from that template.

  • Using rationalized content with a design pack

    To add a migration set to a communication based on a design pack:

    1. In Communications Designer Structure view, select the communication.

    2. In the Properties panel, on the General tab, use the Migration set list to select a migration set.

      If you subsequently create a Content Author template from the communication, then the migration set content will be available for any Content Author communications created from that template.

For information about using rationalized content in designs created in Communications Designer or Content Author, see OpenText Exstream Communications Designer Online Help or OpenText Exstream Content Author Online Help.
