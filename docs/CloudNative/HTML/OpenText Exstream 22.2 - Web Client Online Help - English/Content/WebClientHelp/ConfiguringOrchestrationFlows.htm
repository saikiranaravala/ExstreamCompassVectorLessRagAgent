Adding orchestration flow resources

In Orchestration asset library view, you can create the resources that are used to configure the orchestration flows that determine the manner in which engine runs are processed to produce customer output. Orchestration flows are defined using the following resources:

  • Flow model—Defines the data flow for engine orchestration jobs, starting from incoming data, continuing through document production and ending with document output.
  • Flow context—Contains the input and output channel definitions and the communication settings that are used in designing flow models. All channels and communications, defined in all flow contexts inside the same tenant and domain as a flow model, are available when designing the flow model.
  • Flow script—Lets you create reusable flow scripts in order to implement orchestration logic. These scripts can be assigned to flow context items as before and after events, and also to script processors. Flow scripts are composed using JavaScript.

You use the Flow panel in the Orchestration asset library view to create flow contexts, flow models, and flow scripts, and then configure these resources in Communications Orchestrator or the Flow script composer. For more information about defining orchestration flows or creating flow scripts, see OpenText Exstream Communications Orchestrator User Guide.

Note: After they are created you must publish your flow resources to make them available for processing by the Orchestration service.

To work with flow resources, users must be members of at least one OTDS group that is assigned a role with at least the Access and Creator web client access permissions enabled for Orchestration asset library. For reviewers, the Approver option must also be enabled.

To create a new flow context, flow model, or flow script:

  1. In the Flow panel, click the Add button, then click one of the following options in the selection list.

    • Flow model
    • Flow context
    • Flow script

  2. In the Properties panel, specify a name, and optionally, a description for the resource that you want to create.

  3. Click Create.

To open a flow model or flow context in Communications Orchestrator, or open the Flow script composer, select the resource, then click Open.

Note: Communication settings in flow contexts can be defined only for approved package files or communications in the current tenant and domain.
