Applying metadata tags to resources

Metadata provides the ability to easily identify resources in output files for use in downstream business processes, such as with Core Experience Insights, and in output post-processing, such as applying additional CSS styles to specific design objects.

You can add metadata to Communications Designer communication resources, to individual design objects in these communications, and to DAS output queues and barcode resources. In email and web designs, you can also add metadata to containers. Output metadata is supported for the following output types:

  • HTML5
  • HTML (Email)
  • Messaging
  • PDF
  • Empower

Metadata tags are created and stored as DAS resources and can be managed in the same way as other DAS resource types. For information about managing resources in DAS, see Working with DAS resources.

For information about applying metadata to resources in Communications Designer, see OpenText Exstream Communications Designer Online Help.

Applying metadata to resources in the asset library

To apply metadata to resources, you use the following views depending on the type of resource:

  • Communication asset library view—For output queues
  • Design asset library view—For Communications Designer resources

  1. With the resource selected, click the Metadata tab in the Properties panel.

  2. Click the Assign metadata button .

  3. In the Metadata dialog box, select the check box for each metadata tag that you want to apply to the resource.

  4. Click OK to close the dialog box, and then click Save.

Removing applied metadata

To remove an applied metadata tag, click the Remove metadata button for that tag and then save your changes.
