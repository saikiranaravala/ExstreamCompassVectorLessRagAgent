Setting up email notifications for workflow changes

You can use Tenant management view to configure your Exstream deployment to send out notifications to specified email distribution lists when specific design resources are submitted for approval. For example, when a user submits a Content Author communication for approval, an automated email notification is sent to approvers on a distribution list to prompt them to review that communication.

Notification emails contain content and use settings that you specify on the Notifications panel. Each notification email also automatically includes a redirect link for the submitted resource so that approvers can access and review the resource directly. From the redirect link, resources open in the context that makes the most sense for each type—for example, Content Author communications open in Communications view in Content Author; Communications Designer pages open in Design view in Communications Designer; and so on.

To prevent an excessive number of emails being sent to a distribution list, when a resource is approved or rejected, an email is sent only to the user who originally submitted the resource for approval. In that case, Exstream checks OTDS for the email address that is associated with the user who submitted the resource for approval. If no email address is available in OTDS, then Exstream does not send a notification. You should ensure that approvers on a distribution list do not have the permissions required to change the workflow state for a submitted resource back to draft. If an approver returns a submitted resource to draft state, then no email notifications will be sent to any user about the change.

Important: If more than one notification exists for a resource type, then the user who submits a resource of that type for approval will receive more than one email when the resource is approved or rejected. For example, suppose that a notification is configured for only Content Author resource types, but another is configured for both Content Author and Communications Designer resource types. In that scenario, when a user submits a Content Author resource for approval, then that user will receive two email notifications for the same resource.

There are three basic steps involved in setting up a new notification:

Step 1: Specify parameters and activate the notification

  1. In the Notifications panel in Tenant management view, click Add.

  2. In the Approver distribution list box, enter the email address for the distribution list that will receive notifications. The address for the distribution list is not validated or maintained in DAS; it must exist in your mail server. If the distribution list is incorrect or does not exist, then the notification will not be sent.

  3. In the Resource types section, select the types of resources that will be included in the notifications:

    • All—Select this option to send notifications whenever any of the resources in the list are submitted for approval. Note that selecting no options in the list is the same as selecting All.

    • Communications Designer—Select this option to send notifications whenever a Communications Designer asset is submitted for approval. Communications Designer assets are listed on the Designer panel in Design asset library view.

    • Content Author—Select this option to send notifications whenever a Content Author asset is submitted for approval. Content Author assets are listed on the Author panel in Design asset library view.

    • Communication Asset—Select this option to send notifications whenever a communication asset is submitted for approval. Communication assets are listed on the panels found in Communication asset library view.

    • Design Asset—Select this option to send notifications whenever a design asset is submitted for approval. Design assets are listed on the Common panel in Design asset library view.

    • Orchestration Asset—Select this option to send notifications whenever an orchestration asset is submitted for approval. Orchestration assets are listed on the panels in Orchestration asset library view.

Step 2: Set up the global SMTP configuration

The Global panel is available on each notification, but the settings affect all notifications. If you have created notifications previously, then the fields on this panel will be populated already and do not need to be changed.

Note: If you delete all notifications from a tenant, the SMTP settings will persist in the DAS database. To clear SMTP settings from your DAS database, you should delete the global SMTP settings before you delete all notifications.

To set up the global SMTP configuration:

  1. In the Properties panel, click the Global panel, and then specify the following settings to configure the SMTP email server that DAS uses to send email notifications:

    • Host—The host name of the SMTP server
    • Port—The SMTP listener port
    • Username—The user name that is used to establish an authenticated connection to the SMTP server
    • Password—The password that is used to establish an authenticated connection to the SMTP server

  2. Click Save.

Step 3: Configure the email layout

  1. In the Properties panel, click the Layout panel.

  2. Enter the subject line and content of the email notification in the Email subject box and Email body box, respectively. These are required fields and cannot be blank.

    When you place your cursor in the Email subject or the Email body box, you can also enter resource properties from the Resource property variables list. These properties are treated as resource-specific variables where they appear in the email content. When each notification is sent, the variable values are populated with the current information for the resource that triggered the notification. The following table lists the resource properties that are available as variables. No other variables can be used in email notifications.

    Note: The variables that appear in the Resource property variables list have user-friendly names that don't necessarily match the information inside the brackets after you place a variable. For example, the Updated by variable in the list appears as {{lastModifiedBy}} when placed in the Email subject or Email body fields. The names in the brackets represent actual field names in the DAS database.

    Resource property

    Description

    Created By

    The user that created the resource

    Description

    The description of the resource, if any

    Domain

    The domain in which the resource was created or updated

    Effective date

    The start date on which the resource is available to be included in customer output. This option is available in the list only if effectivity is enabled for the current tenant.

    Expiration date

    The date on which the resource is no longer available to be included in customer output. This option is available in the list only if effectivity is enabled for the current tenant.

    ID

    The DAS resource identifier string for the resource

    Name

    The name of the resource

    Updated

    The date and time when the resource was last updated

    Updated by

    The user that last modified the resource

    Version

    The current version of the resource

    Workflow comment

    The comment, if any, in the workflow state change that triggered the notification

  3. Click Create.

    The notification is created. On the General tab, the Active indicator in the Notification status section turns green to indicate that the notification has been activated. To deactivate the notification in the future, simply click the Inactive indicator.

With the notification in place, whenever a specified resource type is submitted for approval, an email will be sent to the distribution list. The email body will contain the text that you specified, including any populated variables. Additionally, the email will contain a link to the object that was submitted so that it can be accessed directly from the notification email.
