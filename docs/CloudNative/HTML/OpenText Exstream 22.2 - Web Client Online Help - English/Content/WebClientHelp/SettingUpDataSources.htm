Setting up data sources

A data source is what Communications Designer uses to identify which sample file to use for a particular data map. The same sample file can be used in multiple data sources, and be mapped differently for each data source with which is it associated.

When a data source is assigned to a communication, any variables mapped in the data source become available for use in the communication.

Use the Communication asset library view to create and modify the data sources that can be used when fulfilling your communications.

To access the Communication asset library view, in Content Launcher, on the toolbar, click the Communication asset library button .

Note: Only Communications Designer user roles that have the proper Resource access permissions allow access to the Communication asset library view. If you have questions about your access privileges, talk to your system administrator.

Creating a new data source

  1. In the Data panel, click the Add button, then click Data source in the selection list.

  2. In the Properties panel, provide the following information:

    • Enter a name and, optionally, a description for the data source that you want to create.

    • If you want to use resources from a design pack, select Use design pack, click Find design pack, and then select a design pack.

      Note: This option is only available if a design pack with a Communications Designer driver file is present in Orchestration asset library view.

    • Click the Data source type list to select the data file type being mapped:

      • Driver—Driver files contain at least one set of records for each customer. They drive the main processing of the engine and are usually the primary source of data.

      • Reference—Reference files let the engine access a supplemental set of customer data that is not included in a customer driver file. This extra information is retrieved based on a key variable included in the customer driver file.

      • Report—Report files let the engine collect data in order to create a report that includes any information that you select about the production run.

      Note: If using a design pack, this field is pre-populated with the Driver type.

    • Select an option in the Data source format list to define the layout of data in a data file.

      • XML—The data file consists of data that uses tags to define the beginning and end of a data element. For XML files, Communications Designer supports the following features: elements and values; attributes; and namespaces.

      • JSON—The data file consists of data that uses the JavaScript Object Notation (JSON) text format to define data elements.

      • Empower—The data file consists of data located in an Empower document (MPW file).

        This option is only available if the data source type is Driver.

        Note: Empower data sources only support PDF and HTML Email outputs.

    • If you are creating a Report data source:

      Select one of the following options in the I/O timing list:

      • At end of customer—The engine writes data records after each customer is complete. Each customer's information can be summarized for reports, including the current production run.

      • Completion of all customers—The engine writes data records after it finishes processing. This option creates a single output record upon completion of processing. It ensures that all of the variables are processed and written to the report in case the first records are shorter and contain less data than later records.

    • If you are creating a Reference data source:

      Select one of the following options in the Access method timing list:

      • Memory—The engine reads the entire reference file and stores all of the data in memory.

        This option offers the fastest processing but uses the most memory. With small reference files, this option yields the best results.

      • Driver-ordered, required—The engine reads customer data in the customer driver file, looks for corresponding key variable values in the reference file, and then returns to the customer driver file for the next customer. The engine does not store keys in memory so the customer driver file and reference file values must be in the same order. If the reference file does not have a match at the same customer position, the engine reads the remainder of the reference file until it finds a match. The engine requires the specified key to be in the reference file. If the engine reaches the end of the reference file without a match, the run stops and you receive a severe error message.

        Select this option if the customer driver file exactly matches reference files or if the customer driver file contains fewer customers than the reference file.

      • Driver-ordered, optional—The engine reads customer data in the customer driver file, looks for corresponding key variable values in the reference file, and then returns to the customer driver file for the next customer. The engine does not store keys in memory so the customer driver file and reference file values must be in the same order. If the reference file does not have a match at the same customer position, the engine immediately stops reading and resets the reference file position to test for a match for the next customer. If the engine finds a value in the reference file that does not have a corresponding match in the customer driver file, then all subsequent reads to the reference file fail.

        Select this option when the reference file is a subset of the customer driver file key values.

    • Click Find sample file to select the sample file that you want to use.

      The sample file type must match the data source type for it to be available in the list, with one exception: to access a sample file for Empower, the sample file type must be set to Empower driver.

      Note: If you use a JSON sample file, and any of its data element names contain spaces, you cannot map variables to those elements.

      If the file has been mapped previously and contains any variables mapped to those elements, those variables will not be processed properly by the engine (that is, the values for those elements will be missing in the output).

    • Optionally, select a Migration set to associate with the data source. When you link this data source to a Communications Designer communication, the associated migration set will also be linked with that communication.

      Note: This option is only available for driver data files and when a migration set is present in Rationalization asset library view.

    • Provide the Production data source name by entering the symbolic name or the path and file name of the file to use for production.

      Note: If using a design pack, this field is pre-populated with the production file designated in the design pack.

  3. Click Create.

After the data source is created, the following options become available in the properties panel, based on the data source type:

  • Driver: End segment at closing tag—When selected, this option ends the data segment at the corresponding closing element instead of at the start of a new data segment or the start of a new customer. This option is enabled by default.

    Note: The option is not available for the Empower data source format.

  • Reference: Reference key—This identifies the variable that is used as the key for the reference file. This key is used to call the reference file from the customer driver file. The reference key is specified in Mapping view.

  • Report: Tag level indentation—You can select one of the following options in the list:

    • 0–10—Select the number of spaces (0–10) to indent for each tag level in the output. The default is 0.

    • Lean JSON / Lean XML—Select a lean option to suppress indentation, carriage returns, and line feeds in the output. This can reduce the size of the file.

Opening an existing data source

  1. In the Data panel, click one of the following buttons:

    • Library—Displays all of the existing data sources

    • Favorites—Displays a list of data sources that have been bookmarked

      Tip: If you work frequently with a specific data source, you can bookmark that data source so that it appears in the Favorites list. To bookmark a data source, point to the data source in the Library tab, and click the bookmark icon () in the Favorites column.

  2. Select the data source that you want to open, and click Open.

    The data source opens in Mapping view.

Handling orphaned mappings

Orphaned mappings are data areas in a sample file that previously had variables mapped to them, but that are no longer present in that sample file. Orphaned mappings occur when a sample file that is associated with a data source is changed after it has been previously mapped. When you open a data source that contains orphaned mappings, you receive a message that indicates which previously mapped data elements are missing.

To close the message, select one of the following options:

  • To ignore all of the orphaned mappings and remove them from the data source, click Discard.

  • To retain the variable mappings in the data source, click Keep. You might want to do this, for example, if you are using a production file that has the correct data elements, but the sample file no longer has those elements.

    Note: If you choose to keep orphaned mappings, you will continue to receive the message each time you open the data source.

Updating the data source properties

If you have already created a data source but want to change any of its properties, you can do so by selecting the data source, making the necessary adjustments in the Properties panel, and then clicking Save.

Note: If the data source has been approved, submitted for approval, or is in a rejected state, the properties cannot be changed.

Viewing the references for a data source

You can use the References tab in the Properties panel to see all of the linked resources for a selected resource, if applicable.

To see the references for a resource:

  1. In the asset library panel, select a resource.

  2. In the Properties panel, click the References tab.

  3. To display a list of references for the selected resource, click the following buttons:

    • Used by—The list shows all of the parent resources that use the selected resource. Keep in mind that this list displays only parent resources with a direct link to the selected resource. Resources that use the parent resources are not listed. This option is selected by default.

    • Uses—The list shows all of the child resources that are used by the selected resource, including resources that are used by each child resource.

  4. To filter the displayed list by resource type, click the resource type selection list and select from the available options. You can select as many resource types as you want. To select all types, simply clear all existing selections.

By default, the first 50 items are displayed in the Properties panel. To change the number of items displayed in the panel, use the Items per page list. If the number of resources is too large to display in the panel, you can navigate the resource list with the page navigation buttons .
