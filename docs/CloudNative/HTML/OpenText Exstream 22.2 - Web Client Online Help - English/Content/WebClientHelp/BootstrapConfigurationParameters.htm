Specifying bootstrap configuration parameters

At a very high level, the bootstrap configuration is provided in the following format:

Copy

Bootstrap parameter block

bootstrap:
    enabled: true
    config:
        otdsAdmin:
            <OTDS administrator credentials>
        createOauthClients: true | false
        tenants:
            <tenant>:
                <Custom configuration for the new tenant>

Use the following information as a handy reference for specific parameter values and formatting as you create your custom bootstrap configuration.

bootstrap.config.otdsAdmin

In bootstrap.config.otdsAdmin, you provide OTDS system administrator credentials that the bootstrap can use to access your OTDS instance. The following table describes the parameters in this parameter block.

Parameter descriptions

Parameter

Description

username

Enter the user name of the OTDS system administrator user. For example: otadmin@otds.admin

password

Enter the password of the OTDS system administrator user.

bootstrap.config.createOauthClients

The bootstrap.config.createOauthClients parameter determines how you create OAuth clients for the new tenants that will be created with the bootstrap configuration. You can set bootstrap.config.createOauthClients to one of the following values:

  • true—Automatically creates OAuth clients in OTDS for the new tenants as needed. This is the default value if you do not explicitly set a value.
  • false—Does not create OAuth clients in OTDS for the new tenants

bootstrap.config.tenants

In bootstrap.config.tenants, you provide the configuration information for each new tenant that you want to create. Tenant configuration parameters are provided in the following parameter block:

Copy

Tenant configuration parameter block

tenants:
    <tenant>:
        otds:
            <OTDS tenant configuration settings>
        das:
            <Design tenant configuration settings>
        orchestration:
            <Orchestration tenant configuration settings>
        empower:
            <Empower tenant configuration settings>

To create multiple tenants using the same bootstrap configuration, use the following format to add the configuration parameters for each tenant in separate blocks:

Copy

Multiple tenant configuration

tenants:
    <tenant1>:
        <Configuration settings for tenant1>
    <tenant2>:
        <Configuration settings for tenant2>
    .
    .
    .
    <tenantN>:
        <Configuration settings for tenantN>

Tip: When creating multiple tenants, you can alternatively use separate YAML files for each tenant.

The following sections describe the configuration settings that you specify in each bootstrap.config.tenants.<tenant> block.

bootstrap.config.tenants.<tenant>.otds

In bootstrap.config.tenants.<tenant>.otds, you provide the OTDS information that is required to configure the tenant in OTDS. These tenant configuration settings are provided in the following format:

Copy

OTDS tenant configuration

otds:
    admin:
        username:
        password:
    exstreamAdmin:
        username:
        password:
    partitions:
        <partition>:
            groups:
                - name:
            users:
                - username
                password:
                membership: [ <group1>,<group2>,...,<groupN> ]
    resources:
        - name:
        description:
        licenses:
            - username:
            partition:
            prodLicense:
            file:

The following table describes the parameters in this parameter block.

Parameter descriptions

Parameter

Description

admin

Creates a new OTDS system administrator user that will be created in the new tenant. This user is a member of the otadmins@otds.admin group in the new tenant and can manage the OTDS tenants. Specify the following parameters:

  • username—Enter the user name of the new user.
  • password—Enter the password of the new user.

exstreamAdmin

Creates a new Exstream tenant administrator user that will be created in the new tenant. This user is a member of the tenantadmins@otds.admin group in the new tenant and can manage the Exstream tenant environment. Specify the following parameters:

  • username—Enter the user name of the new user.
  • password—Enter the password of the new user.

partitions

Creates a partition in the new tenant that will contain the users and groups for the tenant. In the partitions.<partition> block, specify the following parameters to configure the new partition:

  • groups—Specify the following parameter to create a new group, and repeat it for each subsequent group that you want to create:

    • name—Enter the name of the new group.

  • users—Specify the following parameters to create a new user and assign it to an OTDS group, and repeat them for each subsequent user that you want to create:

    • username—Enter the name of the new user.
    • password—Enter the password of the new user.

    • membership—Enter the OTDS groups to which the new user belongs.

resources

Creates an OTDS resource that will be used to store the Exstream OTDS product key for the new tenant. Specify the following parameters to create a resource, and repeat them for each subsequent resource that you want to create:

  • name—Enter a name for the new resource.
  • description—Optionally, enter a description for the resource.

  • licenses—Specify the product key with the following parameters:

    • username—Enter the user name of a tenant administrator user for this tenant. This can be the same as the user name that you specified in the exstreamAdmin parameter block.

    • partition—Enter the partition in which to create the OTDS resource for this product key.

    • prodLicense—Set to true if the resource will be assigned a production key. Otherwise, set to false.

    • name—Enter the name for the product key. This parameter is especially useful if you create non-production and production license key objects in OTDS for the same tenant. Specify a separate parameter for each key file that you are installing in the tenant.

    • file—Enter the contents of the key file to be installed in this tenant, or use --set-file in your Helm command to set a product key as this parameter value.

      Note: If you use this parameter with --set-file, make sure you comment it out in the bootstrap configuration.
bootstrap.config.tenants.<tenant>.das

In bootstrap.config.tenants.<tenant>.das, you provide the information that is required to configure the tenant for the Design service. These tenant configuration settings are provided in the following parameter block:

Copy

Design tenant configuration

das:
    databaseInfo:
        schema:
        username:
        password:
    roles:
        - name:
        appPermissions: [ ALL or <app1>,<app2>,...,<appN> ]
        permissions:
            - permissions: [ ALL or CREATE, READ, WRITE, DELETE, APPROVE ]
            types: [ ALL or <type1>,<type2>,...,<typeN> ]
    domains:
        - id:
        production: true | false
        roleMappings:
            - role:
            groups:

The following table describes the parameters in this parameter block.

Parameter descriptions

Parameter

Description

databaseInfo

Provides the Design tenant schema information for the tenant. This parameter block is required only if you are using manual schema management for your Exstream databases. You must have first created the tenant schemas in your database. If you are using automatic schema management, the bootstrap process creates the tenant schemas for the new tenant.

Specify the schema information in the following parameters: 

  • schema—Enter the name of the Design tenant schema.
  • username—Enter the user name of the schema owner.
  • password—Enter the password of the new user.

roles

Creates Exstream roles that are used to define user permissions in each domain in the tenant. Specify the following parameters to create a role, and repeat them for each subsequent role that you want to create:

  • name—Enter a name for the new role.

  • appPermissions—Enter the specific application permissions that you want to assign to the new role. To assign all permissions, enter ALL. Otherwise, select from the following supported application permissions:

    • COMM_DESIGNER—Enables access to Communications Designer
    • CONTENT_AUTHOR—Enables access to Content Author
    • DATA_MAPPER—Enables access to Communication asset library view
    • DESIGNER_ASSET_MANAGER—Enables access to Design asset library view
    • ORCHESTRATION_MANAGEMENT—Enables access to Communication asset library view
    • ORCHESTRATOR—Enables access to Orchestration asset library view
    • MONITOR—Enables access to Monitor view
    • EDIT_CATEGORIES—Enables access to creating category tags in supported views

  • permissions—Specify resource permissions with the following parameters:

    • permissions—Enter the specific resource permissions that you want to assign to the new role. To assign all permissions, enter ALL. Otherwise, select from CREATE, READ, WRITE, DELETE, and APPROVE.

    • types—Enter the resource types to which you want to apply the selected permissions. To apply permissions to all DAS resource types, enter ALL. Otherwise, select from the following supported resource types:

      • documentdefinition—Enables resource permissions for Content Author communications
      • enginerundef—Enables resource permissions for engine run definitions
      • exstrapplication—Enables resource permissions for Communications Designer communications
      • exstrbarcode—Enables resource permissions for barcodes
      • exstrcommunicationset—Enables resource permissions for communication sets
      • exstrcomponentobj—Enables resource permissions for Communications Designer components
      • exstrdatasource—Enables resource permissions for data sources
      • exstrdocument—Enables resource permissions for Communications Designer documents
      • exstremail—Enables resource permissions for Communications Designer email designs
      • exstrengine—Enables resource permissions for Exstream engines
      • exstrgraphicalmessage—Enables resource permissions for Content Author graphic messages
      • exstrhtml5—Enables resource permissions for Communications Designer web designs
      • exstrmessagetype—Enables resource permissions for message types
      • exstrmetadata—Enables resource permissions for metadata tags
      • exstrmigrationset—Enables resource permissions for migration sets
      • exstroutput—Enables resource permissions for DAS outputs
      • exstroutputqueue—Enables resource permissions for DAS output queues
      • exstrpackage—Enables resource permissions for Design Manager application package files and subpackage files.
      • exstrpage—Enables resource permissions for Communications Designer page designs
      • exstrpapertype—Enables resource permissions for paper types
      • exstrparagraph—Enables resource permissions for Content Author clauses
      • exstrparagraphsection—Enables resource permissions for Content Author sections
      • exstrtextmessage—Enables resource permissions for Content Author text messages
      • exstrvariablebase—Enables resource permissions for DAS variables
      • flowmodel—Enables resource permissions for flow models
      • fontdefinition—Enables resource permissions for fonts
      • font—Enables resource permissions for DAS font faces
      • image—Enables resource permissions for images
      • orcsettings—Enables resource permissions for flow contexts
      • resourcepack—Enables resource permissions for design packs
      • samplefile—Enables resource permissions for sample files
      • versionedtemplate—Enables resource permissions for templates

domains

Creates and configures the domains in the new tenant. Specify the following parameters to create a domain and repeat them for each subsequent domain that you want to create:

  • id—Enter the name of the new domain.
  • production—Set to true if the domain will be used for production engine runs. Otherwise, set to false.

  • roleMappings—Specify the following parameters to assign a specific role to one or more OTDS groups and repeat them for each subsequent role that you want to assign to OTDS groups:

    • role—Enter a role that exists in this tenant. You use the roles parameter block to create roles in the tenant.
    • groups—Enter the existing OTDS groups to which this role should be assigned, separating each group name with a comma. You use the groups parameter in the partitions parameter block in the OTDS tenant configuration to create new groups in OTDS
