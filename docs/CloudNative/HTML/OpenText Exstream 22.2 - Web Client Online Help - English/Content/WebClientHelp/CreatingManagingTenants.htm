Setting up tenants in Exstream

When you first install your Exstream deployment, the Helm chart includes the default configuration that creates the system tenant in the deployment. If you choose to automatically manage schemas in your Exstream databases, the default configuration also creates a sample tenant environment.

After you initially deploy Exstream, you can use the following methods to create additional tenants in your deployment:

  • Tenant creation wizard—You use System management view in the Exstream web client to access the tenant creation wizard. The wizard provides a convenient user interface to set up each tenant in the Exstream deployment, and ensures that the tenant schemas are associated correctly with each service in the deployment. If you use the tenant creation wizard, you individually create each tenant in Exstream, and then use Tenant management view to configure each tenant environment.
  • Bootstrap configuration—You use a bootstrap configuration with a Helm command to create and configure new tenants. The bootstrap configuration is a predefined parameter block that contains custom tenant configuration parameters. You can specify the bootstrap configuration parameters in the values.yaml file that you used to initially deploy Exstream or you can create a separate bootstrap YAML file. Using a bootstrap configuration lets you automate the tenant setup process with a helm upgrade command. When you specify your bootstrap configuration parameters, you can also customize and select specific parts of the tenant configuration process that you want to automate.

  • OTDS tenants—You use the OTDS REST API to create inherited tenants, which is the type of OTDS tenant that is used with Exstream.

    To access the OTDS tenant REST API, use the following endpoint:

    {OTDS URL}/otdsws/api/index.html?tenants

Note: You can also use a custom bootstrap configuration with the helm install command to create and configure individual tenants during the initial deployment. For more information about using bootstrap files during deployment, see OpenText Exstream Installation and Configuration Guide.

Regardless of which tenant creation method you use, the manner in which you set up a tenant environment is determined by the schema management method that you selected when you deployed Exstream. Manual schema management

  1. Set up empty tenant schemas in the database for each of the Exstream services. The tenant name must be the same for each service.

  2. Assign a schema owner for each schema. At a minimum, the schema owner must have CRUD permissions (Create, Read, Update, and Delete).

  3. Use Liquibase to generate the SQL statements and initialize each of the schemas in your database.

  4. In the tenant creation wizard, configure the corresponding OTDS tenant, and then associate the tenants for each service.
  5. Install a product key in the OTDS tenant to make the tenant environment available for use.
  6. In Tenant management view, configure roles and domains to allow users to work in the tenant.
