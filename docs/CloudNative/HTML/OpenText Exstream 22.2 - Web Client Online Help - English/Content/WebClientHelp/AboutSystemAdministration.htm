System administration

After you perform all of the steps necessary to initially deploy Exstream, you'll need to configure the deployment for your organization needs.

All Exstream deployments use tenants and domains to keep data separate between business groups. Your specific business needs determine which type of separation that you’ll use. At a high level, when you create a tenant, you’re creating a data-isolated schema within your database. Domains, on the other hand, are simply logical groupings of users and content within each tenant.

Each Exstream tenant environment comprises a tenant for each Exstream service , with a corresponding tenant in OTDS. All tenants in the same tenant environment must have the same name. You must set up at least one tenant environment to enable users to sign in and work in Exstream. You can choose to either manually create and configure tenant environments in your deployment or you can use a Helm command with bootstrap configuration parameters to automate the tenant configuration process. For each of these processes, you can decide whether to use automatic schema management for your Exstream databases.

Important: To ensure security in a multi-tenant deployment, you must use a separate schema and schema user for each tenant. If you're using automatic schema management, Exstream creates these schemas and schema users for each tenant automatically.

Regardless of which tenant configuration method you select, the system tenant is configured automatically when Exstream is first deployed, and is used to manage other tenants and engines in the Exstream deployment. In the system tenant, you use the System management view to create and manage tenants, and to upload the Exstream engines that are used in all other tenants. This tenant forms a purely administrative environment; it cannot be used for creating and testing communications. To access the system tenant, you must be a member of the otadmins group in the OTDS system tenant.

After you create tenants, you can use Tenant management view to configure individual tenant environments. In Tenant management view, you can create and configure domains, assign permissions to use roles, and create configurations and categories. To manage a tenant environment, you must be a member of the tenantadmins group in the corresponding OTDS tenant.

The following table describes the steps to access each view.

View

Steps

System management view

Sign in to the Exstream web client as an OTDS administrator user at the following URL:

https://<host>:<optionalPort>/design?tenant=system

Tenant management view

  1. Sign in to the Exstream web client as a tenant administrator user at the following URL:

    https://<host>:<optionalPort>/design?tenant=<tenantName>

  2. Click the Tenant management button .
