Creating tenants in System management view

If you are using automatic schema management for your Exstream databases, you can use the tenant creation wizard in System management view to create and set up a new tenant. If you are using manual schema managed for your Exstream databases, then you will need to first set up tenant schemas directly in the database for each of the Exstream services, and then use System management view to associate the new tenant with the schema for each service in the Exstream deployment.

Note: Regardless of the schema management method that you use for your databases, you must use the OTDS REST API to create inherited tenants, which is the type of OTDS tenant that is used with Exstream.

To access the OTDS tenant REST API, use the following endpoint:

{OTDS URL}/otdsws/api/index.html?tenants

When you use the wizard to create a new tenant environment, it will name and configure tenants for each of the Exstream services that you use. After you create new tenants, you will need to install a product key in each new tenant and use Tenant management view to configure the roles and domains in each tenant environment to make it available to users.

Associating existing tenants

  1. On the Design tenant details screen, enter the Design schema user name and password, and then click Associate.

  2. If your deployment includes Empower, in the Empower tenant details screen, enter the Empower schema user name and password, and then click Associate.

  3. On the Orchestration tenant details screen, enter the Orchestration schema user name and password, and then click Associate.

  4. On the Overview screen, review the schema information for the new tenant, and then click Associate.

    The tenant appears in the Tenants and services panel, with an indicator for each service.

Creating new tenants

  1. Sign in to the Exstream web client as an OTDS administrator user at the following URL:

    https://<host>:<optionalPort>/design?tenant=system

  2. On the Tenants and services panel, click Add tenant.

  3. In the Tenant information dialog box, in the Tenant name field, specify a name for the tenant, and then click Proceed.

    Note: Tenant names are limited to 28 characters. You can use alphanumeric characters and the following special characters: ! $ & ( ) * - . : _ ~

  4. On the OTDS tenant details screen, enter a user name and password to create an OTDS administrator user for the new tenant, and then click Create. The OTDS tenant and the user are created immediately in OTDS.

    If you already have a tenant with the same name in OTDS, then you do not need to specify an administrator user name and password. The tenant creation wizard detects the existing tenant.

    If you cancel later while creating the tenant for any of the other services, this tenant will remain in OTDS, but tenants for the other services will not be created. If you come back later and try to create a tenant with the same name, the tenant creation wizard will recognize the existing OTDS tenant, and you can continue to create corresponding tenants for the remaining services.

  5. Depending on the schema management method that you are using for your Exstream databases, do one of the following for each subsequent screen in the wizard:

    • For automatically managed schemas, click Create to create the tenant for each service in your Exstream deployment.
    • For manually managed schemas, enter the appropriate schema user name and password for the service, and then click Associate to associate the existing tenant schemas with each service in the Exstream deployment.

    In a typical Exstream deployment, you can create and associate tenants for the following services:

    • Design
    • Empower
    • Orchestration
    • Experience Insights

  6. On the final screen, review the schema information for the new tenant, and then click Close. The tenant appears in the Tenants and services panel, with an indicator for each service.

Understanding the color indicators for tenants

Use the color indicators to quickly see general information about each tenant in your Exstream deployment. Keep in mind that the indicators do not show the current status of the tenant; instead, they are meant to tell you at a glance whether a tenant has been created for a particular service or is currently enabled in OTDS. You can point to any of the indicators to see a tooltip that provides more information.

The following table shows the indicators that are available in the Tenants and services panel, and what each indicator means.

Tenant indicators
Indicator Description
The tenant has been created for the indicated service.
An error occurred with the tenant during the current browser session. If you resolve the error, then the indicator will change to green; however, if you do not resolve the error, then the indicator will change to gray during subsequent browser sessions.
An error occurred with the tenant during a previous browser session. If you resolve the error, then the indicator will change to green; otherwise, it will continue to be gray.

You’ll see this indicator in the following scenarios:

  • The tenant has been disabled in OTDS. For information about enabling and disabling tenants in OTDS, see OpenText Directory Services Installation and Configuration Guide.

  • The tenant has been created in an Exstream service, but has not been created in OTDS. You must create a corresponding tenant in OTDS before the tenant environment can be used in your Exstream deployment.

Setting up required OTDS groups

To manage and use individual tenant environments in Exstream, you must set up the appropriate user groups in OTDS and add at least one user to each group. The following table lists the minimum OTDS groups that are required in each new tenant.

Required OTDS groups
OTDS group Purpose
tenantadmins

Tenant administrator users must be members of the tenantadmins group in the corresponding OTDS tenant.

tenantusers

Tenant users that are not part of another required group should be included as members of the tenantusers group in the corresponding OTDS tenant.
empoweradmins

Empower administrators in an Empower tenant must be members of the empoweradmins group in the corresponding OTDS tenant.

empowerintegrators

Empower system integrators in an Empower tenant must be members of the empowerintegrators group in the corresponding OTDS tenant.

empowereditors

Empower Editor users for an Empower tenant must be members of the empowereditors group in the corresponding OTDS tenant.

In addition to these groups, you might need to create additional OTDS groups for various users depending on your organization’s needs. See the default OTDS groups in the sample OTDS tenant for an example of a typical deployment.

For detailed information about creating users and groups in OTDS, see OpenText Directory Services Web Client Help.
