Monitoring orchestration jobs

You can use the Monitor view to see the status of the jobs and flow instances that correspond to your engine orchestration jobs.

When you send data to an input channel, it initiates new orchestration jobs for any published flow models that use that input channel. Each time a flow model is processed, the Orchestration service creates a corresponding flow instance.

Jobs

The Jobs panel displays the following information for orchestration jobs:

Column Description

ID

ID of the orchestration job

Channel

Name of the input channel that initiated the orchestration job

Start

Processing start time for the job

Finish

Processing end time for the job

Status

Status of the job. Select the job to see additional information about the status on the General tab in the Properties panel.
Triggered by The Exstream role that initiated the orchestration job

You can also view the associated flow instances for each job on the General tab in the Properties panel.

Flows

The Flows panel displays the following information for flow instances:

Column Description

ID

ID of the flow instance that was initiated for a specific orchestration job

Job ID

ID of the orchestration job that was initiated by sending data to the input channel

Flow model

Name of the flow model that corresponds to the initiated flow instance

Channel

Name of the input channel that initiated the orchestration job

Start

Processing start time for the flow instance

Finish

Processing end time for the flow instance

Status

Status of the flow instance. Select the flow instance to see additional information about the status on the General tab in the Properties panel.

On the General tab in the Properties panel, for each flow instance, you can also access the following files that are used or generated by the Exstream engine:

Type Files

Input

Input data file that was used to initiate the corresponding orchestration job

Log files

  • Message file—The engine message file that was generated during the engine run for the flow instance
  • Delivery file—The output delivery manifest that contains a comprehensive list of the output composed during the engine run

Output files

The output generated from the engine run

If job tracing has been enabled for the domain, on the References tab in the Properties panel, for each flow instance, you can see which resources—and which version of those resources—were used, as well as who modified and approved them. This information is available as long as the job is in Monitor view.

Note: Only resources that are required for fulfillment of a job are listed on this References tab. However, fonts are an exception to this; to see which fonts were used, check the References tab of the communication in Design asset library view.

Canceling a job, flow instance, or flow step

If an orchestration job is running, you can cancel all or part of the job, if necessary, while the job is incomplete.

Canceling a job

To cancel the entire job:

  1. In Monitor view, on the Jobs tab, click the More actions button at the end of the row of the job that you want to cancel.

  2. Click Cancel job.

    The status indicator turns orange and its tooltip changes to Canceled.

Canceling a flow instance

To cancel a flow instance:

  1. In Monitor view, on the Flows tab, click the More actions button at the end of the row of the flow instance that you want to cancel.

  2. Click Cancel flow.

    The status indicator turns orange and its tooltip changes to Canceled.

Canceling a flow step

To cancel a flow step for a communication:

  1. In Monitor view, on the Flows tab, select the flow instance that contains the communication for which you want to cancel a flow step.

    In the Properties panel, select the appropriate communication.

  2. Click the More actions button at the end of the row.

  3. Click Cancel step.

    The status indicator turns orange and its tooltip changes to Canceled.

In Tenant management view, you can use a Mark jobs for deletion scheduled task to delete any jobs in a canceled state.
