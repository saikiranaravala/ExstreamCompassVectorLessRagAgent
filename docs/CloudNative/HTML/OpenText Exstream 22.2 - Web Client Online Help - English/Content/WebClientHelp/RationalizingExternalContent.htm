Rationalizing external content

Exstream provides the ability to import and rationalize ZIP files that contain external content that you want to use in Communications Designer and Content Author communications. Because it is uniquely a design tool, Rationalization cannot be used in production domains.

Note: Licensing for Exstream CE Rationalization requires a services engagement with OpenText Professional Services. For more information, contact your Exstream account representative.

For example, suppose that you have a set of PDF output files that were generated using a different design tool, but which contain content that you would like to use in communications that you design in Exstream. To do so, create a ZIP file that contains the PDF output files and any XML driver files that were used to create the PDF output, and then use Rationalization asset library view to import this ZIP file as a migration set. You can then use Rationalization to analyze the imported files in the migration set, and identify the content fragments and variables in those files. Fragments can include either text or image content.

If you include XML driver files with your PDFs, Rationalization detects and includes variables in your rationalized content. If XML driver files are not included, the rationalization tool can suggest variables for use, if appropriate.

Migration sets can be linked to DAS data sources and to Communications Designer and Content Author communications.

  • For communications that use a DAS data source with a linked migration set, that migration set is also linked to the communication.

  • For communications that use a design pack, you can link a DAS migration set directly to the communication.

In Communications Designer and Content Author, you can then insert the fragments and variables from that migration set in your designs.

For more information about using content fragments in Communications Designer or Content Author, see OpenText Exstream Communications Designer Online Help or OpenText Exstream Content Author Online Help.

Adding a migration set

When you add a migration set, you are importing either PDF and XML driver files or template files within a ZIP file to make them available for Rationalization. After you add a migration set, you must rationalize the set in order to create the fragments that can be used in Communications Designer and Content Author.

To add a migration set:

  1. In the Migration sets panel in Rationalization asset library view, click Add.

  2. In the Properties panel, specify a name, and optionally, a description for the new migration set.

  3. In the Migration type list, select the source of the content to be rationalized:

    • PDF—If your source content comes from PDF output, you have the option to select a Communication type as well. This option allows you to designate the type of communication base—communication set or design pack—that you plan to use with the rationalized content. In turn, this can help you reduce the possible number of duplicate fragments that the rationalization process creates.

      If you select Design pack as the communication type, you can further narrow the duplicates by selecting a specific design pack that was imported to DAS in the Design pack list.

    • Template—This option is currently a placeholder for future platform formats such as StreamServe.

  4. Click Import files.

  5. In the file browser, select the ZIP file containing either your PDF output files or your template files, and then click Open.

    The contents of the ZIP file are listed in the Properties panel.

  6. Optionally, in the Current rationalization set tolerance area, use the slide or enter a value in the box to set the rationalization tolerance. This value determines how similar different fragments must be before Rationalization considers them to be a match—a higher number requires them to be more alike, and a lower number requires them to be less alike. You can also set this value before you rationalize the newly created migration set.

  7. Click Create.

In the Properties panel for the migration set, the status indicator for the set is gray. To use the content from this migration set in a design, you must first run the rationalization process to identify the content fragments in the imported files.

Rationalizing a migration set

After you have added a migration set, you must rationalize the content to create the fragments that can be used in Communications Designer and Content Author.

To rationalize a migration set:

  1. In the Migration sets panel in Rationalization asset library view, select a migration set.

  2. In the Properties panel, perform the following tasks:

    1. In the Current rationalization set tolerance area, use the slide or enter a value in the box to set the rationalization tolerance, and then click Save. This value determines how alike that different fragments must be before Rationalization considers them to be a match—a higher number requires them to be more alike, and a lower number requires them to be less alike.
    2. Click Run to initiate the rationalization process.

In the Migration sets panel, the indicator in the Status column changes to show the current status of the rationalization process. After the rationalization process is complete, you can click Open to open the migration set in Rationalization to identify and edit the content fragments in that set.

For information about how to work with rationalized content, see OpenText Exstream Rationalization Online Help.

Understanding the status indicators for migration sets

You can use the color indicators to quickly check the status for a migration set. You can point to any of the indicators to see a tooltip that provides more information.

The following list shows the indicator colors that are available for migration sets, and what each color means:

  • Gray: The migration set was created (or is in the process of being created), but it has not yet been rationalized.

  • Yellow: The migration set was rationalized but is not yet associated with a communication or data source.

  • Green: The migration set was rationalized and is associated with a communication or data source.

Tip: While rationalization is in progress, you can view the percent completed by positioning the pointer over the indicator.
