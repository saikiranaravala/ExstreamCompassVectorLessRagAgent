Setting up Empower output

After you create the output object, you can use the Properties panel to view and modify the Empower output object properties. After making changes, click Save, or the changes will be lost.

General settings

When you create Empower output, you can specify the output resolution to ensure that output is clear and readable. The output resolution specifies the sharpness of text and graphics, measured in dots per inch (dpi). If the Empower document is going to be printed, use a dpi at or lower than the maximum resolution supported by the output device you are using for the best processing performance.

In the Resolution (DPI) list, select the dpi setting you want to use from the available options.

If you select Custom in this list, in the Custom box, enter the custom image resolution you want to use. The minimum valid resolution you can set is 72, and the maximum valid resolution is 2400.

Editor settings
Hyperlink behavior

In the Hyperlink behavior list, select how text-based hyperlinks are opened within an Empower document.

Initial behavior of hidden items

The Initial behavior of hidden items setting controls whether hidden objects are visible or not when end users first open an Empower document. A show/hide icon will be available in the Empower document so that end users can toggle hidden content on or off as needed to customize the content.

Revision tracking

Revision tracking lets you follow changes to an Empower document throughout its life cycle. You can track changes made to an Empower document by a single user, but revision tracking is an especially useful tool when multiple end users are opening and editing the same Empower document.

There are a few things to keep in mind about using revision tracking in Empower documents:

  • Revision tracking is available for text boxes, table cells, paragraph objects, and variables in an Empower document. Note that formatted text variables, tagged text variables, and placeholder variables are not supported.
  • Empower Editor uses login credentials to track the revisions made by each end user and automatically assigns one of five different colors to each reviewer's revisions in order to help quickly differentiate between reviewers.
  • End users receive a warning in the Revisions pane if they add content that causes fixed-height text boxes and table cells to overflow.

In the Revision tracking list, select one of the following options:

  • User selectable—End users decide whether they want to track the changes they make. The revision tracking tools are available in Empower Editor, including the button, which turns tracking off and on. This option is the default.

  • Always on—Revision tracking is always on. The revision tracking tools are available in Empower Editor.

  • Always off—Revision tracking is always off. The revision tracking tools are not available in Empower Editor.

Allow user to add comments

Enabling the Allow user to add comments feature lets end users add comments to the editable text areas of an Empower document, so that end users can collaborate with co-workers on the changes that need to be made to a document before sending the final version to customers. Adding comments lets end users explain or add context to any changes that they make, or to ask questions about changes that other people have made. Additionally, end users can take part in discussions within an Empower document itself by adding comments to existing comments.
