Configuring Core Experience Insights integration

Important: This configuration is domain-specific. To use the same configuration in a different domain, you must recreate the configuration in that domain.

  1. In the Domains panel in Tenant management view, click the domain for which you want to enable Core Experience Insights integration.
  2. In the Properties panel, click the CXI configuration tab.
  3. Select the Enable Core Experience Insights integration check box.

  4. To specify Core Experience Insights connection information, use the following boxes:

    • Core Experience Insights URL—Enter the Core Experience Insights URL to which you want to post data.
    • Core Experience Insights client ID—Enter the client ID that is used to connect to Core Experience Insights.
    • Core Experience Insights client secret—Enter the client secret that is used to connect to Core Experience Insights.
    • Core Experience Insights tenant ID—Enter the Core Experience Insights tenant to which you want to post data.

  5. To specify the event schema information, use the following boxes:

    • Schema source—Enter a name that will be used to identify the schema source. This name must be unique for each domain.
    • Schema version—Enter the version of the schema.
  6. To enable audit logging for this Core Experience Insights integration, select the Enable audit logging check box.

  7. To clean up event data that has already been sent to Core Experience Insights from the database, select the Enable cleanup check box. In the Cleanup interval box that appears, enter the number of days to wait before purging data that has already been sent to Core Experience Insights.

    Note: By default, the cleanup cron job for the Experience Insights integration in your deployment runs every 50 minutes. You can change this frequency by using a Helm command to set the ei.schedule.cleanup parameter. For more information, see OpenText Exstream Installation and Configuration Guide.

  8. Click Save.
