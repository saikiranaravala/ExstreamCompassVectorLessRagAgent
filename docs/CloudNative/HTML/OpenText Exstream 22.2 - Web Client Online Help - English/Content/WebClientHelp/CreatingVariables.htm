Creating variables

When you map a data source, you can use variables that are included in the associated design pack, system variables, or variables that you create, either in the Communication asset library view or in the Mapping view.

Variables that are created in the Communication asset library view or in Mapping view appear in italic in the Variables panel of the Mapping view. This distinguishes them from variables that have been imported from a design pack (which are displayed as non-italic text).

After you create a variable, you can use the Properties panel to view and modify its properties to further define it.

When you create a variable, it then becomes available to all data sources within Communications Designer. If you change the variable properties, the changes carry over to all instances of the variable.

Once a variable is mapped to a data source, it is then available to any communication that is associated with that data source.

Creating a new variable in Mapping view

You can either create a new variable from the toolbar, or you can create and map a new variable to a specific data area.

  1. To create a new variable, select one of the following options:

    • On the toolbar, click the New variable button .

    • Point to the data area that you want to map, then click the Add new mapping button .

  2. In the New variable dialog, specify a name, and optionally, a description for the variable.

  3. In the Type list, select a variable type.

    If you are creating a placeholder variable, in the Placeholder type list, select the type of file that you want to import at run time.

  4. Click Create.

Creating a new variable in the Communication asset library panel

  1. In the Data panel, click the Add button, then click Variable in the selection list.

  2. Specify a name, and optionally, a description for the variable.

  3. In the Type list, select a variable type.

    If you are creating a placeholder variable, in the Placeholder type list, select the type of file that you want to import at run time.

  4. Click Create.

Duplicating a variable

If you want to create a copy of a variable, you can duplicate it, rather than starting from defaults. This might be useful if you have highly customized the properties of a variable, and need another variable with only minor modifications.

To duplicate a variable, in the Communications asset library panel, in the Data panel, select the variable that you want to copy and click Duplicate.

Viewing the references for a variable

You can use the References tab in the Properties panel to see all of the linked resources for a selected resource, if applicable.

To see the references for a resource:

  1. In the asset library panel, select a resource.

  2. In the Properties panel, click the References tab.

  3. To display a list of references for the selected resource, click the following buttons:

    • Used by—The list shows all of the parent resources that use the selected resource. Keep in mind that this list displays only parent resources with a direct link to the selected resource. Resources that use the parent resources are not listed. This option is selected by default.

    • Uses—The list shows all of the child resources that are used by the selected resource, including resources that are used by each child resource.

  4. To filter the displayed list by resource type, click the resource type selection list and select from the available options. You can select as many resource types as you want. To select all types, simply clear all existing selections.

By default, the first 50 items are displayed in the Properties panel. To change the number of items displayed in the panel, use the Items per page list. If the number of resources is too large to display in the panel, you can navigate the resource list with the page navigation buttons .

Deleting a variable in Mapping view

If a variable is no longer needed, you can delete it if it is not currently used in any data sources or communications.

Note: You cannot delete variables that come from a design pack.

To delete a variable in Mapping view:

  1. Open a data source.
  2. In the Variables panel, select the variable you want to delete.
  3. In the Variable properties panel, click the Delete button .

Tip: If the Delete button is not active, you can click the Used by card in the Variable properties panel to see where the selected variable is used. Variables that are used in current or approved data sources or communication objects cannot be deleted.

Deleting a variable from the Communication asset library panel

  1. In the Data panel, select the variable you want to delete.

  2. Click Delete.

Tip: If the Delete button is not available, you can click the References tab in the Properties panel to see where the selected variable is used. Variables that are used in current or approved data sources or communication objects cannot be deleted.
