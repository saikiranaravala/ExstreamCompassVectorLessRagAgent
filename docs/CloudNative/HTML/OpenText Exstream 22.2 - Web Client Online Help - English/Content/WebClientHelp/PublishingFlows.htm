Publishing orchestration flows

Flow models and flow contexts determine the manner in which the engine processes communications or package files to produce customer output.

Flow models and flow contexts are created in the Orchestration asset library view and configured using Communications Orchestrator. For more information about configuring these resources, see OpenText Exstream Communications Orchestrator User Guide.

Publishing a flow model or a flow context makes the resource available for processing by the Orchestration service in the following way:

  • Publishing a flow context activates its settings in the Orchestration service. This means that the input and output channels, outcome mappings, and communications that are defined in that flow context can be used in published flow models.

  • Publishing a flow model activates it for execution in the Orchestration service. This means that data sent to an input channel used in a published flow model will trigger the execution of that flow model. Publishing a flow model also generates engine run definitions for communications that are used in that flow model. Engine run definitions are provided to Document Generation to run the Exstream engine. When you publish a flow model that uses settings from multiple flow contexts, make sure that you have first published all of those flow contexts.

Note: Regardless of the resource version that is selected, publishing a flow model or flow context always makes the latest approved version of the resource available to the Orchestration service. When you publish a resource, review these considerations:

  • To publish the latest version of the resource, make sure that you approve the resource before you publish.
  • To publish an older version of the resource, promote that version of the resource and approve it before you publish.
Publishing a flow model or flow context

When you work with versioned resources, the publishing behavior is different for flow models and flow contexts:

  • For flow models, if you publish the latest version of a resource while a previous version is in a published state, the previous version will be automatically unpublished.

  • For flow contexts, if you publish the latest version of a resource while a previous version is in a published state, then both versions are available in published state.

Important: To ensure correct flow execution, make sure that you unpublish previous versions before you publish the latest version of a flow model or a flow context. Otherwise, you might see unexpected results.

To publish a flow model or flow context:

  1. In Orchestration asset library view, in the Flows panel, select a flow model or flow context.

  2. On the General tab in the Properties panel, click Publish.

Unpublishing a published resource
  1. In Orchestration asset library view, in the Flows panel, select the resource.

  2. On the General tab in the Properties panel, click Unpublish.

Considerations for deleting flow resources

Note: You cannot delete a flow model or a flow context if any version of that resource that is in a published state.

When only one version of an unpublished flow model or flow context exists, you can simply click Delete in the Properties panel in Orchestration asset library view to delete the resource from the asset library.

When multiple versions of a flow model or flow context exist, you can choose to delete either a specific version that you select in the version selector in the Properties panel, or you can choose to delete all versions of the resource. To do so, select the Delete all versions check box in the confirmation dialog box that appears when you click Delete.

When you unpublish and delete the latest version of a flow model or flow context, you must then republish the previous version of the resource if you want to make it available to the Orchestration service.
