Setting up Messaging output

After you create the output object, you can use the Properties panel to view and modify the Messaging output object properties. After making changes, click Save, or the changes will be lost.

Encoding

When you create Messaging output, you can use the following encodings:

  • UTF-8—A byte-oriented format that uses at minimum 1 byte (8 bits) when encoding characters. UTF-8 is backwards compatible with ASCII. Compared to UTF-16, it is also better at recovering from errors and output tends to be smaller.

  • UTF-16—A non-byte-oriented format that uses at minimum 2 bytes (16 bits) when encoding characters. Since it is not byte-oriented, it needs to establish a byte order in order to work with byte-oriented networks. UTF-16 is incompatible with ASCII. Output is roughly twice as big as the same file encoded with UTF-8. There are 3 options for UTF-16 output:

    • UTF-16 (Native)—Uses the original byte order of the host operating system

    • UTF-16 (Big-endian)—Outputs the content in big-endian order

    • UTF-16 (Little-endian)—Outputs the content in little-endian order

    Note: While the Exstream engine can generate SMS text messages that use UTF-16 encoding, the OpenText Notifications Service delivers only messages that use UTF-8 encoding.
