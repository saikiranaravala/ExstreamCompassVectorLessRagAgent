Creating customer journeys

Core Experience Insights allows you to run analytics on your event data and create visualizations that summarize customer journeys through business processes.

For example, supposed when you create and send an Exstream email communication that includes embedded TeamSite links, Exstream sends event data for that email to Core Experience Insights insights. In addition, when a customer clicks an embedded TeamSite link in that email, then event data from clicking those links can also be sent to Core Experience Insights from TeamSite. You could then use common metadata for Exstream and TeamSite data points (such as the campaign ID) to visualize those data points in a single Core Experience Insights journey.

At a high-level, creating customer journeys requires the following steps:

  1. In Core Experience Insights, create a new journey visualization.

  2. In the journey, add new data points, filter the data using rules, and then present the data.

    • Adding data points—The schemaType and the schemaSource that are specified in the event JSON are the primary attributes from which to build data points in Core Experience Insights. For example, the source can be a specific campaign ID and the type can be emails opened.

    • Adding rules—Rules let you filter the information that is available in data points and can be applied to the attributes that are included in the event JSON.

    • Presenting data—The filtered data is presented using dashboards and reports.
  3. Activate the journey.

For detailed information about creating customer journeys, see the Core Experience Insights documentation.
