Updating engine run definitions

Engine run definitions provide instructions to the Exstream engine for producing output, and contain the following information:

  • Engine version
  • Input files and output queues
  • Switches for customizing the engine run

Depending on their purpose, run definitions can be of the following types:

  • Production run definitions—Created when you publish a flow model. Production run definitions are required for orchestration jobs that run the engine to produce customer output and are included in the flows that are processed by the Orchestration service. The run definition for a communication in a flow model is based on the corresponding communication settings that are defined in a flow context.

    For information about defining communication settings in Communications Orchestrator, see OpenText Exstream Communications Orchestrator User Guide.

  • Preview run definitions—Created either when you package and upload a Design Manager application that contains a design pack or when you create a communication set. Preview run definitions contain the information that is required to run the engine to generate previews for Content Author and generate simulations for Communications Designer or Content Author communications that are based on the corresponding application template. Preview run definitions are configured in DAS and can be modified in the Exstream web client.

To modify a preview run definition:

  1. In the Run definitions panel in the Orchestration asset library view, select a run definition.

  2. In the Properties panel, update the following settings as needed:

    Option Description

    Name

    The name of the run definition

    Description

    The description of the run definition

    Data source associations

    The sample data files that are associated with the run definition.

    To update an associated file, click the Update <type> file button and select a different sample file. Make sure that you select a file of the appropriate file type.

    Engine

    The engine version that is used to generate output. Click the list and select the engine version that you want to use.

    Note: The engine version must match or be later than the version that is specified in the package file that will be processed using this run definition.

    Engine switches

    The engine switches that are used to customize the engine run. By default, the run definition for a communication created by a template contains at least the following required switches:

    • FILEMAP—Specifies the input file that is used to produce output. By default, the sample file that was included the Design Manager application is used.
    • PACKAGEFILE—Specifies the associated Design Manager application package.

    To add a new switch, click in the Add new switch box and enter the switch and any required arguments. To remove a switch, click the Remove button .

    Note: Some engine switches are not supported when you use the Document Generation service to run the engine. For additional information about engine switches, see Switch reference.
