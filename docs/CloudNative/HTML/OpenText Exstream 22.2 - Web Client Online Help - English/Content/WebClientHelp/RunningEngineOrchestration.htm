Running engine orchestration

Use this section as a high-level overview to define the engine orchestration jobs to run the production engine and produce customer output.

Overview of engine orchestration
Step Explanation Documentation

Step 1: Create and approve the flow context, flow model, and flow scripts (if used)

 Orchestration flow models and flow contexts determine the manner in which engine runs are processed to produce customer output. Flow scripts can be assigned to flow context items as before and after events, and to script processors.

Adding orchestration flow resources

Step 2: Publish the approved flow context and flow model

Publishing makes the flow model and the flow context available for processing by the Orchestration service, and creates the engine run definition that is used by the Document Generation service to run the engine as determined from the flow model.

Publishing orchestration flows

Step 3: Initiate the orchestration job

 You initiate orchestration jobs by sending input data to an input channel. This initiates new orchestration jobs with flow instances of any published flow models that use the input channel.

Initiating orchestration jobs

Step 4: Monitor the orchestration job

From Monitor view, you can check the status of your orchestration jobs and access the files that are produced by the engine.

Monitoring orchestration jobs

Note: If you are using the OT2 platform, Orchestration asset library view and Monitor view are only available if the Exstream subscription is set to Standard (not Base).
