Configuring communication sets

You can use the asset library panel in the Communication asset library view to add communication sets to the asset library for use with a communication. A communication set is a resource that combines other communication resources into one collection. When you create a communication, you can associate it with a communication set, allowing for greater reuse if you want multiple communications to use the same set of resources. A communication set takes the place of a design pack when creating a Communications Designer communication, thus alleviating the need to make resource updates in Design Manager.

Communication sets can also be used to create communications that are the basis for Content Author templates.

In order to use a communication set with a communication, it must first be in an approved workflow state. All child resources of the communication set must be approved first.

Communication sets can include the following communication resources:

  • Data sources

  • Locales

  • Output queues

  • Variables

  • Variations

Communication set limitations

The following are some current Communications Designer limitations when using a communication set instead of a template with a communication:

  • Barcodes are not supported

  • Custom colors in a design are unavailable

  • Library rules are unavailable

  • Reference files are not supported when fulfilling Empower documents

Creating a communication set

  1. In the Communication sets panel, click the Add button.

  2. Specify a name, and optionally, a description for the communication set.

  3. Click Create.

  4. Select the communication set and click Open.

  5. Click Add, then select the type of communication resource you want to add.

  6. In the Properties panel, select one or more of the listed communication resources, then click Insert.

    To select multiple resources, press and hold CTRL, and then select the resources that you want to add.

    Note: Only one data source of the type Driver can be added to a communication set.

  7. Repeat steps 5–6 for each communication resource you want to add.

  8. Click Close.

Setting a primary queue

If you have more than one output queue in a communication set, the first queue added is set as the primary by default. However, you can designate any added queue as the primary. Exstream processes the primary output queue first, then any additional output queue objects in the order in which they were added to the communication set.

As you consider the order of output queues, keep in mind that the output driver associated with the primary queue controls not only the settings for the primary queue, but also most settings for the secondary queues. You can, however, control settings for character fidelity and resource management on a per-queue basis. For example, if the character fidelity settings in the output driver for a secondary queue differ from those on the output driver for the primary queue, the engine honors the settings for the secondary queue.

To change the primary queue:

  1. In the Output queues category, click the More actions button at the end of the row of the queue you want to be the primary.

  2. Select Primary.

Including variation variables

To enable the use of variations in a communication, you have to include at least one variable that includes a static selection list in the communication set. To create a variation in Communications Designer, the communication set must include a variation variable with at least two selection list items.

Note: If you create a language or locale object, its name is automatically added as a static selection list item in the 'SYS_LanguageCustomer' or 'SYS_LocaleCustomer' system variable. If the 'SYS_LanguageCustomer' or 'SYS_LocaleCustomer' variable is then added to a communication set as a variation, the languages or locales can be used to create design variations.

For more information on using variations in a communication, see OpenText Exstream Communications Designer Online Help

Including locales

If you have variables or variations that use locales to differentiate data, OpenText recommends that you include all of the appropriate locale objects in the communication set that the communication is based on. This ensures that they are available to the engine during production runs.

Making resources available in Content Author

Certain resources within a communication set, such as variables and variations, can be designated for use with Content Author. When a Content Author template is created from a Communications Designer communication using the communication set, these resources will then be available in the Authoring view of Content Author.

Designating a resource for use with Content Author is specific to each communication set. That is, if a resource is associated with multiple communication sets, it must be separately assigned for use with Content Author within each communication set.

For more information about using variables or variations in Content Author, see OpenText Exstream Content Author Online Help.

To allow the use of a resource within Content Author:

  1. Click the More actions button at the end of the row of the resource you want to include.

  2. Click Include for Content Author.

  3. Repeat step 1 through step 2 for each additional resource you want to include.

Removing an asset from a communication set

To delete an asset from a communication set:

  1. Click the More actions button at the end of the row of the asset you want to remove.

  2. Click Remove.

Note: You cannot remove system or data source variables.

Duplicating a communication set

If you want to create a copy of a communication set, you can duplicate it. This might be useful if you want a similar communication set with only minor modifications.

To duplicate a communication set, in the Communication set panel, select the communication set that you want to copy and click Duplicate.

Deleting a communication set

If a communication set is not needed, and is not used in any communication objects, it can be deleted from the asset service.

To delete a communication set, in the Communication sets panel, select the communication set that you want to delete and click Delete in the Properties panel..

Tip: If the Delete button is not available, click the Used by button in the References tab in the Properties panel to see if the selected object is used in the current version of a communication object.
