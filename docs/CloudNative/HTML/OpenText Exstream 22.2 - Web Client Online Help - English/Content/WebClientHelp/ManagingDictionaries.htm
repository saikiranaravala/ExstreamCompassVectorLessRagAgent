Managing dictionaries for spell checking

Exstream comes with a set of language-specific dictionaries that it uses for spell checking content in Communications Designer and Content Author. The dictionary language can be set from within Communications Designer and Content Author. To add support for dictionaries that are not included in the Exstream deployment, administrators can use the Configurations tab for Communications Designer and Content Author in Tenant management view to upload language-specific Hunspell dictionary files so that they are available in each product.

For information about how to obtain or create Hunspell dictionary files, refer to the Hunspell documentation on the SourceForge website.

Adding a dictionary for Communications Designer or Content Author

You can add Hunspell dictionary files to make any dictionaries that are not included in the Exstream deployment available to users in Communications Designer and Content Author. Each Hunspell dictionary uses two files: the .dic file that contains the word list, and the .aff file that contains the list of rules and other options.

To add a dictionary for Communications Designer or Content Author:

  1. Sign in to the Exstream web client as a tenant administrator user at the following URL:

    https://<host>:<optionalPort>/design?tenant=<tenantName>

  2. Click the Tenant management button to open the Tenant management view.

  3. In the Configurations panel, click Content Author and Communications Designer.

  4. On the Dictionaries tab of the Properties panel, click Add.

  5. In the following fields, add the necessary files and information:

    • Word list file (*.dic)—Upload the word list file for the spelling dictionary that you're adding.

    • Affix file (*.aff)—Upload the rules and options file for the spelling dictionary that you're adding.

    • Language code—Specify the language code for the dictionary that you're adding.

      Important: If you are adding a German dictionary and you also use Assisted Authoring in your deployment, then you must specify de_DE as the language code. If you use a different language code, then Assisted Authoring will not work for German content.

    • Dictionary name—Specify a name for the dictionary. The name that you specify will appear in the list of dictionaries in Content Author and Communications Designer.

  6. Click Add.

The dictionary is now available in Content Author and Communications Designer. Note that you cannot edit an installed dictionary—if you need to change a dictionary that you've installed, then you must remove it and then add it again. To remove an installed dictionary, simply select the dictionary from the Installed dictionaries list, click Remove, and then verify the removal. You cannot remove dictionaries from the Existing dictionaries list.

Dictionaries that are no longer included with Exstream

Because of licensing restrictions, the following dictionaries are not included with your Exstream deployment:

  • Czech (cs_CZ)

  • Danish (da_DK)

  • German (de_DE)

  • Hebrew (he_IL)

  • Hindi (hi)

  • Indonesian (id_ID)

  • Italian (it_IT)

  • Malay (ms_MY)

  • Norwegian Bokmal (nb_NO)

  • Norwegian Nynorsk (nn_NO)

  • Tagalog (tl_PH)

  • Vietnamese (vi_VN)
