Enabling Core Experience Insights integration

OpenText Core Experience Insights allows organizations to map customer journeys and track customer activity for every event, across all customer journey touchpoints. The solution helps identify which events are working and which need improvement. To let users view event data for Exstream communications, you must integrate your Exstream environment with Core Experience Insights. Event data, which is sent to Core Experience Insights as a JSON file, can include information such as the number and types of documents that are generated by Exstream and user-interaction information for output delivery channels.

To enable Exstream to send data to Core Experience Insights:

  1. Use a Helm command to enable the Core Experience Insights integration service within the Exstream deployment. For information about deploying Exstream, see OpenText Exstream Installation and Configuration Guide.
  2. In Tenant management view in the Exstream web client, configure Core Experience Insights integration for each domain from which you want to send data. For more information, see Configuring Core Experience Insights integration.
  3. In Communications Orchestrator, enable document tracking for communications and output channels. For more information about enabling tracking, see OpenText Exstream Communications Orchestrator User Guide.

By default, the event cron job for the Experience Insights integration in your deployment runs every 10 minutes to send event data from Exstream to Core Experience Insights. You can change this frequency by using a Helm command to set the ei.schedule.event parameter. For more information, see OpenText Exstream Installation and Configuration Guide.

To visualize the data points that are sent from Exstream, you can create customer journeys in Core Experience Insights. For more information about creating customer journeys, see the Core Experience Insights documentation.
