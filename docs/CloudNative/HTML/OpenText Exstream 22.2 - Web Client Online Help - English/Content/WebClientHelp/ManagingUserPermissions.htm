Configuring roles and permissions

The Exstream deployment uses OTDS for user authentication. To sign in to the Exstream web client, a user must be a member of an OTDS group that is assigned an Exstream role with the required permissions.

Roles are always tenant-specific; however, access permissions are defined separately for each domain in a tenant. For each domain in a tenant, you configure user permissions by adding OTDS groups and assigning roles to those groups. When a user signs in to the Exstream web client, their level of access is determined by the roles that are assigned to the OTDS groups to which they belong. For a user in multiple OTDS groups, the level of access is defined by the superset of permissions for all of the groups to which the users belongs.

Depending on which features (design tools) are associated with the product key, the process of allocating users to a product key also grants them permission to utilize one of the Named User Seats (known as reserved seats in OTDS) for the web client design applications (Communications Designer, Content Author, and Empower). However, it does not automatically secure the seat until the user accesses a feature for the first time. Once a user is allocated to a seat, that seat stays reserved to that user unless revoked by an administrator in OTDS.

Note: Deleting a user does not automatically release the seat that is assigned to that user. A seat must be revoked in order for the seat to be available for reallocation.

For specific information about how licensing works with Exstream, see OpenText Exstream Design and Production Online Help. For information about OTDS users, groups, and partitions, or how to allocate or revoke reserved seats, see OpenText Directory Services Installation and Configuration Guide.

Creating a new role in a tenant
  1. In the Roles panel in Tenant management view, click Add.
  2. In the Properties panel, specify a name for the new role and optionally, a description, and then click Create.

  3. (Optional) To set specific resource permissions, expand the Advanced section in the Resource access area, then select Create, Write, Delete, and Approve permissions for individual resource types as needed.

    Tip: To apply a selected permission option to all resources in the DAS asset library, select the All check box in the Resource type list.

    Keep in mind that you do not need to explicitly set permissions to read resources in a domain. A user that is assigned a role that has any permission in a domain, for example, Access permission for Design asset library—or even no permission at all—will also have read access to all resources in that domain.

  4. Click Save.
Duplicating a role

To duplicate a role, select the role in the Roles panel in Tenant management view, and then click Duplicate.

When you duplicate a role, the properties of the new role are identical to the role that you duplicated. However, any role assignments for the existing role in any domain are not duplicated for the new role. To assign this role to users in a domain, use the Group assignments tab in the Properties panel for that domain.

Deleting a role

To delete a role, select the role in the Roles panel in Tenant management view, and then in the Properties panel, click Delete.

If a role is assigned to any OTDS groups for any existing domains in the tenant, then those role assignments will be removed when you delete the role.
