Creating configurations to monitor orchestration jobs

You can use the Configurations panel in Tenant management view to set up scheduled tasks that process completed orchestration jobs on a regular basis. The scheduled tasks that you set up use standard cron expressions to either delete completed jobs or to wipe already deleted jobs. Wiped orchestration jobs, along with their flow instances and flow steps, are fully deleted from the database.

By default, the cleanup cron job for the orchestration service in a deployment runs every 60 minutes, and is configured in the helm chart when you deploy Exstream. You must ensure that the the cron expressions for your scheduled tasks are in sync with the cleanup cron job for the orchestration service, or else you will see unexpected results. If you have questions about the timing of the cleanup cron job for your deployment, see your system administrator. For more information about making helm chart configurations, see OpenText Exstream Installation and Configuration Guide.

To create configurations to monitor orchestration jobs:

  1. In Tenant management view, on the Configurations panel, click Monitor.

  2. On the Properties panel, on the General tab, click the plus button .

  3. In the Scheduled task dialog, specify a name for your task, and then specify a cron expression and a type of action:

    Mark for deletion

    When you select Mark for deletion, any qualifying jobs are tagged for removal but not removed from the database. When you select the Mark for deletion option, the following additional options become available:

    • Start—The Start options let you specify the start dates for the orchestration jobs that you want to process.

      • More than ... days before—Specify the number of days that an orchestration job started before the scheduled task.

      • Fixed dates—Select a specific range of dates during which an orchestration job started. When you select this option, you can use the calendar controls in the From and To fields to select specific dates.

    • Finish—The Finish options let you specify the completion dates for the orchestration jobs that you want to process.

      • More than ... days before—Specify the number of days that an orchestration job completed before the scheduled task.

      • Fixed dates—Select a specific range of dates during which an orchestration job completed. When you select this option, you can use the calendar controls in the From and To fields to select specific dates.

    • Mode—The Mode options let you specify which types of orchestration jobs should be processed whenever the scheduled task runs.

      • Any—Select this option to process all jobs (batch and on-demand) that meet the criteria for the scheduled task.

      • Batch—Select this option to process only batch jobs that meet the criteria for the scheduled task.

      • On demand—Select this option to process only on-demand jobs that meet the criteria for the scheduled task.

    • Statuses—You can use the Statuses options to specify whether to process jobs that ran successfully or unsuccessfully as part of your scheduled task. Both options will be processed if you select neither check box or both check boxes.

      • Complete—Select this option to process only orchestration jobs that completed successfully.

      • Error—Select this option to process only orchestration jobs that encountered errors when they ran.

    Wipe

    This action removes from the database any qualifying jobs that have previously been marked for deletion. Additionally, wiping a job deletes any files and folders from NFS that are associated with the job. This option runs according to the cron expression that you specify and does not have additional settings associated with it.

  4. Click OK, and then click Save.

After you've set up a scheduled task, you will see it in the list of tasks on the Properties panel for Monitor configurations. You can toggle a task on or off by clicking the slider in the Enabled column for the task. Additionally, when you point at the task on the Properties panel, you can click the options button to edit or remove the task.

Supported cron expressions

Cron expressions for scheduled tasks must have five fields (minutes, hours, day of month, month, day of week). The task scheduler supports the following standard cron expressions:

Field Required Values Special characters
Minutes Yes 0-59 * , -
Hours Yes 0-23 * , -
Day of month Yes 1-31 * , -
Month Yes 1-12 or JAN-DEC * , -
Day of week Yes 0-6 or SUN-SAT * , -

Tip: You can combine slashes ( / ) with ranges to specify incremental values. For example, */5 in the minutes field indicates an event that happens every 5 minutes. Exstream does not support other non-standard characters for cron expressions.

Examples of cron expressions

The cron expression for each scheduled task must be in sync with the cleanup cron job for the orchestration service in order to be executed. If you have questions about the timing of the cleanup cron job for your deployment, see your system administrator. The following table provides some examples of how cron expressions for scheduled tasks are actually executed based on the frequency of the cleanup cron job for the orchestration service.

Cleanup cron job frequency Cron expression for scheduled task Expected result Actual behavior
Every 60 minutes (at minute 0) 0 * * * * Runs at minute 0 of each hour Runs at minute 0 of each hour
*/5 * * * * Runs every five minutes Runs once each hour, at minute 0, because the cleanup cron job runs only at minute 0
5 * * * * Runs every hour, at minute 5 Never runs, because the cleanup cron job runs at minute 0, not minute 5
Every 15 minutes (at minute 0, 15, 30, and 45) */15 * * * * Runs every 15 minutes Runs every 15 minutes
0 * * * * Runs at minute 0 of each hour Runs at minute 0 of each hour
*/20 * * * * Runs every 20 minutes, starting at minute 0 Runs once an hour, at minute 0, because the cleanup cron job does not run at minutes 20 and 40

Assuming that you use the default timing for your cleanup cron job (every 60 minutes, at minute 0), the following sample cron expressions might be useful when setting up scheduled tasks:

Cron expression Timing for scheduled task
0 2 * * 6 Runs at 2:00 a.m. every Saturday
0 2 * * 1-5 Runs at 2:00 a.m. every Monday-Friday
0 23 * * * Runs daily at 11:00 p.m.
0 2 1 * * Runs on first day of the month at 2:00 a.m.
0 1 */14 * * Runs twice a month, on day 14 and day 28, at 1:00 a.m.
0 1 1,15 * * Runs twice a month, on day 1 and day 15, at 1:00 a.m.
