Prerequisites for using Content Author

Content Author lets business users add targeted content to communication designs that are created in Design and Production or in Communications Designer. To use Content Author, these communications must be set up correctly and available to Content Author users.

The following table provides an overview of the minimum requirement to enable business users to edit Content Author communications:

Prerequisites for using Content Author
Requirement Explanation Documentation
A communication design that contains Content Author frames

In Content Author, users can add content only to those frames that are designated for use with Content Author. The communication designs that are used to create Content Author templates must include at least one Content Author frame. Communication designs can be created in the following ways:

  • Using Communications Designer—Create design layouts in a Communications Designer communication.

  • Using Design and Production—Create a Design Manager application and use Designer to create the design layout. The Design Manager application must contain the following objects:

    Design pack

    Design packs contain resources for customizing a communication, such as fonts, style sheets, colors, rules, and variables. The Design Manager application must contain at least one design pack.

    Design views

    Design views determine which preview options are available in Content Author. Each design view corresponds to a specific preview device. By default, applications are automatically configured to use the standard design view for print preview.

    Output queues

    Output queues determine the output types that can be generated when the communication is processed using the Exstream engine.

    Data files

    At least one customer driver file is required to produce output from the communication.

  • OpenText Exstream Design and Production Online Help

An approved template in the DAS asset library for use with Content Author

To create a new Content Author communication, users select an approved template in Communication launcher view to create a new communication. Content Author templates are created in the following ways:

  • Using Communications Designer—Create a Content Author template in Communications Designer.
  • Using Design and Production—Package and upload an appropriate Design Manager application to the DAS asset library to create a corresponding template and engine run definition for that application. An engine run definition is required for communication previews.

Starting a Content Author communication

User permissions to access and work in Content Author

Content Author users must be members of at least one OTDS group that is assigned a role with at least the Access and Creator web client access permissions enabled for Content Author. For reviewers, the Approver option must also be enabled.
