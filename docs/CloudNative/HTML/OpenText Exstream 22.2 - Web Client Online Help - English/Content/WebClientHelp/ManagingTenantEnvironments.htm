Managing roles and domains in a tenant

Note: To manage roles and domains, you must be a member of the tenantadmins group in the corresponding OTDS tenant.

As a tenant administrator, when you first sign in to a tenant, you must create domains that let users logically separate resources in the DAS asset library for that tenant. You need at least one domain to enable users to work in the tenant. You must also configure the roles that define the access permissions for applications and resources in that tenant. You use the Tenant management view in the Exstream web client to manage your tenant environment.

To open the Tenant management view for a tenant, sign in to the Exstream web client as a tenant administrator, then click the Tenant management button on the tenant home page or from the application toolbar on the domain home page.

Note: You cannot change the access permissions that are required to open the Tenant management view. Any tenant administrator user who will create domains and assign user permissions in a tenant must always be a member of a tenantadmins group in each tenant that they want to manage. For information about creating OTDS groups and users, see OpenText Directory Services Web Client Help.

The Tenant management view is domain-agnostic. That is, the options that are available in this view are not dependent on the user permissions defined for a specific domain. A tenant administrator user can specify access permissions for that domain even if they do not have access to any applications in that domain.
