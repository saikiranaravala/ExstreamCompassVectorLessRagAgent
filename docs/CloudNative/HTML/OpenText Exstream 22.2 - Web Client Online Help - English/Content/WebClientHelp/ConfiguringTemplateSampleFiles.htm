Configuring sample files for a template

To provide a convenient way to preview communications with different variable values, the Exstream web client supports uploading sample files for use with Communications Designer and Content Author. This functionality is especially useful when previewing a communication that might change when the sample customer data changes, such as when regulations in a particular state require additional content to be included in a communication.

Keep in mind that sample files are not used in engine orchestration jobs; they are used only to map data sources, generate previews for Content Author, and generate simulations for Communications Designer and Content Author.

Associating a sample file with a template

While you can associate sample files with individual communications, associating them with a template ensures that all communications created from that template have access to the same sample files.

For information about associating sample files with a Communications Designer communication, see OpenText Exstream Communications Designer Online Help.

For information about associating sample files with a Content Author communication, see OpenText Exstream Content Author Online Help.

To associate a sample file with a template:

  1. In Orchestration asset library view, in the Templates panel, select a template that is in draft state.

  2. On the General tab of the Properties panel, in the Sample files section, click Add.

  3. In the Sample file dialog box, select a sample file to add and then click OK.

    Note: Only sample files with the Driver file type can be associated with a template.

  4. Click Save.

Setting a default sample file

When you package and upload a Design Manager application to DAS, the sample data file that is defined in that application is uploaded to DAS, and it is associated with the application template as the default sample file. You can associate multiple sample files with a template, but each template must have a default sample file specified. When you open a communication created from a template with multiple sample files, the default sample file is used to populate variable data until you switch to a different sample file.

To set a default sample file:

  1. In Orchestration asset library view, in the Templates panel, select a template that is in draft state.

  2. On the General tab of the Properties panel, in the Sample files list, select a sample file and then click Make default.

  3. Click Save.

Removing an associated sample file

  1. In Orchestration asset library view, in the Templates panel, select a template that is in draft state.

  2. On the General tab of the Properties panel, in the Sample files list, select a sample file and then click Remove.

    Note: If you want to remove the default sample file, you must first set a different default sample file on the template.

  3. Click Save.
