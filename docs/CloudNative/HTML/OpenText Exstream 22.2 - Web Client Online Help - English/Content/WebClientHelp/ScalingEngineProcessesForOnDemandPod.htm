Scaling on-demand engine processes

When you deploy Exstream, the maximum number of engine processes that can be run in the Document Generation On-Demand pod is not specified by default, and is controlled by the number of CPUs that are available in your Kubernetes node.

However, you can use an environment variable to control the maximum number of engine processes that the On-Demand pod can run simultaneously. When the number of engine processes that you specify has been reached, the On-Demand pod queues any subsequent requests to run the engine. Then, as engine slots become available, the On-Demand pod processes those requests.

To control the maximum number of on-demand engine processes, do one of the following:

  • Run a Helm upgrade command that uses the --set flag to provide a positive integer value for the ondemand.maxConcurrentEngineProcesses parameter. To retain your previous deployment configuration, make sure that you also use all of the other parameters that you used in your initial Helm install command.
  • Edit the exstream-ondemand-config configuration map in Kubernetes to add the MAX_CONCURRENT_ENGINE_PROCESSES environment variable value in the data section and set a positive integer as the variable value.

For information about setting the maximum number of on-demand engine processes during deployment, see OpenText Exstream Installation and Configuration Guide.
