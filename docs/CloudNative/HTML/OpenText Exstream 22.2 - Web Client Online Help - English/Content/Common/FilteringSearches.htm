Filtering searches

When you search in Exstream, you can click the arrow in the Search box to open the Advanced search options dialog box and then select filters to apply to your search results. The filters limit your results to content that matches the search term that you enter in the Search box (if any) and the filters that you select.

Note: Search results include matches found in content properties only, such as name and description, unless your deployment supports the filter Search text within content.

Applying search filters

The search filters that you can apply depend on the content that you search for and where you search for that content.

To apply search filters:

  1. (Optional) Enter a search term in the Search box.

    • A valid search term can contain letters, numerals, spaces, and the following characters:

      ~ ` @ $ ^ ( ) _ - = ; " ' , .

    • A search term cannot contain any of these characters:

      ! # % & * + | \ / < > : ?

      Tip: If the Search box outline is red and the search button is unavailable, remove invalid characters from the search term and try your search again.

  2. Click the arrow in the Search box.

  3. In the Advanced search options dialog box, select the search filters to apply, and then click Search at the bottom of the dialog box.

    In the Search box, the search button changes to the filtered search button to indicate that one or more search filters are applied.

  4. To search again with the same filters applied and a new search term, enter the new term in the Search box and then click the filtered search button .

Advanced search options

You can select one or more of the following search filters in the Advanced search options dialog box:

Search filter

Affect

Case sensitive search

Limits results to content that matches the case of letters in the search term.
Show unavailable content Extends results to include content that you cannot access. For example, the results can include older versions of content and content that is reserved by a different user.
Search text within content

Limits results to content that contains text that matches the search term. For example, if the search term is a word or phrase, the results can include messages, clauses, and sections that contain that word or phrase.

If your deployment supports this filter, you can search for text within content in the following views only:

  • Design asset library view

  • Structure view

  • Authoring view, when you add or edit content in the Content Composer dialog box.

Note: If you do not see this filter, contact your Exstream administrator about adding it to your deployment.

The Search text within content filter connects your Exstream deployment to an instance of Apache Solr™ running in the Apache Solr cloud.

For more information about Apache Solr, see the Apache Solr Reference Guide. You can download this guide from the Apache Solr website at solr.apache.org

For more information about integrating Apache Solr with Exstream, see the OpenText Exstream Installation Guide.
Workflow state Limits results to content that matches the selected workflow state or states: Draft, Submitted for approval, Approved, or Rejected.
Quick tags Limits results to content that has the selected category tags or metadata tags applied.
My objects

Limits results to content that you created, reserved, or last updated.
